## 开张核实事实底稿（2026-07-09 联网核实）

### 身份与治理
- A2A = **Agent2Agent Protocol**，智能体间互操作的开放协议。
- Google 于 **2025-04** 发布并开源；**2025-06** 捐赠给 Linux Foundation，成立独立的 **A2A Project**（治理中立化）。
- 现由 Linux Foundation 托管，Google 为原始贡献者。

### 版本
- **当前稳定版：v1.0.0**（2026-04 前后发布，一周年里程碑）。历史版本：0.1.0 → 0.2.6 → 0.3.0 → 1.0.0。
- v1.0 破坏性变更 + 迁移指南；**以 Protocol Buffers 定义为唯一权威规范来源**。
- v1.0 头号新特性：**Signed Agent Cards（签名版 Agent Card）**——给 Agent Card 加密码学签名，接收方可验证该卡确由域名所有者签发。另含：多协议绑定、企业级多租户、现代化安全流、Web 对齐架构。

### 核心对象（数据模型）
- **Agent Card**：JSON 元数据文档，Agent 的"数字名片"——身份、能力、skills、服务端点、鉴权要求。用于发现。默认发布在 well-known URI。
- **Task**：一次工作的基本单元，有唯一 ID，**有状态**、走生命周期。
- **Message**：一次通信轮次，有 role（user/agent），含一个或多个 Part。
- **Part**：最小内容单元，支持 text / file / structured data。
- **Artifact**：Task 的产出，由 Part 组成，可增量流式返回。

### 传输与方法
- **三种传输绑定**：JSON-RPC 2.0、gRPC、HTTP+JSON/REST（均走 HTTP(S)）。
- **主要方法**：SendMessage / SendStreamingMessage（流式走 SSE）、GetTask / ListTasks、CancelTask、SubscribeToTask（订阅实时更新）、推送通知配置 CRUD（长任务离线回调）、GetExtendedAgentCard（鉴权后取扩展卡）。

### 安全
- OAuth2 流、mTLS、授权范围限定（authorization scoping）、Signed Agent Cards。
- 设计强调"opaque agents"——协作方互不暴露内部状态/工具/推理，仅通过协议协作。

### 与 MCP 的关系（关键取舍）
- **互补，不竞争**。MCP = Agent↔工具/数据（纵向，让 Agent 用外部能力）；A2A = Agent↔Agent（横向，让多个 Agent 协作）。
- 2026 参考架构定型：**MCP 接工具 + A2A 做协同** 的两层模型。

### 采用度
- **150+ 组织**（发布时 50+）；核心仓库 **22,000+ GitHub stars**。
- 云平台：微软 **Azure AI Foundry / Copilot Studio**、AWS **Bedrock AgentCore Runtime**、Google Cloud 深度集成。
- **5 个官方 SDK**：Python、JavaScript、Java、Go、.NET。
- 生产落地：供应链、金融、保险、IT 运维等。

### 相关协议生态（Internet of Agents）
- **AP2（Agent Payments Protocol）**：A2A 之上的支付扩展，60+ 组织。
- 其它同赛道：**ACP**（Agent Communication Protocol）、**ANP**（Agent Network Protocol）。2026 有"协议趋同（convergence）"讨论。
