# 2026-07-09 · 联网核实笔记（AI-Infra-Platform 方案阶段）

方案（流程第 3 步）前的时效性事实核实。成品产出后，下列条目按章登记进 MANIFEST"时效性事实"表。

## Kubernetes + GPU
- DRA（Dynamic Resource Allocation）已在 K8s v1.34（2025-08）GA 并默认启用，取代 device plugin 的"只数卡"模式，支持按设备属性表达式申请；GKE/AKS/OpenShift 4.21 等托管发行版均已 GA 跟进。来源：kubernetes.io 官方博客、Red Hat、Google Cloud、AKS 工程博客。
- NVIDIA 在 KubeCon Europe 2026 宣布把 NVIDIA DRA Driver for GPUs 捐给 CNCF；DRA 成为 GPU 调度的社区主线。AKS 已演示 DRA + MIG、DRA + vGPU。来源：KubeCon EU 2026 报道、AKS 博客（2026-03）。

## 调度器格局
- KAI Scheduler：NVIDIA 2025-04 开源（Apache 2.0），与商业版 Run:ai 同一调度核；gang scheduling、拓扑感知、bin-packing、DRA 集成。Run:ai 商业版差异 = 管理 UI、多集群、SLA、fractional GPU 运行时。
- 分层共识：Kueue 管准入与配额（admission/quota），Volcano/KAI 管 gang 与公平调度，生产常两层叠用；小集群（<16 卡）Kueue 或 KAI 够用。Slurm 在 HPC 系与托管服务（HyperPod Slurm 编排）中仍是主力。来源：cloudoptimo/zenml/rack2cloud 2026 对比综述。

## GPU 切分与多租户
- 三板斧口径稳定：MIG（硬件隔离、最多 7 实例、可预测）、时间片（密度高、无显存/故障隔离，不宜生产多租户）、MPS；HAMi（CNCF）做任意 NVIDIA 卡上的细粒度配额（显存/算力百分比运行时强制）。MIG 正与 DRA 打通。来源：NVIDIA GPU Operator 文档、scaleops/collabnix/rafay 2026 教程。

## 云上托管训练/平台
- SageMaker HyperPod：EKS 或 Slurm 双编排；2026 上新——一键建集群（validated cluster creation）、Slurm 集群 continuous provisioning（2026-03）、G7e 实例支持（2026-04）；主打弹性容错（故障自动检测-替换-断点续训）。来源：AWS 官方文档/What's New。
- Google 侧对应 AI Hypercomputer + GKE（DRA GA、TPU/GPU 统一编排）；Ironwood TPU 2025-11 GA（细节记在 Compute 模块核实笔记）。
- 推理平台线：llm-d（K8s 系 P/D 分离）、NVIDIA Dynamo 1.0（2026-03 GTC GA）——已在 LLM-Inference llminf-disagg 核实过（2026-07-09），本模块只讲平台承载视角，事实同源引用。
