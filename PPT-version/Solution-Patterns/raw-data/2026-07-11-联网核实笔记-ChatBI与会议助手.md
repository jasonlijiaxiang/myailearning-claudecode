## 一、ChatBI / 对话式数据分析

### 准确率口径（本章最重要的"口径战场"）
- 裸 LLM Text-to-SQL 在**真实企业任务**上仅 10–21% 准确率（promethium 2026 企业对比口径）；学术基准（Spider 类）能到五六成——与 RAG 第 12 章"裸 SQL 五六成"口径互补：学术题 vs 真实企业库差距巨大。
- **语义模型加持后跃迁**：Snowflake 工程团队内部 150 题基准从约 51% → **90%+**（问题绑定手工语义模型后）；dbt 语义层基准报 98–100%。
- 结论口径：准确率差距不在模型在语义层——"乱 schema 上的整洁层"叙事有硬数据支撑。

### 产品格局（2026）
- **垂直一体派**（绑数据平台）：Snowflake Cortex Analyst（YAML 语义模型 + 2026 新品 Semantic View Autopilot——ML 自动从元数据/BI 使用模式发现指标定义，天级→分钟级）；Databricks AI/BI Genie（坐在 Unity Catalog 上，靠业务定义/表关系/血缘供语义上下文）；Microsoft Fabric Copilot。
- **独立语义/BI 派**：ThoughtSpot（Spotter）、dbt 语义层、Cube；治理型 SQL 工作流三强口径 = ThoughtSpot / Genie / Cortex Analyst。
- 国内：主流 BI（帆软/观远/网易有数等）均已挂 ChatBI 能力；豆包/各云亦有问数产品——按格局一员提及。

## 二、会议与办公助手

### 市场格局（2026）
- 两大类分野：**笔记器**（转写+摘要+行动项：Granola、Otter、Fireflies、Fathom、Zoom AI Companion、Notion AI）vs **agentic 会议平台**（会后自动执行动作）。
- **Granola**：2026-03 Series C $125M、估值 $1.5B——桌面端录音、无入会机器人（bot-free）代表；正以 Spaces/API 进企业。
- **原生集成派**：Zoom AI Companion（Zoom 内置零安装）、Teams Copilot（M365 上加 $30/人/月）、飞书妙记（国内代表）。
- 趋势三条：原生集成 > 独立工具；**bot 疲劳**驱动 bot-free 方案；会议功能并入更大平台。

### 场景架构要点（供参考架构页）
- 链路：音频采集（入会 bot / 桌面采集 / 平台原生）→ ASR 转写（说话人分离）→ LLM 摘要与行动项 → 分发与检索（会议记忆库）→ 与办公流集成（任务/日历/CRM）。
- 与库内积木映射：ASR/说话人分离 → Multimodal 第 4/8 章；会议记忆库检索 → RAG；行动项自动执行 → Agent（审批边界）；录音合规 → Security（知情同意）+ 数字人章合规思路同源。
- 售前口径坑：会议助手的"准确率"分三层——转写 WER、摘要忠实度、行动项召回，混着谈必翻车（评估口径接 Evaluation）。
