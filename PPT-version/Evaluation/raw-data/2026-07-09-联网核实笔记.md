## 1. 模型基准测试(Benchmark)格局

- **MMLU / MMLU-Pro 已功能性饱和**:前沿模型普遍 88%+,顶部分差无统计意义,不再能区分前沿模型。
- **GPQA Diamond** 仍能区分头部:截至 2026-02,Gemini 3.1 Pro 94.3%、Claude Opus 4.6 91.3%、GPT-5.3 Codex 81%、Qwen3.5-plus 88.4%;顶部已接近饱和。
- **Humanity's Last Exam(HLE)**:约 2,500 道专家出题,前沿模型约 35%–50% 区间,人类领域专家约 90%,是当前保留区分度最好的知识型基准之一。
- **Agent/编码类基准**成为主战场:SWE-bench Verified(端到端软件工程)、Terminal-Bench 2.0(89 个高难度终端任务)、OSWorld-Verified(GUI 自动化)、τ²-bench(多轮客服工具调用)、BFCL v4(2026-04 发布,改为整体 Agentic 加权:Agentic 40% / Multi-Turn 30% / Live 10% / Non-Live 10% / 幻觉度量 10%)。
- 截至 2026 年中的头部格局:Gemini 3.1 Pro 领跑 APEX/ARC-AGI-2/MCP-Atlas,Claude Opus 4.6 领跑 SWE-bench/Terminal-Bench/OSWorld,GPT-5.2 领跑 τ²-bench。
- 来源:techjacksolutions.com、kili-technology.com、lmmarketcap.com、benchmarkingagents.com、spheron.network(BFCL v4)、arxiv.org/html/2601.11868(Terminal-Bench 论文)

## 3. 评估工具链格局(2026)

- **开发期开源框架**:Ragas(RAG 评估事实标准:faithfulness / context precision / context recall,有学术方法论)、DeepEval(覆盖最广,50+ 指标,pytest 风格 CI 门禁)、promptfoo(CLI 红队测试与跨模型矩阵对比,40+ 对抗插件)。
- **平台型**:LangSmith(LangChain 生态绑定最深)、Braintrust(数据集管理→打分→生产监控→CI 门禁全生命周期)、Arize Phoenix / W&B Weave(生产侧观测与监控)。
- **主流做法**:成熟团队普遍"两套并行"——开发期评估框架 + 生产期监控平台,没有单一工具通吃。
- **OpenAI Evals 平台弃用**:2026-10-31 起只读,2026-11-30 关停(OpenAI 官方文档)。⚠️ 巡检重点。
- 来源:confident-ai.com、braintrust.dev、aiml.qa、developers.openai.com

## 4. LLM-as-a-Judge 现状与偏差

- **三大已证实偏差**:位置偏差(成对比较中交换顺序可致准确率漂移 >10%)、冗长偏差(把答案改写得更长不加新信息,部分判官 >90% 概率偏好长版)、自我偏好(对更接近自身生成分布、困惑度更低的输出打分更高)。
- **Judge Reliability Harness**(2026-03 开源):对判官做一致性与区分度压力测试,结论是没有一个判官全面可靠。
- **公认最佳实践**:LLM 判官辅助而非替代人工评估;上量前先用人工标注校准判官一致性(OpenAI/Anthropic 官方口径一致);LLM 更擅长判别(成对比较/分类/评分)而非开放式绝对打分,评估任务设计尽量往判别式靠。
- 来源:arxiv.org/pdf/2410.21819(自我偏好)、arxiv.org/pdf/2410.02736(Justice or Prejudice)、adaline.ai、emergentmind.com、en.wikipedia.org/wiki/LLM-as-a-Judge

## 5. 官方评估指南(书单素材)

- **Anthropic**:《Demystifying evals for AI agents》(engineering 博客)+ 平台文档《Define success criteria and build evaluations》:先定成功标准,再设计可自动判分的题型(选择题/字符串匹配/代码判分/LLM 判分);Agent 评估重点放在预上线与 CI/CD。
- **OpenAI**:《Evaluation best practices》:每次变更都跑评估、评估集持续生长、判官先用最强模型并对齐人工标签后再降本。
- **Hamel Husain & Shreya Shankar**:AI Evals 课程(Maven,已训练 2000+ 工程师/PM)+ 免费长文《LLM Evals: Everything You Need to Know》(hamel.dev evals-faq):错误分析(error analysis)驱动的评估方法论,业界实践派代表。
- **Hugging Face**:《The LLM Evaluation Guidebook》(OpenEvals,开源)。
- 来源:anthropic.com/engineering、docs.anthropic.com、developers.openai.com、hamel.dev、huggingface.co/spaces/OpenEvals

## 6. 三大云评估服务(上云落地素材)

- **AWS**:Bedrock 模型评估作业(Model Evaluation jobs,按作业计费),含自动评估、人工评估与 LLM-as-a-Judge、RAG 评估。
