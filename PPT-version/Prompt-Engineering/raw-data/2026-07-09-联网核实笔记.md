# 联网核实笔记 · Prompt Engineering

- **核实日期**：2026-07-09
- **用途**：本模块讲义/书单里"会过期的事实"的依据，MANIFEST「时效性事实」表以此为准。

## 1. 推理模型时代，提示词怎么变（→ 第 4 章）
- CoT（思维链）已从"提示技巧"变成模型内建的推理模式：GPT-5、Claude Opus 4.7 extended thinking、
  Gemini 3 Pro deep think、DeepSeek R1 都内建。对推理模型，重点从"教它思考"转为"何时该花推理 token、
  怎么评估推理轨迹"。
- OpenAI 明确区分：推理模型（内部自带 CoT，少写显式 CoT）vs 通用 GPT 模型（需要更明确的指令）。
- Anthropic 的优先级顺序：先清晰指令 → 多样本示例（multishot）→ CoT → XML 结构化。
- 趋势：从线性 CoT 走向多路径搜索（self-consistency、tree-of-thoughts、verifier），越来越多放在
  提示词之外的编排层实现。
- 来源：futureagi（CoT 2026 / LLM reasoning 2026）、stevekinney（跨 OpenAI/Anthropic/Gemini 提示）、
  Anthropic/OpenAI 官方文档综述。

## 2. 自动提示词优化 / DSPy（→ 第 5 章）
- DSPy 2.x；2026 默认优化器 MIPROv2（对 instructions + 示例做贝叶斯联合优化，结构化任务较手写
  提升约 10–40%）。
- GEPA（Genetic-Pareto 反射式优化器）：用自然语言反思执行轨迹来进化指令，较 MIPROv2 +13%、
  较 GRPO +20%，rollout 少 35×，被 ICLR 2026 oral 收录。
- 现有主要优化器：BootstrapFewShot、MIPROv2、COPRO、GEPA。
- 来源：futureagi（DSPy optimizers 2026）、morphllm（GEPA）、callsphere（MIPROv2 2026）。

## 3. 提示词安全 / OWASP（→ 第 6 章）
- OWASP Top 10 for LLM Applications **2025 版**（2024 年底发布）：LLM01 Prompt Injection 连续两版
  蝉联第一。
- 直接注入（用户直接写"忽略以上指令…"）vs 间接注入（把指令藏在文档/网页里，模型处理时中招）。
- 关键口径：RAG 和微调都**不能**根治注入；要靠纵深防御——最小权限工具、输入/输出过滤、
  高风险动作人审、定期对抗测试。
- 来源：mend.io、aembit、promptfoo、secra 的 OWASP LLM Top 10 2025 解读。

## 4. 提示词缓存（→ 第 5 章）
- 截至 2026-06，三大厂缓存输入均打 9 折（约 -90%）：OpenAI（GPT-5.x 家族）、Anthropic（全系）、
  Google（Gemini 2.5+）。
- 实现差异：
  - OpenAI：自动缓存，无需改代码，≥1024 token 触发、按 128 token 递增。
  - Anthropic：手动 `cache_control` 标记；有写入加价（5 分钟 TTL 1.25×、1 小时 2×）。
  - Google Gemini：显式缓存需手动设置 + 按小时存储计费。
- 一句话：同样 9 折，但"写入免费(OpenAI)/写入加价(Anthropic)/存储计费(Gemini)"三种成本模型。
- 来源：leanlm、prompthub、ofox.ai、artificialanalysis 的 prompt caching 2026 对比。

## 5. 各云提示词管理 / 优化服务（→ 第 5、7 章 上云落地）
- **AWS Bedrock Prompt Management**：提示词的创建、版本化、评估、运行；另有 Advanced Prompt
  Optimization——自动改写提示词提升准确性/精简度，并支持把提示词迁移到新模型、带内建评估反馈环。
- **Google Vertex AI / Gemini Enterprise**：提示词优化工具，可用评估数据集+指标自动精调并跨模型基准。
- **Azure AI Foundry**：prompt flow——提示词编排、评估流水线、变体测试、工作流基准。
- 2026 格局：三家都从"模型托管"长成全栈 AI 平台（含 agent 编排、RAG、微调、评估、治理）。
- 来源：aws.amazon.com/bedrock/prompt-management、AWS News Blog（advanced prompt optimization）、
  InfoWorld、各家 2026 企业 AI 平台对比。
