## 推理优化技术生产化现状

| 事实 | 来源 |
| --- | --- |
| 投机解码已从研究走到生产标配：EAGLE-3 为主流方案，vLLM 集成最高 ~2.5x 加速，接受率 >80% 时 2–4x；SGLang 已把投机解码设为默认 | spheron.network EAGLE-3 实测、sesamedisk 2026 综述、SGLang releases |
| FP8 是 2026 年生产推理默认精度（Hopper/Ada/Blackwell），校准后对 FP16 质量损失约 0.5–2%；vLLM/TensorRT-LLM/TGI 均一等公民支持 | vrlatech.com、sesamedisk.com 2026 量化综述 |
| NVFP4（FP8 micro-scale 16 值块 + FP32 全局 scale）面向 Blackwell，量化误差比 MXFP4 低 88%，但 2026 上半年校准工具链仍在成熟期、未大规模生产部署 | vrlatech.com FP4/FP8/FP16 对比 |
| AWQ/GPTQ INT4 用于显存受限场景，AWQ+vLLM 是 GPU 端常见生产组合；GGUF 面向 llama.cpp/Ollama 本地端 | spheron.network AWQ 指南、meta-intelligence 量化指南 |
| NVIDIA Dynamo 1.0 于 2026-03-16 GTC 上 GA：vLLM 之上的编排层，P/D 分离 + KV 感知路由，NIXL 负责 KV 在 NVLink/InfiniBand 上搬运；官方宣称 DeepSeek-R1/Blackwell 上最高 7x 吞吐 | developer.nvidia.com Dynamo 博客、docs.nvidia.com/dynamo、GTC26 DLIT82000 |
| P/D 分离（prefill 算力受限 / decode 带宽受限，分池部署）已成大规模服务共识；生产先例：月之暗面 Mooncake（FAST'25 最佳论文）、DistServe（OSDI'24）、开源 K8s 方案 llm-d | arXiv 2407.00079、NVIDIA Dynamo 设计文档、2026 综述 |
