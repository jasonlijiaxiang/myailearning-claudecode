# 2026-07-10 · 低代码 Agent 平台增章 · 联网核实笔记

> 元信息：agent-lowcode 增章前的时效性事实核实（流程第 2 步）。

## 四平台现状（2026-07）
- **Coze / 扣子**：字节 2025-07 开源 Coze Studio + Coze Loop（Apache 2.0，可商用可闭源二开）；GitHub 21.1k stars，最新 V0.5.1（2026-02-05）；Go 后端 + React/TS 前端，微服务 + DDD。SaaS 版扣子 3.0（2026-06）：多人多 Agent 协作、跨端协同、企业"行业专家"技能。定位：对话式 AI / 多渠道 bot 最快落地。
- **Dify**：148k stars，v1.15.0（2026-06-25）；许可证是"Dify Open Source License"（Apache 2.0 + 附加条件：多租户 SaaS/去 logo 需商业授权）。定位：LLM 应用 BaaS + LLMOps，RAG 一等公民（内置摄取/切分/嵌入/检索）。云版 $59 起有免费档。
- **n8n**：~127k stars（2026 初）；Sustainable Use License（fair-code，内部使用免费、不得免费转售）；2025-10 融资 $180M（Accel 领投、NVIDIA 参投），估值 ~$2.5B。定位：通用工作流自动化 + AI 节点（几百个 SaaS 集成、LangChain agent 节点）。
- **HiAgent（火山引擎）**：企业级智能体平台，覆盖开发/运行/消费/治理全流程，HiAgent 2.0 主打"数字员工治理"；100+ 插件、MCP 支持、100+ 行业模板；公有云/私有化/混合部署。IDC 口径：火山引擎居中国智能体开发平台私有化 17.8%、公有云 19.3%，双第一。

## 选型口径（2026 社区共识）
- 主负载是知识检索/LLM 问答 → Dify；主负载是自动化集成（SaaS 连接、数据搬运、触发动作）→ n8n；主负载是对话式 bot 多渠道分发 → Coze；要私有化 + 企业治理（权限/审计/配额）→ HiAgent 类企业平台。
- 与 code-first 边界：低代码平台 = 快速验证 + 业务人员可维护 + 内置 RAG/渠道；LangGraph/Agent SDK = 复杂状态机、深度定制、可测试性、版本化。生产常见组合：平台做壳与治理，复杂节点下沉代码。
- 数据安全视角：可自托管的 n8n / Dify / Coze Studio 数据可控性最好。
