# 2026-07-08 · LLM 原理模块 · 联网核实笔记

- 日期：2026-07-08
- 用途：Transformer 与 attention 讲义/书单中时效性事实的核实依据；MANIFEST「时效性事实」表以此为准。

## 核实结论

### 注意力变体在生产模型中的采用（2026-07 现状）

- **GQA 8:1 是生产默认**：生态支持最好，所有主流推理引擎原生支持；MLA 是 DeepSeek/Kimi 系基础设施上的默认。
  来源：pythonalchemist.com attention-variants playground。
- **MLA（多头潜在注意力）**：DeepSeek V2 提出，KV 缓存比同质量 GQA 小约 10 倍，但需定制推理 kernel；
  已被第二波架构采用——Kimi K2 沿用并放大、GLM-5 采用 MLA + DSA。
  来源：Sebastian Raschka llm-architecture-gallery/mla、arXiv 2512.12087。
- **DeepSeek 稀疏注意力（DSA）**：V3.2（2025-09）引入，lightning indexer 给全部历史 token 打分、
  主注意力只跑 top-k，复杂度 O(L²)→O(Lk)。来源：DeepSeek-V3.2 论文 arXiv 2512.02556。
- **DeepSeek-V4**：混合注意力（Compressed Sparse Attention 与 Heavily Compressed Attention 交错），
  面向百万 token 上下文同时压 KV 内存和每 token FLOPs。来源：arXiv 2512.12087 引述。

### 线性/混合注意力（2026-07 现状）

- Qwen3-Next 的 3:1 Gated DeltaNet + Gated Attention 混合被 **Qwen3.5 转正为旗舰架构**；
  Kimi Linear 同 3:1 比例但用逐通道门控。来源：Raschka "Beyond Standard LLMs"、mlabonne Qwen3.5 分析。
- **反例教训**：MiniMax M2（230B）放弃线性注意力退回全注意力——线性注意力在推理/多轮任务上精度不行，
  "linear attention is tricky in production"。来源：Raschka hybrid-attention gallery。
- 滑动窗口交错全局注意力：GPT-OSS、Gemma 3、MiMo-V2-Flash。Mamba 混合：Jamba、Nemotron 系。

### FlashAttention

- **FlashAttention-4**：论文 2026-03-05（arXiv 2603.05451），针对 Blackwell 非对称硬件伸缩设计，
  CuTe-DSL 实现，exp 用多项式近似绕开 SFU 瓶颈；PyPI `flash-attn-4` 包 **2026-07-01 发布**。
  来源：together.ai 博客、arXiv 2603.05451、pypi.org/project/flash-attn-4。

### 上下文窗口（2026-06 口径）

- 托管商用模型 **13 家提供 1M+ 窗口**：Claude Fable 5、Opus 4.8、GPT-5.5、Gemini 3.1 Pro（2M）、
  DeepSeek V4、MiniMax M3、Qwen3.5-Plus 等；Llama 4 Scout 标称 10M（开源自部署）。
  来源：morphllm.com/llm-token-limit、codingscape.com。
- GPT-5.4 标准 272K（API 可扩 1M，超 272K 有 2 倍加价）；GPT-5.5 默认 512K。
- Claude Sonnet 4.6 自 2026-03-13 起 1M 上下文按标准价（$3/$15 每百万输入/输出 token）。
- **标称 vs 有效**：RULER / MRCR v2 / NoLiMa 显示 200K 以上多事实检索得分掉 30–60 分——
  "标称 1M ≠ 有效 1M"，是讲长上下文与 RAG 取舍的关键弹药。来源：ofox.ai 长上下文基准汇总。
