## 文档解析格局（2026）
- 四强：LlamaParse（VLM 解析视觉复杂版面、agentic OCR，托管 API，<1000 页/天且非敏感文档最顺）；
  Docling（IBM Research 开源，AI 版面检测，DoclingDocument 保留语义层级，自托管零按页成本）；
  MinerU（OpenDataLab/上海AI Lab，CJK 复杂版面强，PaddleOCR+自研版面模型，出 Markdown+JSON）；
  Unstructured（老牌管线派）。云文档智能：Textract / Azure Document Intelligence / Google Document AI。
- 基准口径（opendataloader-bench，200 PDF）：hybrid AI 0.909 > pdfmux 0.905 > Docling 0.877 > marker 0.861 > MinerU 0.831；LlamaParse/Unstructured 未公布该基准分。
- 共识：表格与阅读顺序保真是分水岭；输出 Markdown/JSON 面向 RAG 已是标配；自托管 vs 按页计费是选型主轴。
- 来源：mixpeek / pdfmux / llamaindex 对比页 / themenonlab 2026。

## 向量库格局（2026）
- 口径：pgvector 生产可用上限约 50–100M 向量（HNSW 重建时间成瓶颈）；Qdrant（Rust）p50 4ms/p99 25ms、过滤检索最强的开源；Milvus/Zilliz 主打十亿级分布式；Weaviate/Elasticsearch 原生混合检索最成熟（ES = kNN+全文一查双路）；托管派 Pinecone/Zilliz Cloud 按量计费。
- 选型共识：已有 Postgres → pgvector 起步；精简快速 RAG → Qdrant；十亿级 → Milvus；要原生混合 → Weaviate/ES；不想运维 → 托管。
- 来源：firecrawl / encore / layerbase / kunalganglani 2026 对比（"100+ 企业部署决策指南"）。
