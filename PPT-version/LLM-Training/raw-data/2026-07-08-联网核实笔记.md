# 联网核实笔记（流程第 2 步产出）

- **日期**：2026-07-08
- **用途**：LLM-Training 模块讲义与书单的事实依据，所有时效性表述以此为准（写 PPT 时不凭记忆）。

## 后训练范式（2026-07 现状）

- **主流后训练栈已从"经典 RLHF（PPO）"演进为三段式：SFT →（可选 DPO）→ RLVR**（可验证奖励强化学习），
  GRPO / DAPO 一族取代 PPO 成为推理模型训练的主力算法。
  来源：https://llm-stats.com/blog/research/post-training-techniques-2026 ；
  RL for LLM Post-Training 综述 https://arxiv.org/abs/2407.16216
- **DeepSeek-R1 证明纯结果导向 RL（GRPO）可直接从 base 模型激发出链式思考**，无需监督推理数据；
  该范式支撑了 R1、o3 等推理模型。
- **RLVR 一词出自 AI2 的 Tülu 3**（arXiv 2411.15124），Tülu 3 是目前最完整的全开源后训练配方
  （SFT → DPO → RLVR，数据/代码/评估全开放）。来源：https://allenai.org/blog/tulu-3

## 预训练与 scaling laws（2026-07 现状）

- **数据墙是行业公认现实**：Chinchilla 最优（约 20 token/参数）训 1T 参数模型需约 20T token，
  而互联网高质量文本存量估计仅 10–50T token → 合成数据、多轮 epoch、数据配比成为竞争焦点。
- **Scaling laws 已从"训练最优"转向"推理感知"**：Beyond Chinchilla-Optimal（Sardana et al.）把推理成本
  计入最优解；分析机构预计 2030 年推理算力占 AI 总算力约 75%，行业重心转向推理时扩展
  （inference-time compute scaling）。
- **2026 年旗舰开源模型几乎全是稀疏 MoE**：DeepSeek V4-Pro（1.6T 总参 / 49B 激活）、
  Llama 4 Maverick（400B/17B）、Qwen 3.5（397B/17B）、Kimi K2.6（约 1T，256K 上下文）。
  许可格局：Apache 2.0 成主流，DeepSeek 用 MIT。
  来源：https://openrouter.ai/blog/insights/the-open-weight-models-that-matter-june-2026/
- **FP8 混合精度训练已进入主流实践**（如 MiMo-V2.5-Pro 以 FP8 训了 27T token）。

## 训练基础设施（2026-07 现状）

- **Muon 是十年来 AdamW 的首个可信挑战者**：计算效率约 2×；Kimi K2 用 MuonClip（qk-clip 解 attention
  logit 爆炸）在 15.5T token 上零 loss spike 完成预训练（技术报告 arXiv 2507.20534）；
  GLM-4.5/GLM-5、INTELLECT-3 亦采用；**PyTorch 2.9 已原生内置 torch.optim.Muon**，DeepSpeed/NeMo 跟进。
  来源：https://pytorch.org/blog/using-muon-optimizer-with-deepspeed/
- 分布式并行的权威免费教材是 HF **Ultra-Scale Playbook**（约百页，4000+ 组实验、至 512 GPU，
  5D 并行 / ZeRO / CUDA kernel）。

## 稳定事实（不用巡检，写这里备查）

- InstructGPT（arXiv 2203.02155）是 SFT+RLHF 三段式的开山之作（2022）。
- Chinchilla（2022）：同算力下参数与数据应同比扩，约 20 token/参数。
- DPO（2023）：把偏好对齐从"训练奖励模型+PPO"简化为直接在偏好对上做分类式训练。
