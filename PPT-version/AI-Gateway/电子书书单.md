## 阅读建议（售前取用顺序）

1. **先读 #1 LiteLLM 文档**——半小时就能在本地起一个真网关，把"统一接入/虚拟密钥/预算/fallback"从抽象名词变成手上能跑的东西。客户问得最多的也是这些。
2. **要做云上 demo / 讲上云落地，读 #2 Azure APIM**——它的实验室 repo 是把"token 限流 + 语义缓存 + 多模型统一"讲给客户看的现成脚本；再顺手翻 #6 建立三云对照感。
3. **被客户追问"路由/缓存能省多少钱、会不会答错"时，看 #7 RouteLLM + #8 FrugalGPT**——有数据、有方法，能把"省钱"说得让人信。
4. **走云原生/K8s 客户或国内客户，读 #3 Envoy AI Gateway + #4 Higress**——一个是 CNCF 标准派，一个是国内生态与中文资料最全、且 MCP 能力最激进的。
5. **做选型对比表、准备"你们和 X 有什么区别"，翻 #9 横评 + #10 OTel 标准**——前者给格局，后者让你在可观测这一栏说得比对手细。
