# AI 网关 · 联网核实笔记

- **核实日期**：2026-07-09
- **用途**：本模块讲义/书单里"会过期的事实"以此为准；MANIFEST 的"时效性事实"逐条引这里。

## 1. 概念口径（不易过期，作背景）
- AI 网关 = 应用与各家大模型/Agent/工具之间的**统一入口 + 运营管控层**。相对传统 API 网关，多了 LLM 专属能力：按 **token** 计量的限流与预算、多供应商**统一 OpenAI 兼容协议**、**语义缓存**、**护栏/内容安全**、按 token 的成本归集，以及正在收敛进来的 **MCP 托管**。
- 与相邻概念的边界：不是 MCP/A2A（那是互操作协议）；不是单模型推理服务（LLM-Inference，那是"一个模型怎么快省地跑"）；AI 网关在其之上做"多模型/多供应商统一治理"。

## 2. 开源 AI 网关格局（2026-07 核实）
| 产品 | 定位/技术底 | 关键能力 | 时效点 |
| --- | --- | --- | --- |
| **LiteLLM**（BerriAI） | Python，最广泛采用的开源 LLM 代理/网关 | 100+ 供应商统一为 OpenAI 格式；虚拟密钥（virtual keys）、按项目/用户预算与花费追踪、护栏（PII 脱敏/内容安全）、复杂度自动路由（7 维打分选模型档位）、实时护栏（含 OpenAI Realtime 语音） | 版本迭代极快（v1.81.x 系列，2026）；Python 是其性能天花板 |
| **Portkey** | 开源网关 + 可选托管 SaaS | 护栏、语义缓存、提示词管理 | **完整开源发布于 2026-03** |
| **Kong AI Gateway** | 在成熟 Kong API 网关上加 LLM 插件 | 已用 Kong mesh 的团队天然加装 | 老牌 API 网关延伸 |
| **Envoy AI Gateway** | Tetrate + Bloomberg，基于 **CNCF Envoy Gateway**；服务网格血统 | token 限流（按 provider/model/client）、AI 原生可观测（OpenInference 分布式追踪 + OpenTelemetry GenAI 语义约定）、稳定 v1 API/CRD | **v1.0 发布 2026-06-23**；语义缓存、按美元的成本治理、MCP 授权/OIDC token 交换在路线图上；早期 provider 支持仍偏少 |
| **OpenRouter** | 托管模型路由 SaaS（非自托管） | "不想自托管"时的统一入口替代 | 商业托管 |

## 3. 云厂商托管 AI 网关（2026-07 核实）
- **Azure API Management — AI/GenAI Gateway**：`token limit` 策略（可按订阅键/IP 计 token 限流，网关侧预估 prompt token）；**语义缓存**（Azure Managed Redis + RediSearch，`llm-semantic-cache-store`/`llm-semantic-cache-lookup` 策略，走 Embeddings API 按向量相近度命中）；**Unified Model API**（单一 OpenAI 兼容端点代理多后端，含 Amazon Bedrock，自动格式转换 + 统一治理策略）；Build 2026 新增 **MCP 内容安全**。被评为"唯一原生内置按 token 限流 + LLM 专属策略的云厂商网关"。
- **AWS**：以 **Bedrock** 作为多模型接入 + Guardrails 为主，**无** APIM 式"多供应商 Unified Model API"等价物。
- **Google Apigee**：已加入部分 AI 网关能力，协议广度不及 APIM。
- **Cloudflare AI Gateway**：聚焦花费上限（spend limits）与缓存。

## 4. 国内/云原生生态（2026-07 核实）
- **Higress**（阿里开源，AI-Native API 网关）：基于 **Istio + Envoy**，Wasm 插件（Go/Rust/JS）。多模型统一路由 + 协议转换（100+ 模型）、语义缓存、token 级限流、内容安全（**提示注入检测**、敏感内容识别）、多维鉴权与可观测、**模型级 Fallback**；可托管 **MCP Server**，`openapi-to-mcp` 把 OpenAPI/REST 一键转 MCP，支持自建 MCP Marketplace。对 LLM API + MCP Server + Agent 统一代理。
- **Apache APISIX AI Gateway**：云原生 API 网关 + AI 网关，支持 MCP 协议、管理 AI Agent 流量。

## 5. 关键趋势
- **AI 网关正在收敛为"AI 网关 + MCP 网关"二合一**：统一代理 LLM API、MCP Server、Agent（Higress、Envoy AI GW 路线图、APISIX、Azure APIM 均在往这走）。
- 可观测正在标准化到 **OpenTelemetry GenAI 语义约定 / OpenInference**。
