# 2026-07-10 · 语音与实时交互增章 · 联网核实笔记

> 元信息：mm-voice-realtime 增章前的时效性事实核实（流程第 2 步）。

## 延迟预算（2026 生产口径）
- 人类对话容忍空隙 300–500ms 开始察觉；>1.5s 稳定地"感觉坏了"。
- 2026 生产目标：级联管线 P50 <1.5s、P95 <5s；<800ms 端到端是标杆线（gpt-realtime-1.5 ~820ms 量级）。
- 级联管线时延拆帐：STT 100–300ms + LLM 350–1000ms + TTS 90–200ms + 跨厂商网络往返 50–200ms；LLM 在 STT 与 TTS 之间约有 200–700ms 预算。
- 来源：softcery 2026 架构对比、telnyx 延迟横评、rtcleague。

## 级联 vs 端到端（S2S）
- S2S 相比非流式级联延迟可降约 85%，但 2026 企业部署仍以级联为主：可调试、合规可审计、供应商可换（5+ STT、7+ TTS、几十个 LLM vs S2S 仅少数家）。
- 成本：级联 $0.0095–0.17/分钟、可预测；S2S 跨度极大（Gemini Flash Live 低价档到 OpenAI Realtime $0.30/分钟量级，差百倍）。
- gpt-realtime 定价：$4/1M 音频输入 token、$16/1M 输出（缓存音频输入折扣 ~98.75%）；实测 $0.18–0.46/分钟未缓存、开缓存 $0.05–0.10/分钟；gpt-realtime-mini 约便宜 60%。
- 来源：speko.ai S2S vs cascaded、softcery、hackernoon 4000 会话实测、OpenAI 定价页。

## 端到端语音模型格局（2026-07）
- OpenAI gpt-realtime 家族（Realtime API GA，gpt-realtime-1.5/2，128K 上下文，agentic 工具调用入音频环路）。
- Google Gemini Live API：Gemini 3.1 Flash Live（2026-03-26 发布，audio-to-audio，~200ms 首响，200+ 语言，工具调用基准领先）。
- 开源权重主力：Qwen3-Omni / Qwen3.5-Omni（2026-03-30，原生端到端全模态，113 语种 ASR、WER 6.24，流式语音输出，可自部署）；Kyutai Moshi 为早期全双工代表。
- 来源：flowtivity Gemini vs GPT realtime 对比、QwenLM GitHub、marktechpost。

## 语音 Agent 工程框架
- Pipecat（Daily，2026-04 v1.0）：帧处理器管线抽象，自己拼 STT/LLM/TTS 每一段，传输层无关，可控性最强、代码更啰嗦。
- LiveKit Agents：WebRTC 媒体服务器 + Agents SDK，room 模型（agent 作为参与者进房间），多人/音视频/共享屏原生支持；接口更省心。
- 实测端到端延迟同量级：LiveKit 750–900ms、Pipecat（Daily 上）800–950ms。
- 托管一层：Vapi/Retell 等"语音 agent 即服务"。
- 来源：webrtc.ventures 2026-03 选型指南、inworld/cekura/f22labs 对比。
