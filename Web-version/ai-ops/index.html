01为什么 LLM 应用的 Ops 是新学科#ops-what-why

传统运维问「系统挂没挂」，AI 运维还要问两个传统 APM 一个都答不了的问题：答得对不对（质量轴）、烧钱烧得值不值（成本轴）。200 OK 里可能装着一本正经的胡说；一次深思考 = 一百次普通调用。核心痛点叫静默退化：基础设施指标全绿、输出质量在悄悄下滑——AI 运维的第一敌人。

边界先立清楚：网关是数据源（token 计量、成本归因在 AI-Gateway 网页版）、Evaluation 给方法（评估集怎么建、判官怎么用在 Evaluation 网页版）、 GPU 利用率归 AI-Infra-Platform——本页只讲三片之间缺的「生命周期线」。观测自身也要做成本工程：一条 RAG 请求 = 检索 + 重排 + LLM + 后处理四个 span 还带完整 prompt 文本，遥测量是等价传统 API 的 10–50 倍，接入 AI 负载后 APM 账单普遍 +40–200%——采样率、保留期、脱敏是方案里必设的三个开关，也是最常被漏掉的报价项。

观测成本与采样规划器

用你的调用量算遥测数据量与评估采样规模——「我们连观测的账都替您算了」的交互版。系数全部来自本页正文（四 span/请求、采样 5–10%），不新增事实。

02Tracing 与 OTel GenAI#ops-tracing

trace 是 AI 应用的「行车记录仪」——出事故时要能回放每一步谁说了什么、调了什么工具、花了多少 token。OTel GenAI 四类 span：gen_ai.client（model）已稳定（2026 年初出实验期：模型名/token 用量/延迟/参数）、tool / agent / workflow 仍实验态但实践已稳——对客的精确口径值钱；多数属性仍带 Development 标签，封装一层再用。

一条 RAG 请求的旅程：workflow 根 → tool 检索 → tool 重排 → model 生成（token 大头） → tool 后处理。排障三板斧：慢看 model span 首 token 时延、贵看 input token （通常是上下文塞多了）、错先看检索工具的返回（幻觉常源于喂了脏数据）——没有 trace 这三类问题平均要复现半天，有 trace 是分钟级定位。与网关采集是两层拼接：网关看进出模型的流量、应用内 span 看一次请求的内部结构——trace id 要贯通。PII 三开关：记什么（结构全量：span 树/时延/token 数/版本 + 质量信号）、不记什么（prompt 里的 PII 正则 + NER 脱敏后再入库——脱敏要在采集点做不在展示层，一个手机号会出现在 span 属性、日志、评估队列三个地方）、记多久（内容类短保留 7–30 天、指标类长保留、事故 trace 单独归档）。观测系统常是「数据泄漏的第二现场」——prompt 明文原样进日志等于把客户数据复制了一份，按「第二数据库」的安全标准建。

span 标准状态核实 2026-07-10（OTLP 口径 07-12），来源 OTel 官方规范与博客。

03在线评估与反馈回流#ops-online-eval

离线评估像「高考」（固定评估集、发布前跑，回答「能不能上线」），在线评估像「体检」（真实流量采样、持续跑，回答「线上还好吗」）——共享同一套判分器定义，定义不一致两边分数就没法互相解释。「生产团队把模型当移动靶」是 2026 共识：离线过了不等于线上一直好。

四级评估漏斗，越贵的手段处理越少的样本：inline 检查（格式/护栏，唯一同步层，毫秒级全量）→ judge 打分（采样 5–10% 真实流量、异步执行——先回复用户后打分，零应用时延；判官用便宜档 + 定期人工校准）→ 人工评审（判官拿不准的 + 高价值样本进队列，评审动作标准化打失败模式标签，非工程角色也能参与）→ 离线学习（失败进数据集）。为什么只采 5–10% 不全量：质量问题是分布性的，采样足以抓趋势和回归；全量打分的钱不如花在判官质量和人工评审上，急性问题另有 inline 层全量兜底。

闭环三件套让打分变成行动：告警（评估分骤降联动发布记录——先问「刚才谁改了什么」；阈值来自基线不拍脑袋）、评审队列、数据集晋升（打好标签的失败样本进回归集——生产事故变成永久测试覆盖，评估集从一次性资产变「滚雪球资产」）。这是对客最有说服力的资产故事：您的每次故障都会让系统更难再犯同样的错——运维包「越用越值」的证据。

04漂移与静默退化监测#ops-drift

三类漂移：供应商静默换 checkpoint（最阴险——合同里写着同一个模型名，背后的权重已不是上月那份；「上月测的 ≠ 这月答的」要讲给每个客户听）、输出漂移（输入没变，风格/长度/结构在变）、性能漂移（语法还对，事实错得更多）。

全绿仪表盘为什么骗人：可用性 99.9%、延迟正常、错误率 0.1%——因为模型「答了」，只是答错了；质量仪表盘看到的是 judge 分三周从 4.2 降到 3.6、「无法回答」占比翻倍、退款类失败模式聚集。退化以「周」为单位缓慢发生，客诉爆发时已煮熟——破法只有让「质量」成为和「可用性」并列的一等监控对象。检测组合拳（单一手段都有盲区）：judge 持续打分（性能漂移主探测器，日看板）+ 嵌入距离监测（分布偏移，周报告）+ 护栏违规率/格式失败率（廉价信号，进告警）+ 回归集定期重跑（唯一能「证明」供应商换了模型的手段，别省——周跑 + 模型版本变更时立即跑）+ 成本/token 突变（想得更久 = 变了）。巡检节奏：日看板、周报告三个数字（judge 趋势/嵌入距离/回归通过率——玄学变仪表盘）、发现漂移先查供应商公告再查自己的发布记录。巡检为什么持续收费：威胁是持续的，基线断一个月就没了——像体检，价值在连续性。

05发布管理：提示词也是生产工件#ops-release

核心观念：提示词/RAG 配置/工具定义都是生产工件——改动就是发布，发布就要走流程。一个词就能改变线上行为（删掉一句约束导致客服乱承诺是真实事故）。版本注册表六件入册（提示词/评估集版本/模型与参数/RAG 配置/工具与护栏/发布记录），回答一个朴素问题：「上周三线上跑的到底是哪套配置？」——答不出的团队，排障只能靠考古。

五步发布流水线：版本注册打 tag → 评估门禁（回归集不过线连金丝雀资格都没有；高风险变更加专家评审）→ 金丝雀 5–10% 流量 → 观测窗（质量分/错误率/延迟三件套，小时级到天级） → 晋升或回滚。与传统金丝雀的分水岭：LLM 金丝雀必须在金丝雀流量上跑自动评估——坏的是答案质量，HTTP 全是 200；没有在线评估的金丝雀是裸奔。回滚纪律：旧版本不可变永远保留、回滚 = 生产 tag 指回上一已知好版本秒级生效（不是重新部署——达不到秒级通常因为提示词硬编码在代码里）、回滚后必须补事故记录。反模式三个：控制台直接改提示词、回滚靠记忆、金丝雀只看延迟不跑评估。四层版本键定位「一次输出由谁决定」：应用层（谁在运行）/ 生成层（怎么生成的——供应商静默换模就是这层在漂）/ 上下文层（依据什么）/ 质量层（怎么判定的）——拼成组合发布号，从任何一条 trace 一跳定位每层的不可变版本；「什么都没改质量却掉了」，十有八九是没版本化的那层在动。模型升级当成一次大发布走全流程——换模型是流程不是冒险。

06事故响应#ops-incident

AI runbook 四问——传统 runbook 没有的四章：答错了（幻觉率告警 → 降级保守模式/升级人工）、被攻破了（注入检测 + 工具权限急冻，弹药在 Security 网页版）、卡死了（循环检测 → 步数/预算上限强制中断）、复盘不了（必须靠 trace 留证——第 2 节是本节的前提）。

成本尖峰常是第一信号：死循环/被注入的 agent 先表现为 token 暴涨——有写权限的 agent 等人发现时可能已把事情搞大；真实案例：agent 递归循环十分钟烧掉一天预算。所以成本告警要分钟级粒度，不是日账单。「你们的 AI 有没有刹车」的工程回答两层：急停开关（IM 机器人一键暂停所有任务、停任务不砸状态保留现场；恢复要走发布门禁，不会「手一抖又跑起来」）+ 硬上限（步数/预算/权限边界超限自动中断）。HITL 三分级按风险配人：in-the-loop（人开车 AI 副驾——高危写操作：退款/删除/对外发送）、 on-the-loop（人监督 AI 有界行动——客服回复类）、out-of-the-loop（人审计——低风险高频批量任务，事后抽检 + trace 留证）。放权路线图本身是方案书的一节：新场景从 in 起步，评估分稳定后逐级放——放多快由数据说话，不由胆子说话。事故分级写进 SLA（P1 高危输出触达客户 15 分钟响应急停、P2 质量骤降/成本尖峰 1 小时降级、P3 漂移趋势 1 个工作日、 P4 进 backlog）——P1 的定义要和客户共创，什么算「高危错误输出」由业务定。

08运营包速查#ops-cheatsheet

运营包不是「售后服务」，是有清单、有 SLA、有产出物的订阅产品——这张表是 Solution-Patterns「人力账」的答案。

服务项内容报价口径
在线评估运营采样打分、判官校准、评审队列、月度质量报告按月，含人力
漂移巡检日看板、周三数字报告、回归集重跑按月订阅
发布门禁服务版本注册表托管、门禁执行、金丝雀陪跑按次或包月
事故值守on-call 响应、急停授权、事后复盘报告按 SLA 分级
数据集资产运营失败样本标注入集、评估集版本管理按月，资产归客户

指标节奏：实时告警（inline 违规率/成本分钟级突变/错误率——异常先查发布记录）、日看板（judge 质量分/采样覆盖率/延迟 P95）、周报告（漂移三数字）、月度报告是运营包的「可见产出物」——客户续不续费，往往取决于这份报告讲不讲得出价值故事。观测体系自身的验收六个「可」：可见可复现（trace 贯穿 + 四层版本键血缘回放）、可评测（四路打分 + 校准集）、可治理（PII/租户/采样保留有策略）、可发布可恢复（离线→影子→金丝雀→门禁 + kill switch）——终极判断：任意一次质量或成本异常都能回放「哪个版本、哪里变化、为何放行、如何止损、怎样防复发」；看板能打开只是联调，不是运营闭环。

09现场高频追问#qa

只收会改变某个决定的问题——会改变技术方案、风险判断、验收方法、对客沟通或下一步行动。

每题标出它加入本题库的日期：2026-07-10 来自讲义首版对练页，2026-07-12 来自标准与许可证口径补核。

我们已经有 Datadog 了，为什么还要一套 AI 观测？2026-07-10

Datadog 管「挂没挂、慢不慢」很好，但看不到「答得对不对」——200 OK 里可能装着胡说。质量轴需要评估型观测，两者互补不替换，trace id 打通。

预算敏感可用开源自托管（Langfuse/Phoenix）承接专用层——顺带规避「AI 遥测全量进按量 APM」的账单事故。

下一步　用分工表对客：APM 收聚合指标、专用系管质量与评审——「够看贵不贵，不够看好不好」。

系统没报错，但用户说 AI 变笨了，怎么排查？2026-07-10

这就是静默退化——全绿但变笨。排查靠质量遥测：拉评估分趋势、对比模型版本与提示词变更记录、看漂移监测——没有这套东西，这类问题只能靠猜。

退化以「周」为单位缓慢发生，客诉爆发时已煮熟——质量必须成为和可用性并列的一等监控对象。

下一步　周报三个数字把玄学变仪表盘：judge 趋势、嵌入距离、回归集通过率。

依据　静默退化

大厂的模型还能悄悄变吗？2026-07-10

能，而且是常态：API 背后的 checkpoint 会被静默更新，上月验收的模型不等于这月答题的模型。破法是您自己的回归集每周重跑——版本变没变，分数会说话。

回归集重跑是唯一能「证明」供应商换了模型的手段——这也是漂移巡检要持续收费的原因：基线断一个月就没了。

下一步　发现漂移的动作链：先查供应商公告、再查自己的发布记录；供应商侧用提示词补偿或锁版本，自己侧回滚。

依据　三类漂移

线上打分会不会拖慢用户请求？LLM 判官可信吗？2026-07-10

不拖慢：唯一同步的是毫秒级 inline 检查，质量打分全部异步——先回复用户后采样打分。判官有已知偏差，三重保险：定期人工校准、拿不准的进人工队列、关键决策不单靠判官分。

漏斗的灵魂是「越贵的手段处理越少的样本」——inline 全量、judge 采 5–10%、人工只看队列。

下一步　把数据集晋升讲成资产故事：每个生产失败被标注入集变成永久回归覆盖——越用越值。

依据　评估漏斗

改个提示词而已，为什么要走发布流程？2026-07-10

因为一个词就能改变线上行为——删掉一句约束导致客服乱承诺是真实事故。流程不重：注册打 tag、门禁自动跑、金丝雀自动切，多花分钟级时间，换回滚和留痕。

LLM 金丝雀与传统的分水岭：必须在金丝雀流量上跑自动评估——坏的是答案质量，HTTP 全是 200。

下一步　演练一次 30 秒回滚：达不到秒级通常因为提示词硬编码在代码里——这就是要上注册表的原因。

依据　发布流水线

你们的 AI 有没有「刹车」？2026-07-10

有，两层：急停开关（事故频道一键暂停所有任务、保留现场）+ 硬上限（步数/预算/ 权限边界超限自动中断）。恢复要走发布门禁，不会「手一抖又跑起来」。

成本尖峰常是第一信号：死循环/被注入先表现为 token 暴涨——成本告警要分钟级粒度；日账单粒度下死循环会烧一整天。

下一步　高危操作按 HITL 三分级定人机边界，放权路线图进方案书——放多快由评估数据说话。

依据　急停与分级

trace 里有客户的对话内容，安全吗？2026-07-10

内容与结构分离处理：结构全量、内容采样并在采集点脱敏（正则+NER）、保留期分级、事故 trace 单独归档。观测系统按「第二数据库」的安全标准建。

观测系统常是数据泄漏的第二现场——一个手机号会出现在 span 属性、日志、评估队列三个地方；脱敏在采集点做，不在展示层。

下一步　埋点走 OTel 标准（client span 已稳定）——换观测后端不改埋点，不锁定。

依据　PII 三开关

上线之后你们还管吗？管什么、怎么收费？2026-07-12

管，这正是运营包：在线评估运营、漂移巡检、发布门禁、事故值守、数据集资产运营——五件套各有清单、SLA 与产出物，按月订阅或按 SLA 分级。

运营包是「人力账」的展开：不是售后服务是订阅产品——月度质量报告是续费价值的可见证明。

下一步　数据集资产归客户写进合同——「越用越值」的评估集是合作的复利。
