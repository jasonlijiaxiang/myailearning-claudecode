01是什么 / 为什么#mm-what-why

大模型本质是「文字天才」，但真实世界的信息是图、声、视频、表格混在一起的。多模态不是某一个模型，而是一类能力，且分两条完全不同的线：理解侧（输入多模态 → 输出文字：发票识别、图表问答、视频摘要）与生成侧（输入文字/图 → 输出图/音/ 视频）——选型、报价、落地方式完全不同。客户说「要个多模态」，往往只需要能力谱系里的一格，先定位到那一格，方案立刻清晰。

该不该上的一句话判据：非文字信息是不是「关键输入」+ 任务是不是需要「理解」而非「转录」——两个都 yes 才上。与 OCR 的区别：OCR 只把图变字，多模态能「连看带理解」——一张财报截图，OCR 只能吐数字，多模态能回答「同比增长最快的是哪个业务线」。两者不是替代是分工：稳定字段 OCR 省钱，复杂理解多模态兜底。为什么现在爆发：原生旗舰落地（看听做进同一个模型）、统一架构成熟（LLaVA 式投影层范式跑通，开源可低成本复刻）、成本降到「用得起」。

02机器怎么「看」#mm-encoder

模型只吃 token，一张图是几百万个连续像素——核心问题就一句：怎么把连续像素变成离散 token。ViT 的答案：把图切成 16×16 的 patch 当「视觉单词」（224×224 的图 → 196 块），展平、投影、加位置编码、进 Transformer。像把油画剪成一版小邮票，按顺序念给对方听。

CLIP 解决「图和文字怎么对上」：用 4 亿对「图 + 配文」做对比学习——匹配的拉近、不匹配的推远，图和文字进入同一个语义空间。得到三样东西：

图文可比可搜

零样本分类

给类别名就能认新类。

多模态大模型的「眼睛」底座

开源 VLM 多用 CLIP / SigLIP 系编码器。

成本之根在这一节

patch 越小 / 分辨率越高 → 视觉 token 越多 → 看得越细但越贵越慢；动态切分（大图切子图分别编码）会让 token 成倍涨。客户问「传高清图会不会更准」：会，但费用也涨——给他「够用的最低分辨率」而不是「最高分辨率」。

CLIP 不是多模态大模型：它是眼睛，大模型是在它之上接了会推理的大脑。

03模态怎么「拼」#mm-fusion

眼睛有了、大脑有了，怎么接起来？业界三个答案，本质区别只有一句：让 LLM「读」视觉 token 的方式不同。

路线代表怎么接优势代价
投影层LLaVA视觉向量「翻译」成词向量塞进序列（同声传译）简单、生态繁荣、易复刻——开源主流token 多、序列长
交叉注意力FlamingoLLM 内部插层，生成时「侧头看图」省序列、图文交错强改动大、工程重——多见于大厂旗舰
Q-FormerBLIP-2查询向量从上千 token 萃取几十个精华省算力、可冻结主体可能丢细节（小字/密集表格）

「早融合 vs 晚融合」落到工程就是本册最值钱的决定：原生多模态 vs 拼管线。原生（一个模型端到端「看图→理解→回答」）：强在图表推理、复杂版式、开放问答；代价是贵、慢、可能幻觉、能力黑盒。拼管线（OCR/ASR 转文字 → 便宜的文本 LLM）：强在稳定字段、可控、便宜、每步可替换可审计；代价是跨模态弱、误差累积。三个问题落子：版式稳定吗？要转录还是理解？成本/可控性多敏感？多数企业文档场景的最优解是混合：拼管线打底省钱 + 原生兜底疑难——别把它当二选一。

04理解侧能力盘点#mm-understanding

四大场景：图像（「认出/描述」人人都行，「图表推理」「空间关系」才拉开差距）、文档（售前最高频落地点）、视频（多了时间维度，抽帧策略决定成本）、语音（只要字幕纪要用专用 ASR 就够，要实时助手才上原生语音模型）。文档打法：稳定字段交专用 OCR，疑难理解交多模态——成本最优。

05生成侧能力盘点#mm-generation

图像生成两大流派像两种学生：扩散＝美术生（从噪声逐步去噪；Midjourney/FLUX.2 ——光影质感写实强，空间关系和文字渲染弱）、自回归＝逻辑生（像写句子一样逐 token 拼图；GPT Image 2 / Nano Banana 2——听得懂复杂指令、字写得准）。要艺术图偏扩散，要海报/UI/信息图这类「按指令带准确文字」的偏自回归。

06选型与成本账#mm-selection

决策铁三角：成本 / 延迟 / 精度选两个——对客开场先问「哪个最不能妥协」。精度优先（最强旗舰+高分辨率，贵而慢，医疗金融）、成本优先（拼管线打底+开源自部署+ 够用的分辨率，海量场景）、延迟优先（小模型+缓存+预处理，实时交互）。

成本账

图像按视觉 token 计费，分辨率是主变量——低分辨率一张几十到百来 token，高分辨率 / 大图切块可上千；多页 PDF、视频抽帧成倍叠加。

省钱一：降分辨率

降到够用就行。

省钱二：分流

稳定部分走 OCR，不走大模型。

省钱三：缓存去重

结果缓存 + 去重。

回答「传图贵不贵」的正确姿势：按客户的分辨率和图量折算到「一天多少钱」，不是报单价。

为什么比纯文本慢：视觉 token 多 → prefill 变重 → 首 token 延迟明显更高（机制见 LLM-Inference 网页版）。

传图两式：公开图用 URL 省事，私有敏感数据用 base64 或私网——别把内部图片挂公网。

08语音与实时交互#mm-voice-realtime

语音是「实时系统」问题，不是「接个 ASR 就完了」：人类换话的自然空隙是 300–500ms，超过就被察觉；级联生产目标 P50 <1.5s、端到端标杆 <800ms、 >1.5s 用户判定「坏了」开始重复说话——ASR 把两遍话拼在一起，错误级联放大。延迟不只是体验问题，还是准确率问题。

级联管线时延拆账

两个开关：全链路流式（任何一段等全量输出，延迟立刻爆表）；瓶颈在 LLM 首 token——模型档位选小、提示词减脂、KV 缓存复用系统提示、工具调用能预取的别现查。

级联 vs 端到端

级联端到端
怎么走ASR → LLM → TTS，中间落文字声音进声音出，中间不落文字——语气笑声犹豫都保留
延迟基准砍约 85%
代价——中间没有文字可审计：护栏、质检、合规都要重做
2026 企业生产仍以它为主——可审计、可换供应商、成本可预测体验优先的场景

端到端四家：gpt-realtime 家族（GA、128K、工具调用进音频环路）、Gemini 3.1 Flash Live（~200ms 首响、200+ 语言）、Qwen3-Omni（开源可自部署、113 语种）、Moshi（全双工先驱）。

打断（Barge-in）三连动作

少做任何一件都穿帮：

① 停 TTS 播放

② 砍断 LLM 生成

③ 上下文回滚到打断点

VAD 太灵敏背景音就打断、太迟钝半天没反应，现代做法是语义 VAD——不光听音量还判断「话说完了没」（停顿 ≠ 说完，想想报电话号码）。

10现场高频追问#qa

只收会改变某个决定的问题——会改变技术方案、风险判断、验收方法、对客沟通或下一步行动。

每题标出它加入本题库的日期：2026-07-09 来自讲义首版对练页，2026-07-10 来自语音实时章，2026-07-17 来自视频生成章。

我的场景到底要不要上多模态？2026-07-09

一句话判据：输入里有关键的非文字信息 + 任务需要「理解」而非「转录」——两个都 yes 才上。纯文本客服硬上多模态只会更贵更慢。

客户说「要个多模态」往往只需要能力谱系的一格（如图像→OCR）——先定位到那一格，方案立刻清晰。

下一步　用分诊器五问走完，多数企业文档场景的答案是「拼管线打底 + 原生兜底 + 托管起步」。

依据　判断标准

该用原生多模态，还是自己拼 OCR + LLM？2026-07-09

版式固定、只要抠字段、要省钱可控 → 拼管线；版式多变、要理解/开放问答 → 原生。很多真实系统是混合：稳定部分管线省钱，疑难部分原生兜底。

这是本册最值钱的决定：拼管线的短板是 OCR 丢了版式和图表语义，「看图推理」答不了——原生是它能力上限之外的兜底。

下一步　同一份扫描合同两条路各跑一遍，比准确率/成本/ 维护——这就是对客答疑的现成弹药。

依据　灵魂取舍

传一张图到底要多少钱？为什么比文本慢？2026-07-09

图被折算成视觉 token，和分辨率强相关：低分辨率几十~百来 token，高分辨率/大图切块上千。慢也是同一个根：视觉 token 多 → prefill 重 → 首 token 延迟高。

「分辨率 ↔ token ↔ 成本/延迟」是本册所有账的根——省钱三招：够用的分辨率、稳定字段走 OCR、缓存去重。

下一步　上线前用真实图量做 token 压测 + 月度成本估算，设分辨率上限——别等账单来了才发现。

依据　成本账

开源 VLM 能打过闭源旗舰吗？2026-07-09

单项基准已逼近：InternVL3-78B MMMU ≈ 72.2%（MIT 许可）、Qwen3-VL 文档基准最强开源；但综合能力、稳定性、长上下文闭源旗舰仍领先。看场景。

对客报数别只报名字：MMMU 测综合理解、OCRBench 测文档——「在什么基准上多少分」才是可比口径。

下一步　数据敏感/量大再评估 vLLM 自部署开源 VLM；多数项目从托管 API 起步。

依据　模型格局

模型看图会不会瞎编、看错？2026-07-09

会。空间关系常答反、计数是老大难、小字密集表格会丢、图里没有的也可能言之凿凿。关键场景配二次校验、专用检测模型和人审兜底，别让 VLM 单点决定。

把短板和兜底一起讲，比吹「无所不能」更建立信任——构造 5 道陷阱题（空间/数数/小字/推理/无中生有）是最好的边界演示。

下一步　为场景建 20~50 条小评估集（图+标准答案），作为选型和验收依据。

依据　失效模式

生成图片用扩散还是自回归？2026-07-09

要艺术感/写实用扩散（Midjourney、FLUX.2）；要按复杂指令、带准确文字（海报、 UI、信息图）用自回归（GPT Image 2、Nano Banana 2）。

「美术生 vs 逻辑生」的分野在于生成方式：扩散从噪声去噪（美学强逻辑弱），自回归逐 token 拼图（指令与文字渲染强）。

下一步　同一条含空间约束和文字要求的提示词两派各生成一次——是对客解释生成选型最直观的 demo。

依据　两大流派

端到端语音又快又自然，为什么还推级联？2026-07-10

快是真的（延迟砍约 85%），但中间没有文字可审计——护栏、质检、合规都要重做，供应商也只有少数几家。电话客服、金融这类要留痕可控的场景，2026 主流仍是级联。

级联的三大优势正是企业要的：全程文字留痕、ASR/LLM/TTS 各自可换、成本可预测（$0.01–0.17/分钟）。

下一步　进生产先问三件事：接不接电话（SIP）、要不要私有化（Qwen3-Omni 自部署）、合规要不要文字留痕（要 → 级联）。

依据　级联 vs 端到端

语音客服的延迟能压到多少？2026-07-10

分架构说：端到端能进 1 秒（标杆约 800ms）；级联生产目标 P50 1.5 秒内。瓶颈通常在 LLM 首 token（350–1000ms 占大头）——先测基线再承诺数字。

超过 1.5 秒用户开始重复说话，ASR 把两遍话拼串——延迟同时是体验问题和准确率问题；衡量口径永远是「首音时延」。

下一步　三处下手：模型档位选小、提示词减脂（2000 字→ 200 字）、全链路流式化——优化前后对比就是给客户的证据。

依据　时延拆账

视频生成到底能不能商用了？2026-07-17

素材级商用已成立（营销/社媒/预演），成片级还要人工后期。对客设预期：它是「素材工厂」不是「成片机器」，方案里留后期工位。

按用途分层是性价比来源：批量素材 Fast 档（$0.022–0.029/秒）、叙事用 Sora 2 故事板、品牌大片高保真档——34 倍价差要用起来。

下一步　合规前置：生成出口打标识、转码保元数据、真人相关内容走授权链路——「能不能播」比「能不能做」先答。
