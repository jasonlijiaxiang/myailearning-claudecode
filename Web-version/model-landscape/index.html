01全景地图#ml-map

2026 年的模型市场分成三大阵营：闭源西方旗舰卖「最强能力」、开放权重（中国主导）卖「可控与便宜」、端侧小模型下沉到设备。三者是分工不是竞争——一个企业方案常常三者并用：旗舰打最难的仗，开放权重管可控与走量，小模型进端侧。

两个市场：同一批模型，两张榜单

引用任何份额数字前，先说清是哪个市场的口径——企业与消费两张榜单已经分裂。企业市场（Menlo 口径）看能力、信任、合同：Anthropic 40% 第一、OpenAI 27%（从 2023 年的 50% 一路下滑）、Google 21%（三年三倍）。消费市场看品牌与生态：ChatGPT 约 74% 独占、日均 25 亿+ prompts——与企业榜完全两样。

企业侧的量级也值得记：2026 年企业 LLM 支出已达 $8.4B、年底看 $15B——这是生产预算不是试验预算；Claude Code 单产品年化 $1B、企业编码市场 54%，是「编码成为第一生产场景」的标志性证据。

核实 2026-07-10，来源 Menlo 企业报告。份额的赏味期约半年，引用时带口径与日期。

02闭源旗舰五张名片#ml-closed

五家各有一件武器：OpenAI 靠生态、Anthropic 靠企业信任、Google 靠自供算力压价格、 xAI 靠性价比进场、Meta 在转身。没有一家全赢——这一节的正确用法是配合客户的「任务 × 预算 × 信任要求」三坐标查表。

OpenAI · 消费王者

GPT-5.6 三档（Sol $5/$30、Terra $2.50/$15、Luna $1/$6，2026-07-09 GA）。ChatGPT 约 74% 消费份额、工具链最全；企业份额 50%→27% 被 Anthropic 反超。缓存读折 90%、写 1.25×。

Anthropic · 企业新王

Claude 5 家族：Fable 5（Opus 之上的新档位，1M 上下文/128K 输出）、Opus 4.8 企业与 agentic 编码主力、Sonnet 5 尝鲜价 $2/$10（至 2026-08-31，后 $3/$15）。企业支出 40% 第一、企业编码 54%。

Google · 价格压制者

Gemini 3.1 Pro 是旗舰中 API 最低价，推理与 1 小时级长视频是独门强项；Flash-Lite $0.10/$0.40 为最低闭源档。底气是自研 TPU（Tensor Processing Unit）自供算力——别家买卡它自产。

xAI · 性价比刺客

Grok 4.3：$1.25/$2.50、缓存输入 $0.20，比上代降 40–60%，站上性价比 Pareto 前沿； 2026-06-15 上 Bedrock（第三家独立实验室），SOC 2 / HIPAA 合规面快速补齐。生态新、案例少，更要评估集验收。

Meta · 从旗手到转身

Behemoth（2T）实质搁置；Muse Spark（2026-04-08）成首家闭源 API-only 模型； Llama 4 Scout/Maverick 成「最后的开放版本」。给客户的启示：供应商策略无常，可换架构才是常量。

03开放权重格局#ml-open

开源榜首已经换人：前四全是中国队。多数基准差距收敛到个位数，编码基准追平——「开源能不能用于生产」在 2026 年的答案是能，真正的门槛不在能力，在许可证与自部署运维。

追平靠什么：三板斧的因果链

追平不是「砸更多卡」砸出来的，是三项技术叠出来的：MoE（Mixture of Experts）稀疏激活让大参数只花小算力（Qwen3.5 有一档 122B 总参、每次只激活 10B，即以此超过 GPT-5-mini；另有 397B 推理档）；从前沿模型蒸馏把闭源模型的能力「教」给开放模型；合成数据管线解决高质量训练数据的供给。架构机制在 LLM 原理模块，训练侧在 LLM-Training。

追平也有边界：最难的推理任务闭源仍领先半个身位——所以三层路由里两边都有位置，不是二选一。

西方线：收尾、转宽松、下沉

Llama 4（Scout 109B/激活 17B、10M 上下文；Maverick 400B/17B、1M）成最后开放版本，存量生态最深、新项目谨慎押注；Mistral Large 3 / Small 4 已转 Apache 2.0（2026-03），欧系合规友好； Gemma 4 下沉端侧——26B MoE 激活约 4B，E2B 2.3B 有效参数 4GB 内存可跑。三个关键词：收尾、转宽松、下沉——开放权重的创新中心已东移。

榜单口径 2026-07-10 快照，来源 BenchLM / Artificial Analysis。排名月月在换，「前排是中国队」这个结构一年内稳定。

04中国格局与豆包#ml-china

国产四强各有一个「超越时刻」——各自至少一个主流基准超越国际闭源。对客讲国产格局时，「不行」的说法已过期，正确的问题是「哪家配哪个场景」。

豆包：两条腿走路的国民模型

家族一条主线加两条专线：2.1 Pro 主线旗舰（2026-06-23 发布，¥6/¥30，缓存命中 ¥1.2）、 2.1 Turbo 半价走量档、Seed-Evolving 专线月更 2–4 次（Coding/Agent 最激进迭代）、 Seed-2.0-lite 家族首个原生统一全模态（2026-05）。累计 tokens 调用 180 万亿+，国内第一梯队。

理解定位的钥匙是「模型即入口」：C 端免费换规模、B 端绑火山方舟换收入——模型是云的获客入口，云是模型的商业化出口。这决定了它的价格永远激进，也解释了它为什么不开源（开源会削弱这个飞轮，与 Qwen/DeepSeek 的开源换生态是两条都跑通的路）。

绕不开的两个战略题

算力自主：GLM-5 全程用华为昇腾训练、零英伟达——「没有英伟达就训不出前沿模型」的说法已被证伪，私有化方案里「国产算力档」有了前沿模型选项（深潜见 AI-Infra-Compute）。出海与合规双轨：国产模型出海与国内商用是两套规则，国内涉及备案与内容合规（见 Security 第 8 章）——涉及跨境场景先标注「合规待确认」，别当场拍板。

两种货架哲学，决定议价权

百炼/千帆是「全家桶统一纳管」，方舟/腾讯以自研为主轴、三方为辅。货架宽度接近，哲学不同——差别体现在客户的议价权与锁定度，这比单价更值得先聊。锁定真正来自数据与工作流而非 API：用兼容层/网关保住切换能力（机制见 AI-Gateway），重要场景保留第二货源。

选型三问：问完名单自然收敛

① 客户已经绑在哪朵云、哪个生态？——云资源、数据、投放渠道在哪家，迁移成本就偏向哪家，「顺路」常胜过「最优」。② 要自研旗舰还是多模型组合？——认准单一旗舰看自研档位与更新节奏，要组合与议价权看三方货架宽度。③ 私有化与合规的底线在哪里？——需要专有云就先筛掉纯公有云方案；备案与内容安全配套各家都有，材料支持深度不同。

口径对齐四步：表面单价会骗人

¥6/¥30（豆包 2.1 Pro）、¥2.5/¥10（Qwen3-Max ≤32K）、¥0.8/¥2（混元 2.0）三组数字不可直接比大小。对齐口径的固定动作：

① 对分段与档位

方舟按输入长度分段计价、百炼按上下文档位定价（≤32K 与 ≤128K 不同价）——先把客户的真实长度分布落到对应档。

② 对缓存价

豆包缓存命中 ¥1.2 vs 名义 ¥6——命中率 60–90% 的场景等效价差数倍。用第 7 节的缓存公式换算。

③ 对批量与订阅

百炼 Batch 一律 5 折、方舟 Coding Plan 订阅制、腾讯订阅+用量两条线——离线任务和高频开发场景先套这些再比。

④ 再比大小

按客户真实流量分布换算成「每月总账」再对比——差距常明显收窄，结论也才站得住。

06许可证与合规#ml-license

先把词用对：open weight ≠ open source。按 OSI 2024-10 发布的 OSAID（Open Source AI Definition）v1.0，开源要求开放训练数据信息、架构与训练代码——按此标准，Llama/Gemma/Qwen 都只是「开放权重」：送你一桌做好的菜，不给菜谱。法务在场时，这个区别值钱。

级别代表商用含义
放心用（MIT / Apache 2.0） DeepSeek 系（MIT）、GLM-5.1（MIT）、Qwen3 主系（Apache 2.0）、Mistral 新版 可商用、可闭源二开、可再分发——法务最省心
读条款（社区许可证） Llama（700M MAU 帽 + 欧盟条款）、Gemma（自家条款） 多数场景可用，但有触发条款——必须读原文
谈合同（仅 API / 专有） GPT / Claude / Gemini / 豆包（方舟） 无权重；商用边界在服务合同与数据条款里

Llama 的两个雷与蒸馏暗礁

Llama Community License 两处要提前核：700M MAU（Monthly Active Users）帽——月活超 7 亿需向 Meta 另行申请授权；欧盟条款——多模态版本对欧盟实体有使用限制。给大厂或欧企做方案时必查。更隐蔽的是蒸馏继承：用 Llama 蒸馏的学生模型仍可能带 Llama 条款——微调/蒸馏选型时基座许可证要先过。

商用检查四问

① 这个权重的许可证原文是哪份、哪个版本？② 微调/蒸馏出来的衍生模型受什么条款约束？ ③ 允许再分发吗（打进产品卖给客户）？④ 你的客户的客户使用，算不算触发条款？——四问答完再进方案。趋势对采购有利：宽松化是 2026 主旋律（Mistral 转 Apache、中国主力选 MIT），但 Llama 系例外要单独核。

口径依据各 LICENSE 原文与 OSAID v1.0，核实 2026-07-10，来源 LICENSE 原文 / HF blog。许可证是少数「答错会出事故」的领域，拿不准就说「以法务核定为准」。

缓存经济学：最容易忽视的成本杠杆

缓存读折 90%（OpenAI 口径，各家类似）、写按 1.25×；豆包命中价 ¥1.2 对名义 ¥6。等效成本的算法：

真实成本 = 名义单价 ×（1 − 缓存命中率 × 折扣率）。系统提示、少样本示例、工具定义都是高复用段——客服/Agent 场景命中率常见 60–90%，等效价直接砍半以上。对客报价必问一句「你的提示词有多少是重复的」。命中率背后的机制（前缀缓存）在 LLM-Inference 网页版批处理一节。

报价纪律

① 写量级不写精确价，精确价放附录并标日期；② 注明「以官网当日为准」并给活水源链接； ③ 合同锁「档位与降价传导」，不锁单一型号价格——过去一年市场降了 30–60%，不写降价传导客户就吃亏。thinking 模式的账单问题（思考 token 十倍于普通回答）见能力矩阵的预算控制。

08能力矩阵与推理模型#ml-capability

能力比较看三个维度：上下文窗口、模态、思考预算。每个维度都有一个反直觉的「但是」：窗口大 ≠ 记得住，模态全 ≠ 原生，思考久 ≠ 更准。

上下文：窗口 ≠ 有效窗口

三档谱系：128K 常规档（多数生产模型标配）→ 1M 长文档档（Fable 5 / Gemini / Llama 4 Maverick）→ 10M 极限档（Llama 4 Scout，生态检验中）。三个「但是」：①长文中段召回会衰减； ②成本随长度暴涨（平方级根源见 LLM 原理）； ③「1M 窗口杀死 RAG」是伪命题——权限、新鲜度、成本三关 RAG 仍不可替代。对客话术：窗口决定「能不能塞进去」，不决定「记不记得住、划不划算」。

多模态：原生还是拼的

模型线图像音频视频备注
GPT-5.x 系原生原生+实时理解Realtime 语音线成熟
Gemini 3.1 系原生原生+实时原生（1 小时级）长视频独门
Claude 5 系原生——专注文本与工具链
Qwen-Omni / 豆包 Seed-2.0-lite原生原生原生开源与国产全模态代表

按「原生统一 vs 外挂拼接」看：原生的体验与延迟优势明显——机制深潜见 Multimodal 模块。

思考预算：拨盘，不是开关

两种旋钮：L1 可控——你来定预算（Anthropic budget_tokens、Google thinkingBudget、 OpenAI reasoning effort 分档），适合成本敏感、难度可预估的场景；L2 自适应——模型按难度自动调深浅（Fable 5 的 adaptive thinking 是代表），省心但成本波动大，要配预算上限。全部头部实验室已收敛到「快慢混合」范式。

overthinking 有实证：超过临界思考预算后准确率稳定下降（arXiv 2506.04210）——简单任务开满档 thinking 更慢、更贵、还可能更错，像考试检查太多遍开始把对的改错。工程对策：按任务难度分级配预算、评估集里加「简单题不许想太久」的用例。帮客户把预算调到「刚刚好」，一次优化常省 30%+ 推理成本。思考 token 烧的是 decode——机制见 LLM-Inference 网页版。

思考预算与 overthinking 口径来自 arXiv 2507.02076 / 2506.04210，核实 2026-07-10。

09选型方法论#ml-selection

多模型组合是 2026 年企业默认解，单一供应商反而需要特别理由。证据：Walmart Code Puppy 在 GPT/Claude/Gemini 间动态路由（路由器成了核心资产）；ServiceNow 2026-01 数周内同签 OpenAI 与 Anthropic 多年协议；Gartner 预计 2028 年 70% 多模型组织将用 AI 网关能力（2024 年不足 5%）。

三层路由：1 : 8 : 1

五约束决策树（也是下方筛选器的判据）

硬约束收敛方向
数据不能出域自托管开放权重 + 许可证核对（四问清单）
预算敏感 / 超高调用量地板档 + 缓存优化 + 三层路由
要最强 agentic / 编码企业档旗舰（Claude 系 / GPT 系）+ 评估集实测
中文为主 / 国内交付国产四强 + 豆包按场景配对表选
必须保持可换网关抽象 + 评估集回归 + 合同降价传导

顺序有讲究：先用硬约束砍掉大半选项，再在剩下里比性能——反过来会陷入无穷的榜单争论。

选型筛选器

按五约束收敛候选名单。数据为本页已核实口径（价格 2026-07-10、平台 07-17），结果是「首看名单」——终审永远在你的评估集。

两道防线与验收

防线一：对榜单保持怀疑——Leaderboard Illusion 已证实厂商私测多变体、择优公开（曾有厂商私测 27 个变体只发最好那个）；MMLU 类基准饱和在 88%+ 拉不开差距。榜单只做初筛。防线二：自建评估集终审——用客户真实数据抽样建评估集（方法在 Evaluation 第 5 章），换模型 = 跑一遍回归。评估集是客户的资产，模型是易耗品——这也是「保持可换」的技术前提。

升级也要当发布链管：锁定当前版 → 候选版 → 离线评测 → 金丝雀放量 → 晋升或回滚；收到退役通知先跑回归证据，不是直接换模型 ID。终极验收一句话：在真实部署端点上，模型以可接受成本持续产出合格结果，并能被观察、升级、回滚和替换——Demo 里能回答只证明了最浅的一层。

把这句拆成选型验收的六条证据线——每条都要索取具体证据件，不是听承诺：

验收面要索取的证据件
匹配 + 质量任务与输出合同匹配（业务样本 + 错误影响）；关键切片与红线合格（可重跑的评测包）
运行 + 治理p95 / p99、容量、恢复达标（代表负载报告）；许可 / 区域 / 数据 / 工具可控（处理清单 + 责任矩阵）
经济每合格结果成本可承受——端到端 TCO，不是单价表（缓存 / 路由都算进去）
生命周期升级、回滚、主备与退出可执行——版本登记与 runbook

案例与趋势核实 2026-07-10，来源 Walmart/ServiceNow 报道与 Gartner 转引；Leaderboard Illusion 见 arXiv 2504.20879。不可外推边界：Gartner 的占比是预测不是实测、且为转引口径；两个客户案例是公开报道的单点，能说明「多模型已是大客户的现实选择」，不能当成行业普遍做法的证据。

现在到底谁家模型最强？2026-07-10

「最强」永远带定语：企业支出 Anthropic 第一（40%），消费端 ChatGPT 独占 74%，推理基准 Gemini 领先，开源榜首是 DeepSeek。要先问「哪个市场、什么任务」。

这个问题答成排位赛就输了——两张榜单已经分裂，不带口径的「第一」都是误导。它正是需要选型方法的原因。

下一步　把对方的场景落到「任务 × 预算 × 信任」三坐标，再查五张名片表，30 秒内给出带定语的答案。

依据　全景地图

GPT 和 Claude 到底该选哪个？2026-07-10

先问任务：编码与企业 agentic 场景 Claude 系胜率高（企业编码份额 54%）；生态与多模态工具链 OpenAI 更全；成本敏感看 Gemini。通常建议双供应商起步、网关路由、评估集分胜负。

二选一的提问方式本身就该被纠正——多模型组合是默认解，选型从站队之争变成工程问题，才是解决方案人的价值。

下一步　用 20 条对方真实业务 prompt 做迷你评估集盲测，半天出属于他的答案。

依据　五张名片 · 选型方法论

开源模型和 GPT 们到底还差多少？2026-07-10

多数基准差距已到个位数，编码基准追平（SWE-bench Verified 80.6%）。还剩两个差距：最难推理任务闭源领先半个身位、工具链成熟度。结论是分任务用，不是二选一。

追平的机制（MoE + 蒸馏 + 合成数据）决定了这不是昙花一现——「开源不行」的直觉停在 2024 年，答对这题直接建立专业信任。

下一步　按三层路由把对方的任务分层：走量与敏感数据层开源上，最难 10% 留给闭源。

依据　开放权重格局

Llama 不是开源吗，为什么法务不让我们用？2026-07-10

它是「开放权重 + 社区许可证」：有 700M MAU 触发条款和欧盟限制，法务谨慎是对的。要许可证干净的替代，DeepSeek（MIT）和 Qwen（Apache 2.0）能力相当且条款更宽松。

这题的价值在纠正词汇：open weight ≠ open source（OSAID 口径）。用词严谨，法务和采购才会把你当内行。

下一步　用商用四问清单过一遍对方方案里的所有模型，输出一页纸给法务预审。

依据　许可证与合规

用开源模型微调出来的模型算谁的？2026-07-10

权重归你，但受基座许可证约束：MIT/Apache 基座的衍生模型基本自由；Llama 基座的衍生模型继承其条款（含命名义务）。微调选型时基座许可证要先过。

蒸馏是隐蔽雷区——用 Llama 蒸馏的学生模型仍可能带 Llama 条款。这个坑法务常常也不知道，提前点出是加分项。

下一步　微调方案里加一行默认检查项：「基座许可证与衍生条款已核对」。私有化微调的工程侧见 Fine-tuning 模块。

依据　许可证与合规

为什么不全用最便宜的 DeepSeek？2026-07-10

地板档适合批量明确任务；难推理与关键链路需要更高档位兜底。三层路由的做法是 80% 任务走便宜档、难题升档——总成本比全用旗舰低一半以上，比全用地板档少踩坑。

「全用最便宜的」和「全用最强的」是同一个错误的两个方向：都没按任务分层。路由比型号更值得投入。

下一步　拿对方一天的真实请求抽样，按难度分三层估比例，算一遍混合单价对比单一模型单价。

依据　价格与成本 · 三层路由

现在签一年合同，明年价格跌了怎么办？2026-07-10

合同锁「档位与降价传导」而不是锁型号单价：官网降价自动传导。过去一年市场降了 30–60%，不写这条就吃亏。

「模型是快速贬值资产」在合同条款上的直接推论。这条也是把「价格战」从威胁变成客户利益的话术支点。

下一步　把报价纪律四要素写进方案书模板：量级、日期、活水源、降价传导。

依据　报价纪律

上下文窗口越大越好吗？2026-07-10

不是。窗口决定能塞多少，不决定记不记得住（中段召回衰减）和划不划算（成本随长度暴涨）。生产里常见 128K 窗口 + RAG 精准喂料，比硬塞 1M 又准又便宜。

「1M 窗口杀死 RAG」是高频伪命题：权限、新鲜度、成本三关 RAG 仍不可替代——答清楚这题能拦住一个方向性错误。

下一步　长上下文为什么两头吃钱（prefill 平方级 + KV 线性级），用 LLM-Inference 网页版的机制页现场讲给对方。

依据　能力矩阵 · LLM-Inference 网页版

评测榜单第一的模型，为什么你们不推荐？2026-07-10

榜单有已证实的水分：厂商私测多变体择优公开（Leaderboard Illusion），且热门基准已饱和（88%+ 无区分度）。榜单只用来初筛，终审永远是客户的数据、客户的评估集。

这题把「你们是不是不客观」的质疑转化成方法论输出——两道防线讲完，选型就从站队变成了工程。

下一步　给对方看 arXiv 2504.20879 的结论一页，然后提议共建 20–50 条的迷你评估集。

依据　两道防线 · src-arxiv

国内这几个平台，到底选哪个？2026-07-17

先答三问：绑哪朵云、要旗舰还是组合、私有化底线——答案通常自动收敛到一两家。没有普适最优，只有与现状合拍：字节生态看方舟、要全家桶看百炼、搜索增强看千帆、极致性价比与微信分发看腾讯。

平台选型比模型选型多一层「生态引力」：云资源、数据、投放渠道在哪家，迁移成本就偏向哪家——「顺路」常胜过「最优」。

下一步　价格对比前先走「口径对齐四步」，把分段计价、档位、缓存、批量折扣对齐再谈大小。

依据　国内平台格局

平台会不会把我们锁死？2026-07-17

锁定来自数据与工作流而非 API——用兼容层/网关保住切换能力，重要场景保留第二货源。「全家桶纳管」与「自研主轴」两种货架哲学的锁定度不同，签约前值得先聊这个。

这题是「保持可换」总纲在平台层的落点：合同、网关、评估集三件套齐了，锁定风险才真正可控。

下一步　方案里画出网关抽象层的位置，标注切换演练的验证清单——机制见 AI-Gateway 模块。
