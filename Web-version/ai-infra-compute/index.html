01全景总览#aic-overview

五层栈自下而上：芯片（拼算力）→ 服务器（拼装配散热）→ 机柜（拼互联带宽）→ 集群（拼网络与容错）→ 数据中心（拼「电」）。像发电厂：客户买的不是发电机，是稳定出来的电——客户要的是 token，不是卡。与传统机房的差别是三个数量级：功率密度 ×10 （传统机柜 5–15kW vs NVL72 一柜约 120kW——风冷到头、液冷必选）、东西向带宽 ×100 （卡间互传梯度的「海啸」）、故障频率随规模线性放大（同步训练一卡出错全作业停摆）。

评估与规划的顺序：从约束最硬的层开始

训练推理

投入方式一次性重投入持续账单、跟流量走

行业预测 2030 年推理占 AI 总算力约 75%——主战场正从「谁能训」转向「谁跑得省」，「每 token 成本」是长期主指标。训练推理共用一套硬件能但不划算：常见做法是训练集群固定重资产、推理按流量弹性，两套池子各自优化。

02GPU 解剖#aic-gpu

CPU 是延迟机器（几位教授：什么难题都能解，一次只解几道），GPU 是吞吐机器（一万名小学生：只会加减乘，一声令下一万道口算题同时开工）——深度学习 90%+ 是矩阵乘法，天生就是「一万件一样的事」。AI 算力标称值几乎全来自 Tensor Core（矩阵机床，一次冲 16×16）；看规格表主角是 Tensor Core 算力 + 显存带宽两行。小 batch 跑不满的硬件根源也在这：机床一次冲一整块料，你只送来一根料条。

精度阶梯

位宽减半 → 同样硅片和带宽处理两倍的数 → 标称算力翻倍，代价是数值精细度。像记账：日常买菜记整数够用（推理），公司年报还得保留小数（训练主权重）：

精度每个数占现状
FP324 字节训练用混合精度，主权重要保留小数
FP16 / BF162 字节
FP81 字节已是生产推理默认精度、训练主流实践
FP4半字节NVFP4 面向 Blackwell，工具链成熟中

对客铁律：对比两张卡的算力必须对齐精度口径——「B 卡是 A 卡 5 倍」常有一半来自 FP8→FP4 的口径切换，而非硅片进步。

Roofline：这块负载吃算力还是吃带宽

像自助餐厅——大锅菜拼厨师（算力），一人一碗面全堵在传菜通道（带宽）：

负载计算强度顶到哪意味着
训练 / prefill高平屋顶吃算力
decode 逐 token低——每字节只有个位数运算斜屋顶永远带宽受限，算力利用率低是物理规律不是配置错

H100 级平衡点约 300 FLOPs/Byte。

MFU 是诚实指标（把通信等待、数据加载、重算全折进去）：大规模训练 35–45% 算优秀、Llama 3 报告 38–43%——顶级团队也只吃到四成。nvidia-smi 的「利用率 95%」只表示有 kernel 在跑，不等于 MFU。帮客户估训练时长用 MFU 40% 打底，不用标称值。

03显存与 HBM#aic-hbm

HBM 把内存「立起来」焊在 GPU 旁边：DRAM 堆叠 8–16 层、TSV 硅通孔直通（平房改高层公寓），距离近 + 接口宽做到普通内存几十倍带宽（B200 级 ≈8TB/s vs 服务器内存几百 GB/s）。容量决定装不装得下、带宽决定跑得快不快，且两者都出厂焊死不可扩展——换容量 = 换卡，选型按未来两年的模型和上下文留余量。

世代与供应

世代状态（2026-07 口径）
HBM3e在役主流——H200 141GB、B200 192GB、GB300 288GB/卡
HBM4量产开跑：Samsung 2026-02 全球首家量产（11.7Gbps）；SK hynix 2026 商用爬坡、预计拿下 NVIDIA HBM4 约三分之二份额。Rubin、MI400 都指定 HBM4

供应格局 = 三家（SK hynix / Samsung / Micron）供全球，产能锁死整机出货节奏——显存不仅是技术瓶颈，还是商业瓶颈。

H200 比 H100「算力差不多凭什么贵」

141GB vs 80GB、4.8 vs 3.35TB/s——decode 带宽受限，显存升级直接换成推理吞吐，对推理客户每 token 成本反而更低。

两个口袋公式覆盖现场八成显存问题：训练看「参数 × 16 字节 + 激活」（权重 2 + 梯度 2 + 优化器状态 12——优化器才是大头；7B 全参 ≈112GB 单卡装不下，这就是「7B 也要多卡」和「LoRA 能单卡」的算术根源）；推理看「权重 + KV Cache」（FP8 下 70B ≈ 70GB； KV 随并发 × 上下文线性涨，并发一高反超权重成大头——下半本账在 LLM-Inference 网页版）。显存不够的软件三板斧：量化砍半、KV 分页与前缀复用、多卡切分——各有代价，通常先量化再谈加卡。

HBM 世代与供应核实 2026-07-09，来源 TrendForce / DigiTimes 等。不可外推边界：供应与世代节奏是市场快照，随产能和代次切换按季度变；它支持的是「HBM 是瓶颈也是成本大头」这个结构判断，具体交期与价格必须问当期报价。

05Scale-up 互联#aic-scaleup

带宽阶梯记三个量级：卡内 HBM ~8TB/s > 机柜内 NVLink ~1.8TB/s > 跨机柜网络 ~100GB/s——每跨一级慢一个数量级。并行策略的本质是「把最话痨的切法塞进最快的域」：张量并行（两个人抬桌子必须肩并肩、每步喊口号）和 MoE 专家并行的 AllToAll 只能留在 scale-up 域；数据并行（各小组分楼层干活、定期汇总）走普通网络就够。

NVL72 = 一台 120kW 液冷计算机

NVLink 5每颗 GPU 18 端口聚合 1.8TB/s
NVSwitch机柜里的「电话总机」——72 颗 GPU 全连接、任意两颗全速直达、域内总带宽 130TB/s，GPU 访问彼此显存像访问自己的（内存语义、纳秒级）
一柜装什么36 Grace CPU + 72 Blackwell GPU + NVSwitch 托盘，机柜内约 5000 根铜缆
为什么用铜两米内用铜便宜省电免光模块——「把话痨通信留在柜内」既是性能设计也是成本设计

按柜交付、按柜调度、按柜规划供电与故障域。scale-up 域的大小直接决定能高效跑多大的模型——8 卡机 NVLink 域是 8、NVL72 是 72，按模型规模倒推。

挑战者：UALink 与 Broadcom SUE

UALink 1.0 是 AMD/Intel/Broadcom 阵营，2025-04 定稿，单层交换至多 1024 端点。2026 年的诚实判断：标准已立、生态在建、原生硬件看 2027——值得跟踪、暂不建议做主力生产依赖；生产决策按 NVLink 生态、合同留二供评估条款。

06Scale-out 网络#aic-scaleout

RDMA 是地基：网卡直接读写远端内存（配 GPUDirect 直达显存）、全程绕过 CPU 和内核——传统 TCP 像快递层层过收发室，RDMA 是快递员持卡直接把货放进你家货架。买 AI 网络本质是买 RDMA 的承载方式：IB 原生承载、RoCE 用以太网承载、UEC 是重新设计的承载。

先画流量画像再选网

NCCL 把并行策略翻译成网络流量——像合唱团对拍子：慢的那一个人决定全团速度，木桶效应在万卡被放大一万倍：

并行策略翻译成什么流量性质放哪儿
DP 数据并行每步一次大 AllReduce带宽型，可与计算重叠普通网络够用
TP 张量并行每层多次小同步延迟型留在 NVLink 域
EP 专家并行AllToAll，打散全网最考验交换机对网络要求最高

「加了卡反而没快」：按四层对齐查

并行策略与物理网络对不齐，是「GPU 利用率挺高、训练就是慢」的头号病因。把它拆成四层，任何一层错配都会被放大：

层管什么典型错配
并行层DP / TP / PP / EP 的通信图：谁与谁说话TP 放到跨机——该走 NVLink 的走了以太网
集合通信层AllReduce / AllGather / 点对点：说多少collective 量算错，网络被打爆
节点层PCIe / NVLink / NIC / NUMA：走哪条路链路没走对，绕了远路
Fabric 层leaf-spine / rail / 对分带宽：怎么扩展EP 路由撞上超卖的对分带宽

先测再调：collective 微基准与拓扑健康检查先行，再调并行策略；通信图与物理放置当作同一份配置的两半来管版本。客户问「为什么加了卡反而没快」，就按这四层挨个查：通信图变了没、collective 量涨了多少、链路走对没、对分带宽够不够。

07存储与数据管线#aic-storage

GPU 每小时几美元，等数据的每一秒都在烧钱。训练 IO 两副面孔像食堂的两种时刻：数据加载 = 随机小读如雨（每步随机抽样本、读一次就扔、缓存几乎无用——考验出餐口数量）；checkpoint = 顺序大写如洪（TB 级状态整体落盘，1 分钟写完需要每秒几十 GB 聚合写带宽——开饭铃一响全员打饭）。评估 AI 存储只看容量必错：随机读吞吐 + 洪峰写带宽才是验收指标，按峰值不按平均设计。

标准分层三层起步

常见翻车：数据预处理管线没并行化，GPU 集群建好了、数据还在单机脚本里过——数据管线按「能喂饱多少 GPU」验收，不是「能不能跑通」。

08算力经济学#aic-econ

卡价只是首付——像买车：裸车价之外还有油费保险停车保养，开五年总花费接近两个车价。

TCO 四件套

硬件折旧

按 4–5 年摊。

电费

功耗 × PUE × 电价。一颗 1000W GPU 全年满载耗电近 9000 度，五年电费可达卡价两三成。

机房托管

场地、制冷、带宽。

运维人力

常被漏算的那一列。

参照锚点：H100 级云上时租约 $2–3/卡·时。

电是第一约束

风冷极限 30–40kW/柜，NVL72 的 120kW 必须液冷；PUE 从 1.5 降到 1.2 = 省 20% 总电费。

建 vs 租 vs API 决策树的根是利用率：自建对比长期租赁的盈亏平衡在利用率约 40–50%——低于它自建必亏；托管 API 约 $2–5/百万输出 token，小体量下几乎总是最便宜的起点。决策口径：日均利用率稳过一半且有运维团队 → 自建/长租；负载波动大或刚起步 → 云弹性；只调用模型能力 → 直接 API 别碰硬件。多数客户高估自己的利用率——先上云跑三个月拿真实数据再做自建决策。GPU 折旧按 4–5 年、梯次使用（前两年训练、之后退居推理）；真正要防的是「买了跑不满」。

建 vs 租盈亏速算器

TCO 框架的交互版：填客户参数，对比自建与云租的多年总成本、算出盈亏利用率。所有单价可编辑（默认为量级示例，非报价）；模型假设——自建 = 折旧 + 电费（随利用率）+ 托管运维附加，云租 = 时租 × 卡时 × 利用率。

容量与基准要可复核：四层证据

给客户报容量、报基准，先问四层证据齐不齐——四层必须能互相解释，对不上容量承诺就是拍脑袋：

结果层延迟 / 吞吐 / 达到目标质量的时间与成本
运营层排队 / 故障 / 余量 / 供货周期

两条硬纪律：①报告要给原始配置、分布和失败样本，不只报最佳点；②MLPerf 这类公开基准用于校准与筛选，不替代业务流量验收，容量按「可承诺 SLO 下的稳态结果」推导、不按峰值。缺哪层，合同里就给哪层留缓冲（与 LLM-Inference 的容量四证据同一套方法论，那边管推理服务、这边管算力采购）。

09现场高频追问#qa

只收会改变某个决定的问题——会改变技术方案、风险判断、验收方法、对客沟通或下一步行动。

每题标出它加入本题库的日期：2026-07-09 来自讲义首版对练页。

我们已经有机房，直接买卡装进去行不行？2026-07-09

大概率不行，先看电和冷却：传统机柜 5–15kW，AI 机柜几十到一百多 kW——液冷改造和供电扩容往往比卡还难买。很多客户最后选混合：存量机房跑推理、训练上云。

评估顺序是本册的第一课：电 → 冷却 → 网络 → 算力——顺序反了就买回一堆开不了机的卡。

下一步　四问两分钟自建体检：进线容量、液冷条件、楼板承重、然后才谈买卡。

依据　四个本质不同

标称几千 TFLOPS，为什么我们实测差那么远？2026-07-09

标称值是 Tensor Core 在理想大矩阵、特定精度下的峰值。实际负载有通信、搬运、小算子——顶级团队训练 MFU 也就 40% 上下；decode 更是带宽受限，算力天然吃不满。

nvidia-smi 的 90% 只说明「有活在跑」；Roofline 先问负载是算力型还是带宽型——带宽型算力利用率低是物理规律。

下一步　估时长算成本都用 MFU 40% 打底；能把 MFU 提 5 个点，等于白捡一成多的卡。

依据　MFU 与 Roofline

70B 模型到底要几张卡？2026-07-09

看用途：FP8 推理权重约 70GB，单张 141/192GB 卡装得下、还要给 KV Cache 留并发空间，1–2 张起步；全参训练按 ×16 字节算 1.1TB 级，8–16 张起配；LoRA 居中。

两个口袋公式覆盖现场八成显存问题：训练「×16 字节」、推理「权重 + KV」——给出并发和上下文就能当场心算。

下一步　显存不够先软件三板斧（量化/KV 分页/切分）再谈加卡；H200 类「只加显存」的卡对推理客户每 token 成本反而更低。

依据　两本显存账

不用 NVIDIA 到底行不行？2026-07-09

分负载：标准推理 2026 年已经行——vLLM 等框架多后端成熟，AMD/国产/ASIC 都有生产案例；大规模定制训练仍建议留在 CUDA 生态。常给「训练用 N 卡、推理第二供应商」的混合方案，还能增加议价权。

CUDA 护城河是二十年生态不是芯片——换芯片的真实成本是软件移植和踩坑时间；「负载是否标准」是能否离开的钥匙。

下一步　国产线合规/供应链硬约束优先；正确卖法是「确定性 + TCO」，把移植服务打进方案。

依据　格局与画像

NVLink 和 InfiniBand 到底什么关系？2026-07-09

分层关系不是竞争：NVLink 是机柜内的「总机」（1.8TB/s、内存语义，72 卡当一张用）， IB 是机柜间的「快递网」（800Gb/s、消息语义）。话痨的 TP/EP 走 NVLink，汇总类走 IB——两个都要买。

带宽阶梯每级差一个数量级；scale-up 域大小决定能高效跑多大的模型——这就是机柜级产品存在的意义。

下一步　IB vs RoCE 按三前提答：规模、并行画像、运维能力——Meta 双集群证明两条路都能训最大模型。

依据　带宽阶梯

存储要配多少才够？直接用对象存储训练行不行？2026-07-09

按两个峰值倒推：数据加载 = GPU 数 × 每卡消费速率（每卡百 MB/s 量级起步）； checkpoint = 模型状态大小 ÷ 可接受暂停时间。直接读对象存储小规模行、生产不行——延迟高随机读弱，GPU 会被拖垮。

三层标准架构：对象底座 + 并行热层 + 本地 NVMe；数据管线按「喂饱多少 GPU」验收。

下一步　推理集群主动带「KV Cache 分层方案」——存储花的钱从 GPU 账单里加倍赚回来。

依据　两副面孔与分层

自建和上云到底哪个便宜？2026-07-09

取决于利用率：盈亏线约 40–50%——长期稳过且有运维团队，自建更省；否则云更省。多数客户高估自己的利用率，建议先上云跑三个月拿真实数据再决策。

TCO 四件套（折旧/电/机房/人力）里卡价只是首付；五年电费可达卡价两三成——只报裸车价的方案后面每年都在加价。

下一步　用速算器按客户参数当场算盈亏利用率；API 转自建的三个信号：账单摸到盈亏线、合规要私有化、微调常态化。

依据　决策树

现在下单还是等下一代卡？2026-07-09

看上线时间和体量：今年要产出用在役卡——晚半年的产出损失通常大于新卡溢价；明年才用且量大，可谈下一代首批。新代次首年供应紧张，「等到了但拿不到货」是真实风险。

翻倍数字要对齐精度口径看（硅片真实进步每代约 1.5–2 倍）；交付时间永远是第三个选型参数。

下一步　做一版「等 vs 不等」的 TCO 对比；折旧按梯次使用规划（前两年训练、之后退居推理）。

依据　年更节拍
