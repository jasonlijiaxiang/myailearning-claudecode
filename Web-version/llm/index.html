01从序列问题到 Transformer#llm-why-transformer

语言的本质困难是长距离依赖：合同第 12 页的「上述甲方」指向第 1 页的定义条款。 两个硬要求——记得住远处的信息、分得清哪些此刻相关——正是注意力被发明出来要解决的问题。

Transformer 之前的 RNN 像传声筒游戏：逐词读入、把一切压进固定大小的记忆向量——三宗罪是 遗忘（长文装不下）、串行（GPU 上千核心干瞪眼）、梯度消失（学不会 长距离依赖）。2017 年《Attention Is All You Need》做了激进的减法：扔掉递归骨架，让注意力 从配角变唯一主角——所有词两两直连，信息路径长度恒为 1（直达电梯，不是 12 页的接力）。 最值钱的副产品是全序列并行训练：可以疯狂堆数据堆算力，直接解锁了 GPT 路线。

总览像一场圆桌会议：每个词是一位与会者，发言前先环顾全场（注意力）——谁与我相关就多听 谁的；然后回座位独立消化整理（前馈网络 FFN）。会要开 N 轮（N 层），每轮理解更深。 代价写在最后一行：注意力是 O(n²)——大模型的成本、延迟、上下文限制，根子都在这， 它是第 4、5 节全部工程故事的伏笔。一句话：Transformer 赢在可扩展——它让「变大」第一次 变得既可行（并行）又有回报（scaling）。

02注意力：QKV 拆解#llm-attention-qkv

注意力＝可学习的加权平均：每个词对全句所有词打「相关度分数」，按分数加权吸收 信息。「这个苹果很甜」vs「苹果发布了新手机」——同一个词，看谁、看多重，权重由数据 学出来，这就是它全部的魔法。

QKV 三件套用图书馆检索记：Q（Query）是你手里的检索词（我想找什么）、K（Key）是 每本书的书脊标签（我是讲什么的）、V（Value）是书的正文内容（真正被取走的东西）。为什么要 三个分身？「我在找什么」和「我能提供什么」是两回事——K ≠ V；三个矩阵都是训练出来的。 三步计算：点积算相关（方向越一致越相关，n 个词 n² 对关系——O(n²) 诞生在这一步）→ 缩放 + Softmax 分预算（除以 √dₖ 给分数降温防饱和；softmax 把分数变成总和 100% 的 注意力预算）→ 加权求和取货（新「苹果」≈ 0.6×V(发布) + 0.25×V(手机) + …，已经是 「科技公司的苹果」）。公式全貌只此一次：Attention(Q,K,V) = softmax(QKᵀ/√dₖ)·V—— 三步拆开，每步都是常识。

因果掩码：生成式训练是「预测下一个词」，第 i 个词只许看位置 1…i——做完形填空 不许偷看后文答案。妙处：一句 100 词 = 100 道练习题一次并行算完（训练效率的关键），也顺带 解释了 GPT 写到一半不会回头修改。多头注意力像专家会诊：把向量空间切成 h 份各看一面 （语法头盯主谓、指代头盯「它」指谁），拼接融合——头数不是越多越好（头数 × 每头维度 = 固定预算）。白板口诀：Q 问、K 答、V 给货，softmax 分预算。

顺带一条边界：多模态不是新架构——ViT 把图切成 16×16 的 patch，每块地位与词向量完全等同， QKV 原样复用，O(n²) 对图像同样成立（分辨率越高越贵）。深入见 Multimodal 模块。

03Transformer 全解剖#llm-architecture

一个 Transformer 块 = 注意力 + FFN（各带残差），所谓大模型就是同一个块堆 N 次。 解剖的要点是一笔参数账：参数大头在 FFN（约 2/3），不在注意力——「知识难更新」 由此而来，RAG 因此而兴。

组件干什么7B 级参数占比
嵌入表 Embedding语义 GPS：字典义坐标（「国王−男+女≈女王」）；上下文义靠 N 层加工~2%
注意力（Q/K/V/O）信息交换（第 2 节）~31%
前馈网络 FFN知识仓库与深加工：升维 4 倍、逐词独立；事实性知识大量存在这里~65%
LayerNorm 等数值稳压<1%

几个必答的「为什么」：位置编码——注意力天生不分先后（打乱词序点积不变），必须 显式注入位置；RoPE 把相对距离编码成旋转角度（座位牌不写第 5 号，写「你和每个人各隔几个 座位」），是主流开源模型的事实标准，超长上下文靠插值/YaRN 扩展。残差与 LayerNorm ——残差是高速公路主路，信息与梯度沿主路直达，百层网络训得动的根本原因；LayerNorm 是 稳压器。Decoder-only 为什么统治——「预测下一个词」一个目标通吃所有任务、配合因果 掩码训练信号最密、结构单一好规模化；BERT 系双向架构仍活跃在检索嵌入侧（RAG 检索端常见）。

MoE（Mixture of Experts）像三甲医院分诊台：1000 位专家在编（总参数），挂一次号 只见 2 位相关科室的（激活参数）。DeepSeek-V3 总参 671B、每 token 只激活约 37B——2026 主流 旗舰基本都是 MoE。看 MoE 要问两个数：总参数（存储账）和激活参数（计算账），两个都真、 衡量的东西不同。「参数账」不等于「显存账」——推理时显存大头常是 KV 缓存，下一节。

04KV 缓存与上下文窗口#llm-inference-kv

推理的「钱」和「慢」都写在两行账里：O(n²) 的 prefill 与 KV 缓存的显存。 会算这两笔账，就会谈成本——按 token 计费、输出比输入贵、首字慢、并发爆卡，全是它们的 直接映射，不是定价花招。

生成是自回归循环：写 500 字 ≈ 完整跑 700 多次前向——流式输出不是产品特效，是模型本来的 节奏；每个输出 token 都是一轮完整前向，这就是输出单价高于输入的架构原因。两阶段像餐厅： prefill 备菜慢（整段并行、算力密集、O(n²)——50 页文档半天不出字就是它在啃）、 decode 上菜快（逐 token、带宽密集）——TTFT 与 TPOT 分开谈。KV 缓存是空间换 时间：历史 token 的 K/V 算一次存起来反复用（翻会议纪要，不重放录音）——代价是纪要越记 越厚：7B 模型一条 32K 对话的缓存 ≈ 16GB，比 14GB 的模型权重还大；并发 10 条 = 权重十倍量级的显存。窗口长度翻倍：prefill 计算 ×4、KV 显存 ×2。

注意力变体 KV 账速算

按讲义第 4/5 章口径（7B 级 MHA ≈ 0.5MB/token FP16；GQA 8:1 ÷8、MLA ≈ ÷10； 70B 级按 GQA ≈ 0.32MB/token 折算）。教学量级直觉，选型以模型卡实测为准。

标称 1M ≠ 有效 1M：四堵墙

2026 现状：托管商用模型已有 13 家提供 ≥1M 窗口（Gemini 3.1 Pro 2M、Llama 4 Scout 标称 10M），但 RULER/MRCR v2/NoLiMa 等基准显示多事实检索过 200K 普遍掉 30–60 分—— 塞得进 ≠ 记得住。窗口是「训练出来的能力」不是「配置出来的参数」，中间隔四堵墙： ①训练——长程依赖数据天然稀缺（整本书才 100–300K token）+ 平方级训练成本； ②架构——O(n²) 硬撑不可持续，稀疏/线性是有损压缩；③服务——KV 巨量、 HBM 带宽墙、超长 prefill 无缓存则分钟级；④质量——标称 ≠ 有效。最强拉长驱动是 视频（约 300 token/秒，1 小时 ≈ 108 万 token）。主战场已从标称长度竞赛转向 有效长度 × 单位 token 经济性 × 记忆分层。

这也是两条高频答辩的架构论据：「1M 窗口还要 RAG 吗」——要：全量塞入平方级烧钱、 过 200K 有效性衰减、新鲜度与行级权限只能在检索层做（应用视角见 RAG 网页版）；上下文工程为什么非做不可 ——每多塞 1K token，所有后续轮次都为它付费（复利型成本），做法见 Agent 网页版。服务系统怎么管好这笔账 （分页、前缀缓存、容量规划）在 LLM-Inference 网页版——互为前后篇。

05注意力的工程进化#llm-attention-zoo

近十年的注意力改造全部沿三条战线：压 KV 缓存（MQA→GQA→MLA，少记/压缩纪要）、 减计算量（滑窗→稀疏→线性混合，少看但看对地方）、榨硬件（FlashAttention， 数学不变只改搬运）。客户抛出任何新名词，先问：它在打哪座山？

变体打哪座山主要代价谁在用（2026-07 核实）
MHA（基线）—KV 缓存最大教学基准与小模型
GQA 8:1压 KV（↓8×）质量轻微让步生产默认：Llama、Qwen dense 系
MLA压 KV（↓约 10×）需定制推理 kernelDeepSeek 系、Kimi K2、GLM-5
滑窗交错 SWA减计算 O(n·w)远依赖靠层间接力Gemma 3（约 5:1）、GPT-OSS
稀疏 DSA减计算 O(L·k)索引器额外开销DeepSeek V3.2+、GLM-5——已进旗舰生产
线性混合 3:1减计算 O(n) 为主复杂推理精度风险Qwen3.5 转正、Kimi Linear；MiniMax M2 回退全注意力
FlashAttention榨硬件（不改数学）无（纯工程收益）全行业默认集成（FA-4 面向 Blackwell，2026-07 上 PyPI）

几条讲透的机制：GQA 让每 8 个 Q 头共用一组 KV（32 位专家每 8 人共用一套百科全书，够用又 省地方）；MLA 不砍头数、把 K/V 低秩压缩成潜在向量存缓存（把书压成 zip，看时解压——柜子省 得更狠，代价是解压要定制 kernel）；DSA 稀疏像读 500 页标书先扫目录标出 20 页重点再精读； 线性混合的格局一句话——混合线性是效率前沿，全注意力仍是精度前沿（MiniMax 从线性 退回全注意力是行业公开的教训）；FlashAttention 是换了动线设计的厨房——菜谱一个字没改， 出菜速度翻几倍。

价格差从哪来：注意力变体决定 KV 显存 → 决定单卡并发 → 决定每 token 摊薄成本； MoE 决定激活计算量；FA 与引擎决定硬件利用率。「DeepSeek 为什么便宜」三行拆完： MLA（KV ↓10×）+ MoE（激活 37B/总 671B）+ 自研推理工程——便宜是设计出来的，不是补贴 出来的。看懂一家模型的注意力设计，就看懂了它的成本结构：架构表就是价目表的底稿 （市场侧对照见 Model-Landscape 网页版）。

客户问题一句话答案深入看
为什么按 token 计费、输出更贵？自回归：每个输出 token 都是一轮完整前向第 4 节
为什么首字出得慢？Prefill 在啃 O(n²)——备菜慢，上菜快第 4 节

四类失败，四种控制：「模型答错了」不是一种病——知识缺口（上 RAG 与拒答）、推理 失败（拆步骤加验证，换大模型只能治这类的一部分）、上下文失败（管预算与排序引用）、治理 失败（授权隔离人工）。像医院分诊：没分诊就换大模型，相当于头疼直接吃止痛药。

「Transformer 要被 Mamba 取代了吗」三段式：①承认趋势——注意力确实在被稀疏化、 线性化、混合化；②校准现状——混合已成主流实践（Jamba/Nemotron 交错、Qwen3.5 混线性层）， 但全注意力仍是精度基准，旗舰闭源没有一家公开弃用；③收回主动权——客户该关心的是效果、 成本、生态三角，架构是手段不是采购目标。这个三段式适用于一切「X 要取代 Y」。

选型七问（把 8 张 H100 的采购变成一页算术题）：①上下文需求多长、有效窗口有第三方 基准吗 ②目标并发多少（KV/token × 并发 × 长度 = 显存账）③候选是 MHA/GQA/MLA、引擎支持吗 ④是 MoE 吗、总参与激活各多少 ⑤GPU 代次与 kernel 匹配吗（FA-3 对 Hopper、FA-4 对 Blackwell）⑥请求里有多少可复用前缀、能吃到缓存折扣吗 ⑦任务是长文档归纳还是多轮复杂推理 （决定敢不敢用线性混合）。

为什么现在的大模型清一色 Transformer？2026-07-08

不是它最精巧，而是它最能「堆」：全并行训练让数据和算力的投入能持续变成效果， 这是 RNN 做不到的。架构竞争的本质是可扩展性竞争。

参数量 7B、70B 到底意味着什么？2026-07-08

主要是 FFN「知识仓库」和注意力矩阵的规模——参数大头（约 2/3）在 FFN。参数越大记的 知识越多，推理成本同步涨；MoE 就是想解耦「记得多」和「算得贵」。

「参数账 ≠ 显存账」是最常见的误解：推理时显存大头常是 KV 缓存， 不是权重——混淆这两笔账，容量规划必错。

依据　参数账 · 显存账

都有 1M 上下文了，还要 RAG 吗？2026-07-08

要。三条架构论据：全量塞入按 token 计费 + 平方级 prefill 贵得离谱；过 200K 有效性 明显衰减（掉 30–60 分）；知识分钟级更新与行级权限只能在检索层做。

「标称窗口」是参数上限、「有效窗口」是工程参数——这对区分是长窗口 营销话术的解药，四堵墙讲清为什么。

下一步　客户需求若依赖 200K 以上可靠检索，直接引 RAG 网页版 的架构方案；追问友商「有效长度多少、超长怎么计费」。

私有化部署为什么并发上不去？2026-07-08

显存被 KV 缓存吃掉了——一条长对话的缓存能比模型权重还大。解法三管齐下：选 GQA/MLA 架构的模型、用 vLLM 分页管理、给上下文长度设限。

这笔账当场算给客户看比十页材料有说服力：7B 一条 32K 对话 ≈ 16GB 缓存 vs 14GB 权重。

DeepSeek 为什么那么便宜？2026-07-08

架构上两板斧：MLA 把 KV 缓存压到约 1/10（单卡并发大增），MoE 让 671B 参数每 token 只算 37B；再加自研推理工程。便宜是设计出来的，不是补贴出来的。

这套三行拆法对任何一家都适用——把「他们为什么便宜/贵」变成架构 分析，是本册最值钱的售前技能。

模型知识为什么会过时？能打补丁吗？2026-07-08

知识压在 FFN 权重里，截止于训练数据。改权重（微调）成本高且容易误伤别的能力； 主流做法是外挂检索（RAG）——新知识放库里，模型负责理解与组织。
