01什么时候该微调#ft-when

微调改的是模型本身（权重），把「每次都要在提示词里交代的要求」内化成默认习惯。招人视角：预训练模型 = 名校毕业生，微调 = 入职培训，RAG = 发工作手册——绝大多数企业只需要「培训」，不需要「养人」。会劝退的售前，比什么都推微调的售前更容易赢单。

定制光谱是一把梯子，左边一档没榨干之前不动右边：

不知道知识问题上 RAG

不够聪明能力问题换底座

微调能兑现五类价值

小模型顶替大模型 + 砍长提示词。

四条劝退清单

如果……先做别的
知识频繁更新用 RAG
样本攒不够先 few-shot 顶着
需求还在天天变先提示词快速迭代
只是想「更聪明」微调不加智商

成本收益账：一次性成本 = 数据整理 + 训练算力 + 评估验收（数据是大头）；每次调用的收益 = 更短提示词 + 更小模型 + 更低延迟——调用量 × 单次收益 > 一次性成本才立项。微调是「高频窄任务的规模生意」：日均百万次的客服几周回本，日均一千次的内部工具一年省的 token 钱不够付一次标注。

02方法谱系与显存账#ft-methods

三种方法三个比喻：

全参微调是整本书重编——上限高、要集群、毁版风险
LoRA是贴便利贴——冻结底座 + 低秩旁路，可训参数不到 1%，产物几十 MB
QLoRA是压缩影印再贴——4-bit 底座 + 分页优化器，单张消费卡微调大模型
模型规模全参微调LoRA（16-bit 底座）QLoRA（4-bit 底座）
7–8B~110GB（多卡）~18–20GB（单卡）~6–8GB（笔记本级）
13–14B~220GB（多卡）~35GB（单张专业卡）~10–12GB（单卡）

03数据准备#ft-data

微调项目九成的翻车不在训练，而在数据：格式拼错、样本太脏、量级误判。格式即协议： messages 数组 + JSONL 每行一条 + 正确的聊天模板——训练用 A 家模板、推理用 B 家模板，效果「神秘变差」且无报错，是最阴险的一类事故。

结论
量级官方口径 50–10,000 条，窄分类 200–500 条常见效。先 100 条跑一版看验收信号，有信号再加码——数据是逐步投资
质量 > 数量LIMA 用 1,000 条精选样本逼近大流程对齐的效果——能力早在预训练里学会了，微调只是「唤醒」。经验是 100 条脏数据的伤害大于 1,000 条好数据的收益，预算优先花在清洗与精选

不够就造

蒸馏是名师写详解

大模型连推理轨迹一起产出教学数据，小模型对着学——「frontier 效果 + 小模型成本 + 权重可私有」，是讲降本与私有化的主角。

数据治理三问（答不好技术再好也进不了采购）：①隐私与 PII——训练前清洗脱敏是刚性前置；②权属与条款——用第三方模型产出的数据训练要核对其服务条款，方案书写明依据； ③数据出域——托管微调 = 数据交平台，金融政务提前备好 VPC 内训练/本地自建的替代路径。

04训练实操与框架#ft-training

框架功能已趋同（LoRA/QLoRA/全参/DPO/GRPO 四家全支持），差异在工作流——别耗在选型上，一周内跑通第一版比「选最优框架」重要得多。代码只有十来行；复杂度守恒：省下来的代码复杂度，都转移到了数据质量与验收标准上。

框架Stars（2026-07）一句话定位适合谁
Unsloth~54K单卡速度王：快 2×、省 70% 显存单卡/Colab 起步、快速实验
LLaMA-Factory~68KWeb UI 全家桶，零代码全流程想点鼠标训练的团队
Axolotl~11KYAML 配置驱动，FSDP/DeepSpeed 开箱多卡生产、要可复现
TRL~18KHF 官方库：SFT/DPO/GRPO/KTO 原语要底层控制、自己写流程

盯 eval 不盯 train

train loss 是刷题正确率（背题也会高），eval loss 是没见过的模拟考——eval 掉头上翘就是过拟合起点，取上翘前的 checkpoint 交付，别「再训久一点」。

九成事故来自四个坑，按顺序排查

顺序坑症状与解法
①学习率过大loss 震荡 / NaN——降一个数量级
②epoch 过多小数据转十几轮 = 背题，1–3 起步
③模板不匹配最阴险——永远用框架自带模板
④停止符缺失生成停不下来、答案拖垃圾

效果不对先查四坑、再怀疑数据、最后才怀疑方法——顺序反了浪费一周。

成本结构：LoRA 级单卡几小时、云上几十到几百美元——算力是零头，数据与评估的人力是大头，报价拆开给客户看。

框架格局与版本边界核实 2026-07-09/12（PEFT v0.19.1、TRL v1.8.0 KTO 转正），来源 GitHub releases 与框架对比。不可外推边界：版本号是当日快照，微调框架在小版本间也会破坏兼容；方案里锁版本号，别写「用最新版」。

05偏好对齐#ft-alignment

SFT 教「照示范做」，但示范写不出「分寸感」——客服模型格式全对却过度道歉、车轱辘话：你写不出完美示范，却一眼能挑出哪个更好。这正是该上 DPO 的信号。对齐三级： SFT 照做 → DPO 好坏对纠偏 → RFT 对着分数练，按症状逐级升级。

症状处方为什么
格式、语气、话术不对SFT 就够行为示范写得出来，模仿就能解决
会答但分寸差（啰嗦/过度拒答）SFT + DPO完美示范难写，好坏对比容易挑
结果可机器判分（代码/抽取/分类）RFT / GRPO有客观分数，强化比模仿上限高
通用能力不足（推理/数学弱）换底座微调不加智商，别硬调
方法怎么做门槛与用量
DPO直接在偏好对（chosen / rejected）上优化，跳过奖励模型成本接近再跑一次 SFT——「RLHF 很贵」是 2023 年的旧闻。几百到几千对开工
RFT用可编程打分器当奖励（代码跑单元测试、抽取对错）几十到几百条样本就能显著提升窄任务；门槛在「写不写得出靠谱 grader」——判分规则含糊、能被钻空子就退回 DPO / SFT

偏好对三个来源混用：线上 A/B 选择、专家挑选、judge 合成。生产纪律是「先小批人工、再对齐 judge（一致率 ≥80%）、最后放量」。

现实比例：多数企业项目止步于 SFT + 少量 DPO，RFT 是尖刀不是标配。

过度拒答是对齐失败的常见形态

偏好数据里要有「该答就答」的正例，拒答率和有用性两头设门禁。

RFT/GRPO 为 2026 微调新风向，核实 2026-07-09，来源 OpenAI RFT 文档与行业综述；原理（奖励模型/RLHF/RLVR）在 LLM-Training 第 5/6 章。不可外推边界：「新风向」说的是方法关注度上升，不等于已成生产默认；引入按探索型排期，别按成熟件承诺。

06托管服务与上云#ft-cloud

自建还是托管，三问定方向：①数据能不能出域（不能→自建/VPC，这是第一道分水岭） ②有没有 GPU 和会炼丹的人（都没有→托管）③要多深的定制（标准 SFT/DPO→托管够了；换底座/改流程/全参深灌→自建开源）。托管试点 + 自建放量的混合也常见。

07评估与部署#ft-eval-deploy

训练结束不等于项目结束。验收四层：held-out 测试集指标（绝不用训练数据考试）→ 任务硬指标（JSON 合规率/意图准确率）→ LLM judge 评分 → 线上 A/B。顺序不能反：先定验收标准，再开始训练——先有考卷，再补课。

防遗忘是微调的安全带：目标任务涨分、通用能力掉分的偏科风险，四件套防住——LoRA 天然温和、低学习率少 epoch、训练集混通用数据、上线前跑通用回归。上线门禁双达标：目标任务过验收线 + 通用回归不低于基线的 98%；几百题的回归集跑一次几分钟，性价比最高的保险。四类实验失真要防：数据泄漏（近重复没扫，测试分虚高十个点上线现原形——冻结切分哈希）、随机性挑 run（多 seed 报区间不报单点）、judge 偏差（人工校准兜底）、指标替代目标（loss 好业务差）。

部署两形态

形态怎么做适合
合并权重W + BA 合成普通模型单一稳定任务，运维简单
挂 adapter1 个共享底座 + N 套 LoRA，vLLM 多 LoRA 按请求路由多任务 / 多租户——省一个数量级显存、秒级回滚（动态加载有官方安全警告：仅限隔离受信环境）

发布控制五层

迭代闭环才是交付物

上线 → bad case 归因 → 修数据 → 再训 → 回归验收 → 灰度，每圈同一套门禁。一次性交付的微调，半年后一定劣化成新的痛点。

08速查与问答#ft-field-guide

常见误区一句话纠正
「微调 = 让模型学会我们的知识」知识用 RAG（可更新可溯源）；微调管行为，两者常组合
「数据越多越好」质量优先：LIMA 证明千条精选可胜万条粗料
「微调会让模型更聪明」智商来自底座；能力不足换模型，别硬调
「什么模型都能微调」闭源各有开放范围（Claude 仅 Bedrock+Haiku），先查再承诺
「微调一次就完事」是持续闭环：bad case 回流、季度迭代、回归门禁
「微调很贵，要大集群」QLoRA 单卡起步；大头在数据人力，不在卡

30 秒说清：提示词、RAG、微调怎么选？2026-07-09

先分类：模型「不知道」是知识问题上 RAG；「不听话」是行为问题上微调；「不够聪明」换底座。顺序永远是提示词 → RAG → 微调，左边没榨干不动右边。

九成的「要微调」，追问两句会退回提示词或 RAG——梯子原则既省客户的钱，也是方案可信度的来源。

下一步　用决策器过一遍需求；先把四条劝退清单反着问客户一遍。

要多少数据、多少钱、多久见效？2026-07-09

窄任务几百条优质样本、单卡几小时、两周出第一版可验收的模型。大头是数据人力不是算力——先小试点再放量。

官方口径（50–10,000 条、窄分类 200–500 条）客户会拿来对表；「先 100 条看信号」的节奏让报价有依据。

下一步　报价按成本结构拆：数据准备（人天）+ 训练算力（零头）+ 评估验收 + 迭代运营（常被漏报的成败项）。

依据　数据准备 · 成本结构

微调会不会把模型调笨？2026-07-09

有「灾难性遗忘」风险，但可控：LoRA 天然温和 + 保守超参 + 通用回归门禁，掉超过阈值不放行。每个版本都带回归报告。

「学得少也忘得少」是 LoRA 的实证取舍——行为定制它两头都稳；深度灌注才用全参，且必须配防遗忘。

下一步　建几百题的通用回归套件（常识/安全拒答/多轮），双达标才放行——跑一次几分钟。

依据　取舍 · 防遗忘

GPT / Claude / 开源，微调路径分别是什么？2026-07-09

OpenAI：SFT/DPO 调 GPT-4.1 系、RFT 调 o4-mini；Claude：仅 Bedrock 上 Haiku SFT；开源：自由度最大，QLoRA 单卡起步。按事实表作答，不口头加码。

下一步　方案第一步查证当日平台事实；数据不出域的需求直接切 VPC/本地开源路线。

SFT 和 DPO 我们都要做吗？2026-07-09

先 SFT 看验收——八成场景到此为止。上线后如果「会答但分寸不对」（啰嗦、过度拒答），再补一轮 DPO，偏好数据从线上反馈里攒。

对齐已平民化：DPO 成本接近 SFT、RFT 有托管——「RLHF 很贵」是旧地图；但 RFT 是尖刀不是标配，别让客户以为要走全流程。

下一步　按症状→处方决策表定层级；RFT 先问一句「结果能不能机器判分」。

怎么证明微调真的有效？2026-07-09

事前签验收：held-out 指标 + 任务硬指标 + judge 评分 + 通用回归，四项打包成报告交付。绝不拿训练数据自证——这是验收纪律。

四类实验失真（泄漏/挑 run/judge 虚高/指标替代）会让「高分不可信」——先冻结基线与切分再谈提升，报区间不报单点。

下一步　训练开始前把验收单（合规率 ≥99%、准确率 ≥92%、回归不降）签下来——考卷先于补课。
