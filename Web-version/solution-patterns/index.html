01从技术轴到场景轴#sp-what-why

客户不会问「你们 RAG 用什么重排序」，他问「我的客服能不能少雇十个人」——场景轴负责 把后者翻译成前者。工作循环五步：需求发现 → 参考架构 → POC → 报价 → 落地运营； POC 失败回到需求发现重挑场景是常态不是事故。角色像全科医生：先分诊、开检查单， 深度问题转回技术模块（转专科）——价值在业务语言与技术语言之间的「翻译」。

同一块积木在不同场景里角色完全不同——RAG 在客服里是「知识兜底」（防幻觉第一道闸）、 在企业搜索里是「绝对主角」（权限感知检索就是产品本身）、在营销里是「幕后配角」（产品 参数的事实源）。对客永远从场景讲起、技术配角出场：先说「少雇十个人」的漏斗，再说漏斗 里那格是 RAG——顺序反了就成了推销技术。「先做哪个场景」的通用答案：内部知识库或客服 （数据现成、风险可控、ROI 好算），更负责的答法是按数据就绪度 × 容错成本排矩阵。

场景分诊器

客户开口第一句 → 场景定位 + 积木清单 + 成本量级 + 口径提醒。分诊错了 后面全错——宁可多问三个问题再分诊，也别顺着第一句话直接进方案。判据全部来自本页 各节，不新增事实。

02方案共性方法#sp-method

通用参考架构五层：入口（网关：限流/审计/成本治理）→ 编排（Agent/工作流/ 低代码）→ 模型（选型/提示词/微调）→ 数据(知识库/检索/权限) → 保障（评估/护栏/内容安全）。POC 常只建模型 + 数据两层，方案书必须画全五层 ——客户的安全团队和采购看的恰恰是后三层，这也是「POC 成功了为什么上生产还要收钱」 的答案。

POC 三要素（像装修先砌样板墙）：真数据（拒绝演示数据——拿不到真数据本身就是 项目风险信号）、签字的指标（提前书面定义「好」，指标要能自动测）、验收线（低于多少算 失败、高于多少扩大——没有验收线的 POC 会永远「再试试」）。前置一层是一页 intake（把 诉求变成决策合同）：结果/用户/数据/风险四个字段，任何一个没证据就退出——「能停」是 专业不是失败。成本三本账：token 账（输入输出分开算，缓存能砍 50–90%）、并发账 （峰值定限流档位，私有化直接决定卡数）、人力账（提示词维护/知识运营/评估回归—— 最常被漏报的一笔，漏了它报价第二年爆雷）。报价公式：单位成本 × 用量 × 1.3 缓冲 + 人力运营包——先量级后精确。

口径鉴别术（性价比最高的专业度信号）：deflection 挡走率 ≠ resolution 真解决率 （追问「被挡走的用户后来去哪了」）、演示数据 ≠ 真数据（「换我的数据现场跑一遍？」）、 最佳延迟 ≠ P95、起步价 ≠ 满负荷 TCO（「按我的量算三年总账？」）。数据锚点：客服 AI 厂商自报解决率 51–80%，独立口径（Zendesk 企业全量）中位仅 41.2%——30–40 个百分点的 差距是常态。五类高频坑没有一个是技术坑（需求/数据/期望/验收/安全）——AI 项目烂尾， 八成烂在方法和预期管理上。

03智能客服#sp-customer-service

客服 AI 的目标不是替代人工，是漏斗前移：L1 自助扛量（FAQ/RAG 直答）→ L2 AI 深度办事（Agent 查单改地址退款，带审批）→ L3 人工处理复杂/情绪/高危。升级率是健康 指标不是失败指标；升级时带摘要，别让用户再说一遍。对客话术：让人工从「答 FAQ」升级 到「处理值钱对话」。

积木清单：RAG 兜知识（答案必须有出处）+ Agent 办业务（高危操作 AI 草稿 + 人工确认） + 语音链路（有电话渠道才上：SIP 网关接呼叫中心、级联为主可审计、P50 <1.5s、 $0.01–0.17/分钟——深潜在 Multimodal 网页版第 8 节）+ Gateway 入口治理 + Evaluation 永远不是可选件。口径战场是 本场景的灵魂：自报 51–80% vs 独立中位 41.2%——承诺测法不承诺数字：用客户真实 工单建评估集测基线，验收线一起签。成本对标着讲 ROI：文本 $0.01–0.05/会话、语音 $0.01–0.17/分钟 vs 人工 ¥3–8/通——但 ROI 只对「被真解决的那部分」成立，解决率口径 直接决定 ROI 真假。四个高频坑：知识库荒废（运营机制进合同）、人设失控（护栏+红队）、 只测挡走（验收用 resolution + CSAT）、高危无兜底（退款被薅羊毛）。呼叫中心与工单系统 集成是工程量大头，排期如实体现、集成费单列。

04企业知识库与 AI 搜索#sp-knowledge-search

先分形态再报价，两者差一个数量级：部门级知识库问答（圈定文档集、RAG 直答、 低代码平台周级上线，工程量以知识运营为主）vs 全公司 AI 搜索（全司异构源：邮件/IM/ 文档/工单/CRM，连接器 + 权限是主工程，季度级）。客户说「我要个企业知识库」，第一件 事是分清他要哪种。

四段管线：连接器 → 索引（向量+图谱+ACL 元数据）→ 权限感知检索（查询时实时过滤）→ 生成与引用（答案必带出处）。权限是命门：搜索越好、泄密越快——ACL 从源系统继承 不另建一套、查询时实时过滤不是索引时过滤一次、无权限时连「有这份文档」都不暴露 （存在性也是机密）。POC 验收必含越权测试集：无权限账号问 20 条敏感问题一条都不能 漏，安全团队签字才上线。连接器工程最容易被低估：每个源系统都是一个小项目（认证、增量 同步、格式解析、ACL 映射）——报价按源系统个数算。市场锚点 Glean 模式：ARR $300M（2026-05，+89% YoY）、估值 $7.2B、Agent 年化 1 亿+ actions——卖的是「权限感知 知识图谱」不是搜索框，「找得到 + 不泄密」被市场定过价；且搜索正在变成 Agent 的地基： 先找得到，才谈得上会办事。文档质量差怎么办：垃圾进垃圾出，但方案可带知识运营—— 覆盖率仪表盘找高频无答案问题，反向驱动文档补全。

05营销与内容生成#sp-content-gen

三条线成熟度与成本各不同：文案最成熟（$0.001–0.01/条，按吨算——Jasper 模式 风格指南全局生效）、图像是素材工厂（$0.02–0.1/张，迭代是乘数）、视频最 烧钱（$0.5–5/条起步）。按线报价、按线设坑检。

品牌一致性是工程不是运气（像新员工入职培训：不给手册的实习生写不出「像你家」 的文案）四件套：风格注入（风格指南+语气示例进系统提示）、禁用词护栏（违禁表述/竞品词 进输出过滤）、事实源（产品参数走 RAG——纯生成不能保证参数不错，架构上堵死）、品牌 资产库（素材复用，重度需求训定制模型）。商业安全是营销场景的入场券（Firefly 模式）：训练数据授权可溯 + 企业 IP 赔付条款 + 内容凭证标识——法务问「侵权谁负责」的 两层兜底：选商业安全定位的供应商 + 发布前人审留痕，合同权责写清。生成 ≠ 发布： 机器初筛 → 分级人审（对外广告/法务敏感 100%、社媒日常抽检 10–30%、模板化变体机器初筛 即可）→ 发布带留痕 → 效果回流。人审不是不信任 AI，是风险定价：万分之一翻车概率 × 指数级翻车成本 = 必须有人签字。生成技术侧（扩散 vs 自回归、视频四家）在 Multimodal 网页版。

06AI Coding 与研发提效#sp-ai-coding

「三选一」是伪命题——双层格局配组合：IDE 补全/编辑层管单人效率（Copilot/ Cursor，按人头月费）+ agentic 层管整任务委托（Claude Code 类终端 agent，端到端修 bug、按用量计费）。70% 工程师同时用 2–4 个工具；网关统一管预算，工具迭代不伤架构。

2026 市场三个第一（引用永远带口径——数字漫天飞的领域，口径干净本身就是专业度）： Copilot 4.7M 付费用户量第一（+75% YoY）、Cursor $2B ARR 收入第一、Claude Code 46% most-loved 满意度第一（JetBrains 2026-04）；市场 $12.8B、85% 开发者在用。企业三 关注对应三个方案件：代码安全（企业版合同承诺不训练 + 网关脱敏审计 + 私有化选项）、 私有库上下文（代码库索引/RAG/MCP 接内部文档——效果差异的主要来源）、ROI 度量（别只 看补全接受率，看 PR 周期、缺陷率、DORA 指标——用自己的基线不用厂商宣传，行业口径 20–50% 但方差大）。落地路径：网关接入 → 2 团队 × 8 周试点 → 基线对比 → 分层推广 → 持续运营——只买 license 不做 enablement，效果差 3 倍；评审门禁不降级 + 新人配 AI 结对规范：AI 提高产量，评审守住质量，两者不能互换。

市场数字核实 2026-07-10，来源 ideaplan / JetBrains 2026 调查。

07数字人#sp-digital-human

先问客户要「内容工厂」还是「交互服务」——差一个量级的是架构和成本结构： 离线生成（文本→口播视频，渲染一次卖多次、按分钟计费——HeyGen Avatar V 级 15 秒素材 建分身、10 分钟身份不漂移）vs 实时驱动（WebRTC 双向对话，每秒都在算、按并发计费）。

实时数字人 = 语音 Agent + 一张会动的脸：RTC 收流 → 语音链路 → 大脑（LLM+RAG 人设知识）→ 口型与渲染（在语音延迟预算上再加 100–200ms，目标仍是 1.5s 内开口—— 口型不同步比慢更穿帮）→ 推流。三类场景分格子报价（按时长 × 互动深度）：口播/ 培训视频最成熟（已过「不细看分不清」线，改稿零成本）、直播带货是量的生意（7×24，中国 特色主场）、实时交互是体验的生意（最烧钱，客服的「有脸版」）。合规三问不过项目直接 停：形象授权（真人分身书面授权、范围期限撤回写清、离职员工分身要能下线）、深度 合成标识（显著标识义务，直播/视频角标常态化）、声音克隆授权（声音与肖像同等保护、 名人音色是高压线）——答得好，合规就是竞争力。语音链路深潜在 Multimodal 网页版第 8 节， 标识义务在 Security 网页版第 8 节。

08ChatBI 与数据分析#sp-chatbi

三层需求先分清（混着报价必翻车）：L1 查数（「上月华东区收入多少」，两三周能上）→ L2 归因（「为什么环比降了」，依赖语义层厚度）→ L3 报告（Agent 工程）。准确率的 真相在语义层不在模型：裸 Text-to-SQL 真实企业库实测仅 10–21%（学术基准五六成， 给客户一定讲前者），绑定语义模型后 51%→90%+（Snowflake 内部 150 题），dbt 语义层 基准上限 98–100%。

把「你们准确率多少」转化为「语义层建到多厚」，口径治理问题就赢了。三道闸控住 「算错数」风险（算错一个数比答错一段话严重得多）：高频指标只走语义层确定性生成、 SQL 只读执行 + 行级权限过滤、答案附生成的 SQL 与口径出处可复核——裸 SQL 只留给内部 临时分析。产品格局：垂直一体派（Cortex Analyst YAML 语义模型 / Databricks Genie / Fabric Copilot——自家平台内最准最顺，代价是数据进平台）vs 独立派（ThoughtSpot / dbt 语义层 / Cube——跨平台不搬数据，集成自己做）；选型第一问不是「哪家准」而是「客户 数据在哪」。成本结构：大头是语义层建设与运营的人力不是模型调用——起步 10–20 个 高频指标定口径，两三周出 POC。真难点在甲方组织内部：各部门「收入」定义打架是 组织问题不是技术问题——把口径评审会开起来，比任何模型调优都关键。对 BI 团队讲：从 人肉写报表升级为维护语义层口径，是换岗不是替代。技术底座在 RAG 网页版结构化数据节。

09会议与办公助手#sp-meeting

两大形态、三层价值，按层递进卖别一次全卖：笔记器（转写+摘要+行动项，单会价值、按 席位订阅——2026 已是「有没有」而非「要不要」）→ 会议记忆库（最被低估的一层： 单会摘要是快消品，可检索的组织记忆才是资产——「上次谁承诺了什么」秒级检索）→ agentic 平台（会后自动建任务/发跟进/更新 CRM，要审批边界）。

五段链路：采集（bot 入会最通用但有「bot 疲劳」/ 桌面采集免打扰 / 平台原生最顺——按 客户会议栈选）→ ASR + 说话人分离 → 摘要行动项 → 记忆库（权限感知检索：谁能搜到谁的 会）→ 办公流集成。市场格局：原生派（Zoom AI Companion 内置零安装 / Teams Copilot M365 +$30/人/月 / 飞书妙记）vs 独立派（Otter / Fireflies / Granola bot-free 桌面采集， 2026-03 融资 $125M）——企业十有八九已有会议平台，原生够用就别引入新工具；跨 平台记忆库/深度集成才是独立方案的入场券。「准确率」分三层，混谈必翻车：转写 WER（中文口音/专名是重灾区）、摘要忠实度（有没有编造）、行动项召回（漏了「老板 随口交办」才是客诉源——漏项通常比编造多）。合规按「企业搜索」标准做不是按「效率 工具」标准（会议数据是企业最敏感数据之一）：录音知情同意、记忆库 ACL 与保留期、纪要 显著标注 AI 生成 + 重要决议人工确认——AI 纪要是草稿，决议以人工确认版为准。

格局核实 2026-07-11，来源 read.ai / zapier / 2026 横评。

依据　口径战场

员工会不会搜到不该看的东西？2026-07-10

这是方案的第一设计约束：权限从源系统继承、查询时实时过滤、无权限连存在性都不 暴露。POC 验收带专门的越权测试集，安全团队签字才上线。

搜索越好、泄密越快——权限缓存过期、索引时过滤一次、另建权限 都会漏；权限必须跟着「查询这一刻的身份」走。

下一步　权限不是「搜索的一个功能」，是「搜索能不能上线」 的开关——安全评审主答卷提前准备。

依据　权限命门

生成的内容侵权了算谁的？2026-07-10

两层兜底：选商业安全定位的模型（训练数据可溯源、企业 IP 赔付条款），加发布前 人审留痕。合同权责写清——这是方案区别于「随便接个 API」的地方。

对外内容的翻车成本是指数级的——商业安全不是加分项，是营销场景 的入场券。

下一步　人审按风险分级：对外广告 100%、社媒抽检 10–30%、 模板变体机器初筛——风险定价而非不信任。

依据　商业安全

AI Coding 该买 Copilot 还是 Cursor 还是 Claude Code？2026-07-10

不是三选一：主流是 IDE 层选一个 + agent 层选一个的组合，网关统一管预算。按团队 画像配比——工具会迭代，接入架构不变。

双层价值不同：补全管单人效率（按人头），agentic 管整任务委托 （按用量）——70% 工程师同时用 2–4 个工具。

下一步　2 团队 × 8 周试点出自己的数字（PR 周期/缺陷率/ DORA）；enablement 决定效果差 3 倍，写进方案。

依据　双层格局

数字人会不会有法律风险？2026-07-10

三条红线管住就可控：真人形象与声音书面授权（含撤回机制）、AI 生成显著标识、 内容审核照常跑。合规是默认件不是选配。

三问（形象哪来的、标不标识、授权链全不全）不过项目直接停—— 答得好，合规就是竞争力。

下一步　先分清内容工厂还是交互服务——差一个量级的是 架构和成本结构，不是价格策略。

依据　合规红线

ChatBI 准确率能到多少？算错数怎么办？2026-07-11

取决于语义层：裸 Text-to-SQL 真实企业库只有一两成，绑定语义模型后能到 90%+。 先建 10–20 个高频指标口径，用验收集实测。算错数靠三道闸：高频走语义层确定性 生成、SQL 只读+行级过滤、答案附 SQL 与口径出处。

把准确率问题变成治理问题是本场景的胜负手——真难点是各部门口径 打架（组织问题），售前提前做预期管理。

下一步　选型第一问「数据在哪」：已集中就顺势用原生问数， 多源分散用独立语义层——不是「哪家模型准」。

依据　语义层跃迁

会议助手和 Teams/Zoom 自带的 AI 比，强在哪？2026-07-11

原生助手管单会够用。差异在三处：跨平台统一的会议记忆库、与业务系统的行动闭环 （纪要直接变任务/CRM）、私有化选项。只要单会纪要，直说用原生的就够。

「直说用原生的就够」本身是信任建设；独立方案的入场券是差异化 需求，不是功能对标。

下一步　验收分三层设指标：转写 WER、摘要忠实度、行动项 召回——漏了「老板随口交办」才是客诉源。

依据　两形态三层价值
