# 网页版首页：模块按层级 + 链接图 · 设计（2026-07-23）

> 状态：**已批准**（用户 2026-07-23「Web 首页把所有模块按层级列出来，像 PPT 那样；
> 打造一个链接图把知识点串联，而不是一张表格」）。改的是 `Web-version/index.html` 的
> 「关系网」节。

## 一、现状与要改的

网页版首页（`Web-version/index.html`）现在的「关系网」是一张 `table.net`：19 行，每行列出
一个模块的出边/入边。两个问题正对上用户的话：①它不按层级、看不出全库结构（模块按层级的
卡片矩阵只在根 README，网页首页没有）；②它是表格，不是「图」。

## 二、方案：一个分层链接图（SVG，生成式，Portable）

**一个组件同时满足两个诉求**：节点＝19 个模块，按 KB-CONFIG 的 7 层从上到下排成 7 条带
（就是 PPT 总览那套层级），边＝各模块清单「串联出边」里的跨模块关系。

- **节点**：圆角胶囊 = 层色字标（favicon 位，复用 `hue-0..6`）+ 模块名，整块是链到该册
  网页版的 `<a>`。
- **层带**：左侧层名 + 层色点，带底浅层色（`--hue-bg`）。
- **边**：无向去重（A→B 与 B→A 合一），三次贝塞尔曲线；跨层走竖向 S 曲线，同层走下凸弧。
  **默认极淡**（opacity ~.12）——看得出这是张连通的网，但不糊成一团。
- **交互（site.js 增强）**：悬停/聚焦一个模块 → 它的边和邻居点高亮、其余变淡；点击进该册。
- **布局**：build.py 用 4 轮重心排序（barycenter）给每层内节点定序，减少交叉；确定性、无随机。
- **响应式**：SVG 包在 `overflow-x:auto` 容器里，窄屏横向滚动（页面本身绝不横滚，守 §三·五）。
- **无 JS / 可访问**：SVG 静态即完整可读（节点都是链接、边带 `<title>`）；再留一个
  `<details>「换成文字表看」`，里面就是原来那张 `table.net`——既保留无 JS 阅读路径，
  又让 `table.net` 类不失联（`check_css_classes` 不会报）。

## 三、为什么不违反「同一内容只铺一处」

`web-design-system §三` 定：知识地图（带简介的卡片矩阵）只住根 README，网页首页是全库视图
不复刻地图。本方案**不是复刻那张卡片矩阵**——它是「关系网」这一节的形态升级（表 → 图），
关系网本就是网页首页该有的、纸面做不到的东西（§六·六）。按层排布只是让这张关系图更好读，
不是把 README 的浏览入口搬过来：README 的卡片给「进哪一册」（带两行简介），这里的图给
「谁连着谁」（带边）。两者用途不同，与现在「地图在 README、关系网在网页首页」的分工一致。

## 四、真源与生成

边与层级的真源仍是各模块 `MANIFEST.md`（串联出边 / 所在层），图由 `build.py` 从 data 生成、
注入 `NET:BEGIN/END`，`build.py --check` 守幂等——不手写第二份（§五·五）。改 `kb.css` 加
`.kgraph` 组件族后跑 `bump_style_version.py --apply`。

## 五、验收
八道门禁全绿（含 `check_css_classes` 认识新类、`build --check` 幂等）；亮/暗 × 桌面/窄屏
目检图不糊、悬停高亮可用、窄屏只在容器内横滚；file:// 冷检查无外链。

## 六、明确不做
- 不画 161 章的章级图（节点太多必成毛球）——章级串联已由每章末「串联条」承担。
- 不引任何图布局库（D3 等）——Portable 禁外链，布局在 build.py 里算好、出静态 SVG。
- 不动根 README 的知识地图卡片矩阵。
