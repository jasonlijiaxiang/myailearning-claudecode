# 链接图 · 点选聚焦动效 · 设计（2026-07-23）

> 状态：**已批准**（用户 2026-07-23「想做成动态的，选中哪个就移到中间，动态联系其他模块」）。
> 改 `Web-version/index.html` 的知识地图链接图交互。

## 一、目标交互（ego 网络探索）

- **总览态（默认）**：19 模块按七层排布，彩色连线（现状）。这也是无 JS 的静态形态。
- **点一个模块 → 聚焦态**：该模块**滑到画面中央**，它关联的模块**呈环形排到它周围**，
  连线变成从中心射出的高亮曲线；其余模块淡出、层带压暗。
- **点环上的模块** → 以它为新中心重排（可一路串下去）。
- **点中心模块** → 打开那一册（导航）。
- **返回全景 / Esc** → 动画退回总览态。

## 二、实现（Portable：纯静态 SVG + 原生 JS，无图库）

- **节点改成 `transform="translate(cx,cy)"` 的 `<a>`**，子元素相对中心画——这样挪节点＝改一个
  translate，JS 用 `requestAnimationFrame` 插值（easeInOutCubic）平滑移动，19 节点/帧无压力。
- **中心的连线每帧随节点重算**（跟手滑向中心）；其余连线淡出。总览态的彩色边由服务端静态
  渲染（无 JS 兜底），JS 进/出聚焦时接管。
- **环形布局**：中心 (W/2, H·0.47)，邻居按角度均分在椭圆（rx≈430, ry≈H·0.34）上。
- **淡出的模块留在原位、只把不透明度降到 0**（不乱飞），退出时原地淡回。
- **层带**在聚焦态压暗（opacity .12），让中心网络凸显。
- 无 JS：节点是 `<a href>`，点击直接进该册；静态分层图完整可读（可访问性底线）。
- 状态类（focus/self/on/off/faded）走 `classList`，登记进 `check_css_classes` 白名单。

## 三、明确不做
- 不引 D3/力导向库（Portable 禁外链；布局与动画自己算）。
- 不做物理弹性/碰撞（确定性环形布局即可，稳定可复现）。
- 章级（161 章）不进图——只在模块级做 ego 网络。
