# 追加：v4.1 索引拆分（同日）

- **提议**：用户指出现在的 README 通篇是 PPT 面的内容，它算 PPT 面的总览；根目录该单独出一个总纲。
- **判据**：一块内容换到另一面就不成立，它就是面特有的，不该住根总纲。与 v4.0 把共享规则
  抽进 `shared/` 同源。

## 拆法

| | 根 `README.html`（总纲） | `PPT-version/README.html`（PPT 面总览） |
| --- | --- | --- |
| 装 | 知识地图（25 张卡片）、每日取用（`_prep/` 三件套）、两面入口卡、库目录约定 | 模块列表表格（19 行，合计 1732 页）、预留扩充位、本面目录约定 |
| 链接 | 卡片 → `./PPT-version/<模块>/`；两面入口 → `./PPT-version/README.html`、`./Web-version/` | 表格 → `./<模块>/`；返回 → `../README.html` |

模块 README 的返回链**未动**（仍 `../../README.html` 指根总纲——"返回知识库首页"要回的是库的家，
不是某一面的清单页）；`_prep/` 三件套未动（链的模块与根总纲都没挪）。旧 README 已落档
`_maintenance/history/2026-07-20-README.html`。

顺带修正：原 README「目录约定」还写着每个模块含 `ebooks/`——迁移后已不成立，两页的目录约定
均按 v4.0 布局重写。

## 抓到一个已经发生的漏检

给 `check_page_ledger.py` 搬账面位置时，把三处「匹配不到就静默跳过」改成报 issue，当场抓出：
**v4.0 号称的"账实脚本双布局兼容"只做了一半**——顶层 README 的行匹配加了 `(?:PPT-version/)?`
前缀，但 `_prep/` 全库一页纸的模板漏了。本次迁移把 `_prep` 链接改成 `../PPT-version/<模块>/`
之后，一页纸这一账从此失配、被静默跳过，**上午那次"19 个模块四账一致"的绿灯实际只查了三账**。

已补前缀，并逐账打印比对验证四账真读到（RAG 128 / MCP 74 / Security 89 / AI-Ops 71，
一页纸账与面总览账两两相符）。

**教训**：静默跳过是比误报危险得多的失效模式——门禁匹配不到目标时若不作声，绿灯的含义会从
"查过且通过"悄悄退化成"没查"，外部看不出区别。凡"找不到就跳过"的分支一律改成报出来。
