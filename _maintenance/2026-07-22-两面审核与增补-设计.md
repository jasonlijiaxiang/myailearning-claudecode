## 一、PPT 面：实战包 55 → 91 题

### 1.1 为什么加：三条硬证据，不是「感觉少」

1. **违反已有纪律**：`prep-rules.md`「A 类收尾口径回刷清单」第 3 条写明「新模块对练页至少贡献
   1–2 题……零覆盖必须是显式决定，不能是遗忘」。AI-Gateway 与 AI-Infra-Platform 两册各 8 章
   全零，`_prep/MANIFEST.md` 里没有任何豁免登记 —— 这是遗忘，不是决定。
2. **07-17 增的四章无人接**：中国监管合规（12 页）、国内平台格局（11 页）、视频生成（10 页）、
   Sub-agent 编排（12 页），共 45 页新内容，实战包 0 题。其中中国监管合规是本画像
   （国内 AI 平台售前）**答错直接踩线**的一章。
3. **题池远大于用量**：181 道对练题里只有约 55 道被提炼上来，且分布严重偏斜
   （Model-Landscape 一册占 12 题，Multimodal 十章占 1 题）。

### 1.2 立题判据（防「为了加而加」的闸门）

复用 `web-rules.md` §七的立题判据，加两条实战包特有的。一道题**四条全过**才进：

1. **有源**：出自模块对练页/框架深潜页/生产验收页 —— 不自创事实（prep-rules 铁律）；
2. **会改变一个决定**：技术方案 / 报价 / 风险判断 / 验收方法 / 下一步动作。答完客户「哦」
   一声就结束的，不收；
3. **15–30 秒答得完，且答案里有一个可复述的判据**：一条线、一个比例、一个顺序、或一个反问。
   答案只能是「视情况而定」的，不收；
4. **不与已有题同句**：如果它能被现有某题的一句话覆盖，改成扩写那道题，不新开一题。

**已按此判据拒收的（写进设计，避免以后反复讨论）**：

| --- | --- |
| AI Coding 该买 Copilot / Cursor / Claude Code | 产品推荐，半年即过期，且不改变「我们交付什么」 |
| vLLM 和 SGLang 选哪个 | 深潜题，30 秒答不完；留模块页 |

| 现在下单还是等下一代卡 | 判断会随发布节奏翻篇，且不由我们答 |

## 二、Web 面：站级三件 + 证据加深

### 2.1 定位：19 册已封版，加深只能在站级和呈现层

`web-rules.md` §六·六定死了闸门：只有事实级 / 缺口级 / 结构级三类可回流 PPT，
呈现形态、信息节奏、交互件、措辞打磨**一律留在网页版**。八批收官时 19 册全部
「事实级 0、缺口级 0」= 已收敛封版。所以本轮网页改进**全部落在闸门内侧**：
站级聚合、呈现层、交互层。这既是纪律，也正好是 §六·六「优先做纸面做不到的事」指的方向。

### 2.2 W1 · 全库问答库（最高价值，先做）

**问题**：181 道模块追问锁在 19 个页面里，55 道实战包在 `_prep/` 另一张静态页，
两处同源同性质、互不可见、互不可搜、**都没有稳定锚点**（`<details>` 一个 id 都没有）。
客户问一句话，你没有任何地方能按这句话去搜。

**做法**（`Web-version/qa/index.html`）：
- **它是索引，不是副本**——只放题目 + 模块 + 加入日期 + 深链，答案留在原处。
  这一条是防双源漂移的关键：答案文本只有一份，改答案不用改两个地方。
- 三种切法：按模块 / 按维度 / 按 ⏳ 时效；顶部输入框实时过滤（本地、零依赖、file:// 可跑）。
- 每题稳定锚点 `#q-<模块>-<序号>`，可复制深链发同事。
- 生成方式：扩展 `build.py`，从各模块页 `<summary>` 抽题写进 `data.js`，
  并**幂等回写 `<details id="...">`**。零手工双写，符合 §一「账本唯一」。

**连带收益**：首页搜索索引从 398 条涨到约 670 条（+181 道问答 +91 道实战题）。
这是「可检索」这项从「搜得到章」升级成「搜得到问题」。

### 2.3 W2 · 全库术语表（`web-rules` §九点名的通用底座，一直没做）

- 19 册已按 v3.13 语言策略回刷过「缩写，英文全称」，素材是现成的。
- `Web-version/glossary/`：术语 → 英文全称 → **一句话释义直接截自源章节正文**（不重写，
  防自创）→ 主归属模块章节深链。
- 这是 §五「同一概念只有一个主要归属模块」那张落位表的读者面：术语表本身就在展示
  「这个词该去哪一册看」。

### 2.5 W4 · 证据加深：五要素回刷 17 册

`web-rules` §七要求「定量事实绑定五要素——数据集或工作负载、模型或产品版本、条件、日期、
**不可外推边界**，比 PPT 面更严，网页面先行执行」。实测：
- 「核实 YYYY-MM-DD」标注 73 处（账本登记 218 条事实）；
- **「不可外推边界」只有 MCP 5 处、Agent 1 处**，其余 17 册 0 处。

也就是说这条规则只在样板册 MCP 上真执行了。这是本轮网页版**唯一的真·知识密度缺口**——
不是缺内容，是缺「这个数字在什么条件下不成立」这一句。它恰好也是对客最值钱的一句
（客户拿你的数字去别的场景，翻车算你的）。

做法：17 册逐册把正文里的定量结论补上边界句与核实日期，**不新增事实**，只把账本里已有的
条件写到读者面前。收尾把 `不可外推边界` 覆盖率并进 `check_css_classes` 同级的门禁或巡检项。
