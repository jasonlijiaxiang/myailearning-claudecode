# 库级产物规则（D 类任务 + A 类收尾回刷）

D 类任务和 A 类收尾（流程第 5 步）用这份。**库级产物**指不属于任何主题模块、面向"每日取用"
的横向成品，统一放在库根目录 `_prep/`。它们是全库内容的**再加工**：模块讲义是仓库，库级产物
是随身弹药——出门前 30 分钟翻的是它，不是 1600 页讲义。

## 三件套（默认形态）

| 产物 | 是什么 | 素材从哪来 |
| --- | --- | --- |
| 实战包 | 高频交锋题集（默认口径：把每次对客交锋当一场面试），每题给 15–30 秒电梯版答案 + 「深潜→」源指针 | 各模块讲义的对练页 |
| 学习路径 | 几条按场景取用的阅读路线（如：七天通读 / 出门前 30 分钟 / 交锋前夜） | 全库章节结构 + 各模块速查章 |
| 全库一页纸 | 书架总览（层 × 模块 × 页数实测）+ 串联主动脉 + 保鲜看板（已知翻篇日集中可见） | 顶层索引 + 各模块 MANIFEST |

三件套是**默认**不是铁律：画像非对客型的库，实战包改成对应提问者（面试官/评审）的题集；
库主人另有横向取用需求（术语总表、跨模块问答库）也归 D 类，同样适用本文件全部纪律。
成品用 HTML（门户面约定，色板见 library-rules），文件名用日常词、不带内部工作用语。
三件套的一切对外文案执行**对外文字通用纪律**（反 AI 腔、术语、语序）——唯一出处在
`writing-rules.md`，生成时执行、收尾快速自查，此处不复写。

## 生产纪律：不自创事实 + 引用映射（防双源漂移的关键）

库级产物**不自创任何事实**——所有带数字、带版本号、带日期的内容都必须引用自源模块
（源模块的 MANIFEST / 联网核实笔记是唯一事实源）。为什么：同一事实写两处，刷新时必漏一处，
半年后两处口径打架，读者不知道信哪个。所以：

- `_prep/MANIFEST.md` 的时效性机制**不重复登记事实本身**，而是登记**引用映射表**：
  「源模块（事实源头）→ 实战包引用题号 → 一页纸引用处 → 核实日期（随源模块）」。
- 巡检不联网核查 `_prep`，只按映射检查：某源模块的事实在巡检中被标记"已过时/已刷新"，
  就同步修订库级产物的对应题目与看板行。巡检报告的"建议动作"里包含"_prep 同步"一项。
- 含会过期数字的题目加时效标注（如 ⏳ + 口径日期），提醒读者引用前先核日期。

## D 类触发与执行

用户点名库级产物（"更新实战包""一页纸该刷了""建个学习路径"）即走 D 类。执行要点：

1. **先读源，再动产物**：更新实战包先读涉及模块的对练页与 MANIFEST；更新一页纸先实测页数
   （放映序口径）。不从记忆里写。
2. **留痕同模块规范**：改写前旧版存 `_prep/raw-data/history/`（`YYYY-MM-DD-<原文件名>`，
   重名保护——存在即顺延后缀，禁止静默覆盖）；本次输入存 `_prep/raw-data/`。
3. **收尾更新 `_prep/MANIFEST.md`**（章节清单、引用映射、最后更新、产出 skill 版本），
   并检查三件套之间口径互洽（模块数、层数、题数在三份文件里必须一致）。

## A 类新增模块后的口径回刷清单（流程第 5 步点名动作）

本清单是 **A 类任务的切片**——展开的是「全库口径」与「时效性事实」两行账；全库派生账的
账目全集与真源定义见 `library-rules.md`「派生账登记表」（v3.13）。

新增一个模块会让库级产物的"全库口径"过期。A 类收尾时逐项检查，**当场回刷或明确登记待办
报告用户，不许静默跳过**：

1. **数字口径**：模块数、层数、总页数——三件套正文与 `_prep/MANIFEST.md` 里所有出现处；
2. **一页纸书架表**：新模块加行（层、页数、一句话）；新设层则加层行；
3. **实战包题源覆盖**：新模块对练页至少贡献 1–2 题进对应维度（题数口径同步改），
   或明确登记"暂无高频交锋题，待实战反馈"——零覆盖必须是显式决定，不能是遗忘；
4. **学习路径**：新模块涉及的路线行补上（不涉及就不硬塞）；
5. **引用映射表**：新模块若贡献了带日期事实，加一行映射。

历史教训：库从 18 长到 19 个模块后，实战包正文仍写"18 个模块"、新模块在"每日取用"层
零覆盖——没有这份清单时，局部同步全靠随手想起。

## 初建时机

库级产物是**后期件**：模块 ≥5 个、对练页有了积累再建才有意义；初始化新库时不建，
在顶层 README 留一句"模块攒够后可加实战包/一页纸"即可。首次建 D 类产物按本文件
三件套形态起步，挂到顶层 README 最上方"每日取用"卡。
