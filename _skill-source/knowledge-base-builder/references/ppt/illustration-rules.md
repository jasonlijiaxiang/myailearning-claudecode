# 讲义配图（信息图）规则

给已有讲义 PPT 补信息图，让"文字为主"的深度页变成一眼看懂的图。属 **B 类改成品**：改任何成品之前先存 history、收尾走一致性与 audit，改成品的字节级安全机制（zip 级最小手术、禁 round-trip、theme/notesSlide 共享、OOXML 顺序、app.xml 同步）全部见 `ppt-design-system.md`，本文件只讲**配图特有**的判据、数量、工作流、绘图与踩坑。

## 何时触发
用户说"给讲义配图 / 配信息图 / 加几张图 / 图太少 / 文字太多 / 这册图不够 / 继续配图"之类 → 走本规则。可对单册、也可"全库都配"。

## 一、配哪些页（判据，别重犯）
- **可靠判据**：「概念 / 类比 / 案例 / 小结」四卡结构的**纯文字深度页**，且**做成图明显更好懂**——能落成流程 / 对比 / 结构 / 关系 / 状态机 / 循环 / 时间轴 / 比例 / 图表的那种。判据只有一句：**"这页做成图，是不是明显比读文字更清楚"**，是就配。
- **跳过**：① 已有序号圆①②③或文本"→"流程的页（已是流程图，别重复配）；② 含表格的页（`<a:tbl>` graphicFrame——文字 dump 抓不到，页面圆角卡数为 0 常是信号，拿不准看渲染）——除非做成「对比卡 / 阶梯 / 比例条 / 决策树」比原表格明显更直观才配；③ **角色页无增益**：对练（客户会怎么问）、导览、速查、本章小结、学习目标、动手做、术语对照、串联、生产验收——这些配图没意义。
- **反例**：不要按标题机械配图；不要用 OOXML 统计（"连接线 cxnSp=0 → 缺流程图"）武断下判——序号圆 + 卡片 + "→"字符本身就是流程图。

## 一·五、替代型 vs 补充型（v3.8 铁律，配图前先定）
配信息图有两种意图，动手前必须想清楚，否则会留下重复页（v3.7 的系统性事故：全库 62 页源文字残留）：
- **替代型**：信息图版把整页内容图解化，读者只需看图、原文字页变冗余 → 嫁接信息图后**必须删掉源文字页**（`kb_delete.kb_delete_pages`）。判据：信息图版**完整覆盖**源页信息（AI-Ops 41/42、绝大多数四卡深度页属此）。
- **补充型**：信息图只图解某个局部、或提供新视角，源文字页仍有独立信息 → **保留两页**（如 Prompt-Engineering「是什么/为什么」那种，配图前本就两页）。
- **拿不准按替代型**——售前讲义忌同一主题连着两页高度重叠；真要两页时第二页得有明显增量。
- **收尾自检**：配完跑 `audit` 检查项 15（相邻同题重复页对），逐对用「配图前基线计数」定夺：该主题配图前 1 页→现 2 页 = 替代型残留该删（删前页留后页）；配图前本就 2 页 = 补充型保留。判源页最硬的证据：前页与配图前基线原页 4-gram 相似度=1.00。删源页后本册总页数变了 → 一并按第八节回刷四面页数账。

## 二、数量由内容深度驱动，不设配额（铁律）
- **别卡固定张数**。厚模块（80–100 页、深技术 / 富售前）该配 8–9 张；精干模块 6–7 张；最基础的大册可到 10+。让每册图数跟着内容厚度走、**不要每册等长**。
- **历史教训（用户两次纠正）**：第一轮每册卡 4 张、推广时又齐刷刷卡 5 张——都是变相配额。用户原话："没限制加图数量，只是不用为了加而加；一个模块需要多点图就多配、不强求。""后面都是 5 太巧合了。" 正确做法：把整册**所有真正该配的页**都配上，多寡由页面决定，不由配额决定。

## 三、工作流（一张图约 20 分钟的循环）
1. **扫候选**：按**放映顺序**（`presentation.xml` 的 `sldIdLst` → `.rels` → `slideN.xml`；**文件名序 ≠ 放映序**，插过页的册尤其）列每页的 字符数 / 序号圆数 / `cxnSp` 数 / 圆角卡数 / "→"字符数，挑四卡纯文字深度页。
2. **取源页文字**：按放映序 dump 目标页每个形状的文本，确认它当前是纯文字、无表格，再配。
3. **写 `<模块>_figs.py`**：`import` `kb_draw` 的设计系统 helper + 共享的 `line`/`dot`；每个 `draw_xxx(slide)` 画一页。绘图规范见第四节。
4. **渲染目检**：`soffice --headless --convert-to pdf --outdir . x.pptx` 后逐页 `Read` PDF（**PNG 只导出首页，必须走 PDF 分页**）；看溢出 / 重叠 / 字形 / 留白。有问题改脚本重渲。
5. **嫁接**：`kb_insert.insert_figures(src, dst, [(放映页号, draw_fn, 标题), …])` —— zip 级把新页插在源页**之后**（`after` 参数=放映页号，内部按 `-after` 降序插不串位、自动同步 app.xml）。**原件先 `cp` 存** `raw-data/history/YYYY-MM-DD-<原名>`（同日多轮用 `-b`/`-c`/`-d` 或 `-配图前` 后缀，**禁 cp 覆盖同名**）。
6. **audit PASS 才正式化**：`python3 scripts/audit_pptx.py <新档>`；再渲染抽验衔接（每张图紧跟其源页）+ 核验 app.xml 一致（多对册见第六节）→ `mv` 替换主文件。**本册总页数一旦变了，按第八节回刷四面页数账**（配图必留的收尾，别漏）。
7. **收尾请用户双击抽验一册**（触碰成品的标准动作，见 ppt-design-system）。

## 四、绘图规范（kb_draw 设计系统）
- 用 `kb_draw` 的令牌与 helper（`bg/tb/sr/para/eyebrow/title/footer/node/arrow/band/label` + 蓝青橙绿令牌），视觉与全库讲义**同源**（设计语言见 `ppt-design-system.md`）。
- **node 排版**：中文一行 / 英文一行 / 说明一行（英文别跟中文挤一行折断）。
- **字号**：正文与节点说明行 **≥11pt**（避免 audit「表格外 <11pt 占比 >40%」告警）；网格图 / 密集卡片图的小标签是**合法注释**（类比图表轴标），先把节点说明提到 11pt，仍超再判是否真偏小。**绝不用 <8pt 空行占位**（触发字号硬门禁）——要留白就用定位文本框，别写小字空 `para`。
- **小画布册插页字号预抬档（v3.15，行为级）**：`kb_insert` 生成页按大画布（13.333×7.5）绘制，小画布 9 册（AI-GW / AIC / AIP / Eval / FT / LLM-Inf / LLM-Train / PE / Security，10×5.625）靠 `fix_canvas_scale` ×0.75 事后缩放——这条路径会把常规 11–12pt 正文压到约 8.6pt，**跌破自读档 11pt 契约线**（audit 对此仅告警不拦截；07-14 配图批次属存量抢救被容忍，新增页不得沿用）。给小画布册插页而仍在大画布绘制时：**正文 ≥15pt、标注 ≥13.5pt**（×0.75 后 ≥11.25pt）。已实证：2026-07-20 LLM-Inference 第 2 章 3 页插页按此执行，零新增小字告警。根治路径是给 `kb_insert` 加目标画布参数、按目标册 sldSz 直接绘制（未实施，留待需要时做）；「插页前先读目标册 sldSz」的铁律见 `ppt-design-system.md`（v3.12）。
- **字形风险**：`✓ ✗` / emoji / 生僻符号在 Calibri/Cambria + LibreOffice 下可能渲染成方框（且不稳定）——**用纯文字 + 红绿色**区分，别依赖 glyph。
- **换行**：`node` 的 zh/desc 参数、以及单个 run 里的 `\n` **不生效**（渲染成连写或坏字）——要换行用多个 `para` 或多个文本框。
- **语义色**：状态机的 error 态 / 安全威胁用红（自定义 `REDINK/REDBG`），成功 / 达标用 `GREEN/GBG`——单页出现的非核心令牌色远低于 audit「同色 ≥10 页且 ≥25% 页 = 系统性漂移」阈值即可，属数据可视化的合法配色。

## 五、布局踩坑（audit 会抓的硬伤）
- **负坐标 = FAIL**：环形 / 多边形布局（五边形循环、菱形闭环、hub-spoke）的节点算出**负 x/y** = 跑出画布，audit「带文字负坐标形状」判 FAIL。环形布局的中心 `cx` 与半径 `rx` 要保证**最外侧节点 `x - W/2 ≥ 0.2`**、且右侧不压说明面板（实测五步循环 `rx=2.85` 安全、`3.4` 越界）。
- 别让节点 / 比例条压到底部收束条：阶梯卡、比例条、金字塔算好总高，给底部 `band` 留位。
- **克隆既有页元素按 shape id 定位（v3.15，真实踩坑）**：给对练页补卡这类"克隆同页既有元素再改文字"的操作，定位参照行**用 shape id 指名**，不用"同 x 坐标"类几何启发式——页顶标题 shape 恰好同 x 就会被混进行集合，克隆的底卡落错行（2026-07-20 实例，渲染前几何自查抓住）。克隆后**必做几何自查**：列出全部 shape 的 y / 高度对照行网格核对无误，再进渲染。
- **成例可抄**：三列卡对比、两栏 vs、hub-spoke、流水线、漏斗、铁三角、状态机有向图、菱形/五边形循环、比例条、量级条、折线/尖峰小图、分层栈、决策树——这些图型库里都有现成实现（各 `_maintenance/*_figs.py`），照抄改数据最快。

## 六、嫁接与 app.xml（改成品安全，详见 ppt-design-system）
- **只做 zip 级最小手术，绝不用 python-pptx 整包 round-trip 成品**（会保留生成器病根、Mac PowerPoint 弹修复）。`kb_insert` 逐部件复制原件字节，只改 4 个索引（`presentation.xml` / `.rels` / `[Content_Types].xml` / `docProps/app.xml`）+ 加新页；新页是 python-pptx 独立生成的干净 slide（形状全 srgbClr、rels 指 `slideLayout1`）。
- **多对 app.xml 册**（PowerPoint 亲手 Repair 过的册，带 `notesMaster` + 双 `theme`）：其 `HeadingPairs` 是 `[字体数, 主题数, 幻灯片标题数]` **三对**、`kb_insert` 只改标题数、`TitlesOfParts.size` = 三者之和。插页后**必核验四项一致**：`Slides` == 幻灯片标题数、`HeadingPairs` 各计数之和 == `TitlesOfParts.size` == 实际 `lpstr` 个数，且 `notesMasterIdLst` 仍在 `sldIdLst` **之前**（只有 PowerPoint 校验它，漏了就弹）。
- **页码册**（页脚带「第 X 章 · 页码」或「p.N」，如 AI-Gateway / Security / LLM / Prompt-Engineering / Evaluation）：配图页用**图系统统一页脚（无页码）**，接受插入点之后真实页码轻微漂移——**保图系统跨册一致 > 页码严丝合缝**（无 `renumber_footers` 工具时的既定取舍）；要页码严丝合缝，得写一个按放映序重排全册页脚数字的 `renumber_footers`。

## 七、页脚与眉题
- 图的 `MOD = "<模块>讲义 · <副题>"`、`footer(MOD, "第 N 章 · xxx")`；匹配该模块**无页码时**的页脚口径。眉题（顶部页眉）绑 KB-CONFIG「知识库显示名」（见 ppt-design-system）。

## 八、页数账回刷清单（v3.9 铁律：配图 / 删页改了总页数，收尾必跑）

本节是配图/删页任务的**切片**——只展开「讲义页数」这一行账；全库派生账的账目全集与真源
定义见 `library-rules.md`「派生账登记表」（v3.13）。

配图（插页）与替代型删源页都会改变一册讲义的**放映序总页数**。这册的最终页数一旦与配图前不同，
**四个"派生页数账"面必须同步回刷到实测值**，否则库里会出现同一册三四个页数打架（v3.9 就是清理这种漂移：
配图 07-14 的增页从没回刷过模块 README 与一页纸，删源页那轮只回刷了 MANIFEST）。逐面核对，缺一不可：

1. **模块 `MANIFEST.md`**：`最后更新`叙事里记本轮页数变化（`旧→新 页`），数值以**放映序实测**为准
   （`unzip -l x.pptx | grep -cE 'ppt/slides/slide[0-9]+\.xml$'`）；
2. **模块 `README.html`**：`（主力成品，NN 页…）`（个别册页数嵌在 `✅ NN 页` span 内）改到实测；
3. **顶层 `README.html`** 模块表该行 `✅（NN 页）`；
4. **`_prep/` 库级产物**：`全库一页纸.html` 该模块 `<span class="pg">NN 页</span>`；若全库总数变，改一页纸标题/页脚总数与 `_prep/MANIFEST.md`。

一次配图多册就逐册过一遍。**只改页数、不改内容日期**：配图/删页是呈现回刷、模块内容没变——README「更新日期」、
MANIFEST 内容态维持原样，别假装内容改过。册多时可写一段临时脚本按「模块→实测页数」映射批量回刷（改完即删），
再交叉核对四面一致。零成本兜底：季度巡检的呈现轴已加「派生页数账 vs 实测」核对（见 `patrol-rules.md`），
但那是事后兜底，账要在配图收尾当场回刷。

## 工具（随库 `_maintenance/`，本技能 `scripts/` 亦备一份）
- `scripts/kb_draw.py` — 绘图库：设计系统令牌 + 形状 helper（`node/arrow/band/label/para/dot/line`…）+ 若干成例 `draw_*` 函数（可直接抄改）。
- `scripts/kb_insert.py` — 通用 zip 级插页嫁接：`insert_figures(src, dst, specs)`，中英文 app.xml 计数、单对 / 多对 HeadingPairs 全通吃。
- `scripts/kb_delete.py` — 通用 zip 级删页（kb_insert 的逆操作）：`kb_delete_pages(src, dst, del_show_pages)`，**替代型配图收尾删源文字页**用；其余部件字节不动、禁 round-trip，app.xml 单/多对、中英文 `幻灯片标题/Slide Titles` 通吃。删页定位一律用 `presentation.xml.rels` 解析 + 内容双验证（**python-pptx `part.partname` 在插过页的非规范命名册会误报放映序↔文件号**）。
- `<模块>_figs.py` 是每册的**临时绘图脚本**（scratchpad，存库 `_maintenance/`，**不进技能包**）。推广新模块 = 写它 + `kb_insert` 嫁接 + `audit` + 渲染目检 + 存 history 正式化。
