# 实战包 · 对客交锋 50 题（产品解决方案岗）

> **定位**：这不是应试材料，是实战弹药——**把每次和客户的"交锋"当作一场面试**（这也正是全库对练页
> 「客户会怎么问（面试题）」的本意）。拜访、竞标、评审、汇报、以及真正的面试，都是同一场考试。

> **本包的事实纪律**：每题答案均提炼自 18 个模块讲义的对练页（各模块有联网核实笔记与 MANIFEST 信源背书），
> **本包不自创任何事实**。每题末尾是「源指针」：`模块#章节ID`——被追问时翻到源头，那里有完整版答案与出处。
> **时效性**：带 ⏳ 的题含有会过期的数字（口径日期见标注）；引用前按源指针查源模块 MANIFEST 的最新核实日期。
> 本包由 `_prep/MANIFEST.md` 挂入季度巡检；巡检刷新源模块后，按引用映射同步本包。
> 生成：2026-07-10 · 全部答案口径截至当日

**用法**：每题答案都是 15–30 秒的「电梯版」。重要交锋（拜访/竞标/评审/面试）前通刷一遍；单个维度弱就按源指针回源补课。

## A · 格局与选型（9 题）——开场最高频，答好定基调

**A1. 现在到底谁家模型最强？** ⏳2026-07-10
先问「哪个市场、什么任务」：企业支出 Anthropic 第一（40%），消费端 ChatGPT 独占 74%，推理基准 Gemini 领先，开源榜首是 DeepSeek——「最强」永远带定语，这正是需要选型方法的原因。
→ `model-landscape#ml-map`

**A2. 开源模型和 GPT 们到底还差多少？** ⏳2026-07-10
多数基准差距已到个位数，编码基准甚至追平（SWE-bench 80.6%）。剩两个差距：最难推理任务闭源仍领先、工具链成熟度。结论是分任务用，不是二选一。
→ `model-landscape#ml-open`

**A3. 国产模型到底赶上国际水平没有？** ⏳2026-07-10
分层答：四强（GLM/Kimi/Qwen/DeepSeek）各有至少一个主流基准超越国际闭源，开源榜首整体是中国队；最难推理任务与顶级闭源还有半个身位。多数生产场景国产已够用，且价格优势显著。
→ `model-landscape#ml-china`

**A4. 豆包和 DeepSeek、Qwen 比强在哪？** ⏳2026-07-10
三家打法不同：DeepSeek 赢在价格与开源、Qwen 赢在谱系与生态、豆包赢在迭代速度与云一体——Seed 专线月更 2–4 次，加 180 万亿 tokens 调用规模验证。按场景配，不搞排位赛。
→ `model-landscape#ml-china`

**A5. GPT 和 Claude 到底该选哪个？** ⏳2026-07-10
先问任务：编码与企业 agentic 场景 Claude 系胜率高（企业编码份额 54%）；生态与多模态工具链 OpenAI 更全；成本敏感看 Gemini。建议双供应商起步、网关路由、评估集分胜负。
→ `model-landscape#ml-closed`

**A6. Llama 不是开源吗，为什么法务不让用？**
它是「开放权重 + 社区许可证」：有 7 亿 MAU 触发条款和欧盟限制。要许可证干净的替代，DeepSeek（MIT）和 Qwen（Apache 2.0）能力相当且条款更宽松。顺口纠正用词：多数「开源模型」按 OSAID 只是 open weight。
→ `model-landscape#ml-license`

**A7. 模型更新这么快，今天选的半年后会不会亏？**
会贬值，所以策略从「选对」变「可换」：网关统一接入、评估集守质量线、合同不绑单一模型——这是过去一年最重要的选型方法论变化。
→ `model-landscape#ml-selection`

**A8. 榜单第一的模型为什么你们不直接推荐？**
榜单有已证实的水分：私测多变体择优公开（Leaderboard Illusion，曾有厂商私测 27 个变体只发最优），热门基准已饱和无区分度。榜单只做初筛，终审永远是客户数据上的评估集。
→ `model-landscape#ml-selection` · `evaluation#eval-benchmarks`

**A9. thinking 模式为什么这么贵？**
推理模型按思考 token 计费，深想一次的输出量可达普通回答十倍。对策是思考预算控制（budget_tokens/thinkingBudget），且有 overthinking 实证：超过临界预算准确率反降——thinking 是拨盘不是开关。
→ `model-landscape#ml-capability`

## B · 架构与场景方案（10 题）——解决方案岗的本职

**B1. 你们和纯卖模型 API 的有什么区别？**
模型只是发动机，我们交付整车：场景架构、数据管线、评估验收、安全护栏——每个场景一张整车图纸，按客户场景对号入座。
→ `solution-patterns#sp-what-why`

**B2. 先做哪个场景最容易见效？**
通用答案是内部知识库或客服：数据现成、风险可控、ROI 好算。更负责的答法：按「数据就绪度 × 容错成本」排矩阵挑。
→ `solution-patterns#sp-what-why`

**B3. POC 成功了，为什么上生产还要再收钱？**
POC 只建了模型+数据两层样板墙；上生产要补入口、保障、运营三层——高可用、权限、审计、评估回归都是生产件，这才是工程大头。
→ `solution-patterns#sp-method`

**B4. 客服解决率能承诺多少？** ⏳2026-07-10
不先拍数字：厂商自报 51–80%，独立口径中位仅 41.2%，差在 deflection≠resolution。承诺测法：用真实工单建评估集测基线，验收线双方签字。
→ `solution-patterns#sp-customer-service`

**B5. 员工会不会搜到不该看的东西？**
这是企业搜索的第一设计约束：权限从源系统继承、查询时实时过滤、无权限连存在性都不暴露。POC 验收带越权测试集，安全团队签字才上线。
→ `solution-patterns#sp-knowledge-search`

**B6. AI 生成的内容侵权了算谁的？**
两层兜底：选商业安全定位的模型（训练数据可溯源、企业 IP 赔付条款，Firefly 模式），加发布前人审留痕，合同权责写清。
→ `solution-patterns#sp-content-gen`

**B7. RAG 和微调到底该选哪个？**
看要改什么：知识频繁更新、要可溯源 → RAG（改数据不改模型）；要定型说话风格、输出格式 → 微调。顺序永远是提示词 → RAG → 微调，左边没榨干不动右边。
→ `rag#rag-what-why` · `fine-tuning#ft-when`

**B8. 都有 1M 上下文了，还要 RAG 吗？**
要。三条理由：全量塞入按 token 计费贵得离谱；超长上下文中段有效性衰减；知识更新与权限控制只能在检索层做。长窗口和 RAG 是互补不是替代。
→ `llm#llm-inference-kv` · `rag#rag-what-why`

**B9. Agent 和普通对话机器人有什么区别、什么时候该上？**
机器人只「说」，agent 能「做」——带工具在循环里自主行动直到办完。但流程固定、要求每步可审计的场景用工作流更便宜可控；agent 留给路径写不死的开放任务。
→ `agent#agent-what-why`

**B10. 自然语言问数（ChatBI）算错一个数怎么办？**
比答错话严重，所以三道闸：高频指标只走语义层（确定性生成）、SQL 只读权限+行级过滤、答案附生成的 SQL 与口径出处可复核。裸 Text-to-SQL 只有五六成准确率，语义层是解药。
→ `rag#rag-structured`

## C · 成本账（8 题）——算得清账是解决方案岗的硬通货

**C1. 为什么按 token 计费、输出凭什么比输入贵？**
计算量真按 token 走：输入一次并行 prefill，每个输出 token 都是一轮完整前向——计费模式是架构成本的直接映射，不是定价策略。
→ `llm#llm-inference-kv`

**C2. 语音客服一分钟多少钱？** ⏳2026-07-10
级联约 $0.01–0.17/分钟、可预测；端到端未优化 $0.2–0.5，开缓存和 mini 档能压到 $0.05–0.10。报价按「分钟成本 × 并发 × 时长」算总账。
→ `multimodal#mm-voice-realtime`

**C3. 缓存能省多少钱？** ⏳2026-07-10
主流 API 缓存命中约一折到一成（读折 90% 量级），系统提示、few-shot、工具定义都是高复用段——客服/Agent 场景命中率常见 60–90%，等效价直接砍半以上。报价必问「你的提示词有多少是重复的」。
→ `prompt-engineering#pe-engineering` · `model-landscape#ml-price`

**C4. 为什么不全用最便宜的 DeepSeek？**
地板档适合批量明确任务；难推理与关键链路要更高档位兜底。三层路由（难题→旗舰、80% 走量→便宜档、敏感→自托管）总成本比全用旗舰低一半以上、比全用地板档少踩坑。
→ `model-landscape#ml-selection`

**C5. 两家报价差三倍，是不是有人坑我？**
大概率是口径差：档位、有无缓存、输入输出比例假设不同。对齐三个口径再比，可以当场用真实流量假设重算。
→ `model-landscape#ml-price`

**C6. DeepSeek 为什么那么便宜？**
架构设计出来的：MLA 把 KV 缓存压到约 1/10（单卡并发大增），MoE 让大参数每 token 只算一小部分，加自研推理工程——便宜是设计出来的，不是补贴出来的。
→ `llm#llm-attention-zoo`

**C7. 自建算力和上云到底哪个便宜？** ⏳2026-07-09
取决于利用率：盈亏线约 40–50%——能长期稳过这条线且有运维团队，自建更省；否则云更省。多数客户高估自己的利用率，建议先上云跑三个月拿真实数据再决策。
→ `ai-infra-compute#aic-econ`

**C8. 多 agent 协作成本会不会失控？** ⏳2026-07-07
会显著上升——公开口径研究类任务约 15 倍 token。所以配预算上限、按任务价值分层：高价值任务才开多 agent，日常走单 agent 或工作流。
→ `agent#agent-orchestration`

## D · 工程保障与翻车题（9 题）——答好这组 = 「这人踩过坑」

**D1. 提示注入能彻底防住吗？**
不能，这是 OWASP 和各大厂公开承认的——病根是指令与数据同通道。能做的是纵深防御：护栏拦已知手法 + 最小权限 + 高危动作人审，让注入成功也偷不走数据、动不了要害。诚实讲边界反而专业。
→ `security#sec-prompt-injection` · `prompt-engineering#pe-security`

**D2. 大模型不就是个聊天框，能有多大风险？**
只要它能读外部内容或触发动作，风险就是真的：一段精心构造的文字就可能让它泄密或误操作（EchoLeak 零点击偷 M365 Copilot 数据是真实案例）——这正是 OWASP 单列 LLM Top 10 的原因。
→ `security#sec-landscape`

**D3. 幻觉是不是训练没做好？**
根源在机制：模型是互联网的有损压缩，知识融在权重里，「解压失真」就是幻觉。训练能压低，工程上治本靠 RAG 提供可溯源事实 + 让模型学会说不知道。
→ `llm-training#llmtrain-pretrain`

**D4. 系统没报错，但用户说 AI 变笨了，怎么排查？**
这是静默退化——基础设施全绿、质量在下滑，传统 APM 抓不到。排查靠质量遥测：评估分趋势、模型版本与提示词变更记录、漂移监测。没这套只能靠猜。
→ `ai-ops#ops-what-why` · `ai-ops#ops-drift`

**D5. 上线之后你们还管吗？管什么？**
管，这是运营包五件套：在线评估运营、漂移巡检、发布门禁、事故值守、月度质量报告——每项有清单、有 SLA、有可见产出物，是订阅产品不是售后服务。
→ `ai-ops#ops-cheatsheet` · `solution-patterns#sp-method`

**D6. 模型是大厂的，还能悄悄变吗？**
能且是常态：API 背后的 checkpoint 会被静默更新，上月验收的模型不等于这月答题的模型。破法是自己的回归集每周重跑——版本变没变，分数会说话。
→ `ai-ops#ops-drift`

**D7. 改个提示词而已，为什么要走发布流程？**
一个词就能改变线上行为。流程不重：注册打 tag、评估门禁自动跑、金丝雀 5–10% 流量（且必须在金丝雀上跑自动评估——这是与传统金丝雀的关键差异）、回滚是改指针秒级生效。
→ `ai-ops#ops-release`

**D8. 你们的 AI 有没有「刹车」？**
两层：急停开关（事故频道一键暂停、保留现场）+ 硬上限（步数/预算/权限边界超限自动中断）。另一个冷知识：成本尖峰常是事故的第一信号——所以成本告警要分钟级。
→ `ai-ops#ops-incident`

**D9. 怎么证明 agent 上线后不出乱子？**
三层保障：上线前评估集全量回归 + pass^k 稳定性；上线后全链路追踪 + 在线评估盯退化；权限最小化 + 高危动作人工审批（HITL 分级）。不承诺不出错，承诺错了兜得住。
→ `agent#agent-eval-guardrails` · `ai-ops#ops-incident`

## E · 评估与验收（6 题）——所有承诺的最终落点

**E1. 榜单分数能信吗？**
能信一半：标准化统考能快速淘汰不行的，但有污染和刷榜水分。三层走：榜单初筛、Arena 看真实偏好、最后拿客户真题实测——试用期表现才是唯一作数的。
→ `evaluation#eval-benchmarks`

**E2. 让 AI 评 AI（LLM-as-a-Judge），靠谱吗？**
校准过的判官与人工一致率能到九成上下，足够规模化初筛；没校准的确实不可信。关键是流程：人工标注校准、判官与被评模型错开厂家（防自我偏好）、高风险人工终审。
→ `evaluation#eval-judge`

**E3. 我们没数据没标注团队，评估怎么起步？**
三条路并行：业务专家半天写 30 题、试运行日志采样、大模型合成再人工抽审。起步 20–50 题就能指导决策，两周从 0 到 100 题很现实。
→ `evaluation#eval-build`

**E4. RAG 答错了，是检索还是生成的锅？**
分开评：先看检索指标（该命中的文档进没进前 k），命中了再看忠实度（答案是否只基于材料）。经验上大头是检索的锅——验收时两端各设指标。
→ `evaluation#eval-scenarios` · `rag#rag-evaluation`

**E5. 评估体系要建多久、多少钱？**
最小可用两周：50 题黄金集 + 判官校准 + CI 接入，一名工程师加业务专家兼职。工具开源免费，主要成本是判官推理费——量级远低于一次返工。
→ `evaluation#eval-cheatsheet`

**E6. 换了新模型，评估要重做吗？**
题集和细则不用重做——这正是资产价值：直接跑回归对比新旧，只有判官抽查一次一致率。有评估体系的客户，换模型是一天的事。
→ `evaluation#eval-cheatsheet`

## F · 基础原理与协议（8 题）——技术深度追问的防线

**F1. 用一句话给 CEO 讲什么是大模型？**
一台「下一词预测机」：读过互联网量级的文本，把知识压进几千亿参数，生成每个词时靠注意力从上下文挑最相关的信息。能力来自规模，成本也来自规模。
→ `llm#llm-presales-map`

**F2. 为什么模型回答一个字一个字蹦？**
工作原理如此：每个词依赖前面所有词，本质串行，任何厂商都一样。流式反而是优点——首字几百毫秒就到。产品正解是拥抱流式、把首字时间做小。
→ `llm-inference#llminf-anatomy`

**F3. 私有化部署为什么并发上不去？显存明明够装模型。**
装下权重只是入场券，真正的座位是剩余显存里的 KV Cache：每路对话占一段、随轮次膨胀。并发上限 ≈ 剩余显存 ÷ 每请求 KV——这笔账可以现场算。
→ `llm-inference#llminf-kv-budget`

**F4. 量化会不会把模型变笨？** ⏳2026-07-09
FP8 校准后损失通常 0.5–2%，是 2026 年生产默认精度。稳妥做法：拿业务评测集做上线门禁——损失可测可控，不是赌博。数学/代码类任务敏感，按任务分档。
→ `llm-inference#llminf-quant`

**F5. MCP 和 Function Calling 什么关系？**
不替代：Function Calling 是模型「决定调哪个工具」的能力，MCP 是「工具怎么被标准接入」的协议（会用手 vs 统一插座标准），MCP 内部仍用 function calling 决策。
→ `mcp#mcp-what-why`

**F6. A2A 和 MCP 到底啥区别？**
MCP 连工具/数据（纵向），A2A 连 Agent（横向），一个系统常两个都用。都已捐给中立基金会（AAIF / Linux Foundation），不是私有协议。
→ `a2a#a2a-what-why`

**F7. 微调会不会把模型调坏？**
有「灾难性遗忘」风险但可控：LoRA + 保守超参 + 通用回归门禁三层保险，每版双达标（目标任务+通用能力）才放行，随时秒级回滚。
→ `fine-tuning#ft-methods` · `fine-tuning#ft-eval-deploy`

**F8. 为什么训练要几千张卡，推理几张就行？**
两笔账不同：训练每参数要存权重、梯度、优化器状态约 16 字节，数周内吃完数万亿 token；推理只装权重、一次算一批。这也是训练与推理集群分开设计、分开报价的原因。
→ `llm-training#llmtrain-infra`

## 附 · 三个「万能兜底」话术（源自各模块共同方法论）
