开始使用

三件事：每天怎么看、整库怎么分享、拿到分享的人怎么把它变成自己的知识库。

每天怎么看

双击 README.html 进首页。常用的就三个入口：

实战包

对客交锋 91 题，每题 15–30 秒的电梯版。时间紧只刷标 ★ 的 24 道必背，约 10 分钟；重要场合前通刷全量。每题「深潜」直达网页版对应章节。

网页版

19 册每册一页读完，可搜索、按层浏览；关系网告诉你讲一个模块还该带上哪几块，保鲜看板告诉你哪些数字该复查了。被问住时翻全库问答库——按一句话搜，直达答案所在处。

PPT 总览

19 册讲义与精选书单，内容出处。要投屏、要逐页讲，用这边。

整库怎么分享

分享单元是整个文件夹——网页样式、检查脚本、清单都在里面，只拷单个文件会缺东西。整个文件夹拷给对方，断网也能双击打开，全程不需要装任何软件。

要打一个干净的压缩包，跑：

python3 _maintenance/make_share.py

打出来实测 5.2 MB（2026-07-22）：带全部讲义、书单、网页版和工具，不带作者的历史留痕，也不带本机的权限配置。打包后会自动解压、在解压出来的那份里把八道检查全跑一遍，再查一遍有没有脚本写死了本机路径——确认拆开还是一个能用的库，而不只是压缩没出错。

换一台机器

两条路，结果一样。拿到之后先跑一次自检，别等出问题才发现少了东西。

① 拿到库

压缩包解压到任意目录即可，路径随便放——库里不存任何本机绝对路径。有 Git 的话 git clone 同样可以，还能带上完整改动历史。

② 先读起来

双击 README.html 就能读，断网也行，不用装任何东西。讲义 PPTX 用 PowerPoint 或 Keynote 打开。

③ 验一遍

在库根跑一次全套检查（下面那条命令）。八项全绿，说明这份拷贝是完整的：页数账、书单账、链接、样式契约、生成物、问答指针都对得上。

cd <库目录>
for g in check_kb_layout check_html_links check_page_ledger check_ebook_ledger \
         check_css_classes check_prep_coverage; do
  python3 _maintenance/$g.py >/dev/null 2>&1 && echo "OK  $g" || echo "FAIL $g"
done
python3 Web-version/build.py --check

需要什么：只要 Python 3.7 以上，用系统自带的就行——检查脚本、网页生成器、打包工具全部只用标准库，不联网、不装包。唯一的例外是改讲义 PPTX 的那批工具（配图、插页、缩放）需要 python-pptx；只读、只看网页、只跑检查都用不上它。

拿到分享的人：把它变成你自己的知识库

这个库不是死文档——维护它的那套方法打包在库里（_skill-source/knowledge-base-builder.skill），装上就能在这份内容的基础上继续建你自己的库。三步：

① 装技能

把 _skill-source/knowledge-base-builder.skill 上传到 claude.ai （设置 → Capabilities → Skills）；用 Claude Code 的话，解压到 ~/.claude/skills/knowledge-base-builder/。库根的 CLAUDE.md 还列了两个可选插件，装上更顺手，不装也不影响。

② 改口味

打开 KB-CONFIG.md，改「读者画像」「知识领域」「知识库显示名」——库的全部个性都在这一个文件里，改完下次任务就生效。

③ 开始说话

对 Claude 说「把 ×× 整理进知识库」——一个概念、一段对话、一份资料都行。它会自己判断该新增模块还是更新已有的，做完自动跑检查。

之后这份库就是你的了：加你的模块、删你不要的、按你的领域改书架——和原库各自演化，互不影响。

库的约定（接手前值得知道的）

东西在哪一句话
每个模块的清单PPT-version/<模块>/MANIFEST.md 章节、会过期的事实（带核实日期）、模块间关系都登记在这，网页版由它生成
检查脚本_maintenance/ 页数账、书单链接、坏链、讲义规范……改完东西跑一遍，全绿再算完
素材留痕PPT-version/<模块>/raw-data/ 每次输入和联网核实笔记存在这，事实可以溯源
网页版Web-version/ 知识地图、关系网、保鲜看板都从清单自动生成——用技能更新内容时会顺手重建，不用手动管

书单只列正规渠道链接、不含第三方文件，整库可以放心分享。
