# KB-CONFIG · 知识库配置

> 这个库的"个性"都在这里：技能每次任务第 0 步读它，产出跟着它走。
> 想改口味直接改"值"列，下次任务生效；字段名不要动。

| 字段 | 值 |
| --- | --- |
| 配置版本 | 1 |
| 知识库根目录 | 本文件所在目录（整库可整包拷走，不写绝对路径） |
| 知识领域 | AI 技术（大模型应用） |
| 读者画像 | 售前技术：有 Python/API 基础，重底层概念与取舍，不啃重理论 |
| 对练页提问者 | 客户（页标题：客户会怎么问（面试题）） |
| 工作场景挂钩 | 上云落地：技术环节 → 主流云服务对照，有真实关联才写 |
| 选书口味 | 偏实践少理论；理论类全库最多 1–2 本，标"可选读" |
| 语言策略 | 中文为主；缩写型术语首次出现展开英文全称（缩写，英文全称），译名视语境 |
| PPT 面根目录 | PPT-version |
| Web 面根目录 | Web-version |
| 知识库显示名 | AI 知识库（讲义封面眉题等成品对外署名用） |
| 阅读场景 | 自读（紧凑字号档；投屏授课需全库重刷换档） |
| 巡检阈值 | 90 天 |
| 巡检节奏 | 季度（定时任务 kb-quarterly-patrol，1/4/7/10 月 1 日 10:00） |

## 知识地图层定义
| 层 | 放什么 |
| --- | --- |
| 解决方案层 | 面向对客决策的知识：场景方案与模型选型（2026-07-10 增设） |
| 应用模式层 | 用模型解决问题的模式 |
| 协议层 | 生态互操作标准 |
| 工程保障层 | 让系统可靠的工程手段 |
| 基础层 | 模型与提示词原理 |
| 算力底座层 | 模型层以下的硬件与平台 |
| 数据底座层 | 数据就绪工程：解析、同步、向量库、质量与标注运营（2026-07-11 增设） |
