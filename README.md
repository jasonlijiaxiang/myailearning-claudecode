# 我的 AI 知识库

面向"售前技术"角色的持续扩充知识库。一套知识、两面呈现：**PPT 面**是内容真源（讲义 + 书单），**Web 面**是网页延伸。

当前规模：**19 个模块 / 1732 页讲义**，七层书架。

> **在线浏览提示**：本库的门户与各模块索引是 HTML，GitHub 不渲染 HTML，点开只会看到源码。
> 完整体验请 clone 到本地后打开 [`README.html`](README.html)。

## 目录约定（v4.0 一源两面）

| 目录 | 作用 |
|---|---|
| `PPT-version/` | 内容真源，19 个模块目录整体在此（讲义 pptx + README + MANIFEST + 电子书书单） |
| `Web-version/` | 网页派生站点，待建 |
| `_reference/` | 库级参考材料与电子书馆藏 |
| `_prep/` | 库级取用产物（全库一页纸、学习路径、实战包） |
| `_maintenance/` | 巡检报告与门禁脚本 |
| `_skill-source/` | 知识库构建技能源码 |

每个模块目录下的 `MANIFEST.md` 是该模块的唯一账本；`_prep/MANIFEST.md` 以「引用映射」挂靠各源模块，源模块事实刷新后需同步顶部三份取用文档。

## 未纳入版本库的内容

以下两类留在本地，不同步到 GitHub（见 [`.gitignore`](.gitignore)）：

- **`_reference/ebooks/`（302M，72 份）** —— 第三方版权材料，不对外分发。
  因此各模块《电子书书单.md》中指向 PDF 的链接在 GitHub 上不可用；**书单本身完整保留**，按条目名自行取书即可，本地则一切照常。
- **`PPT-version/*/raw-data/history/`（46M，约 170 份历史 pptx）** —— 讲义历史快照。
  pptx 为二进制不可 diff，每次提交都会存整份副本；版本历史交由 git 本身承担。

各模块 `raw-data/` 下的建设笔记（markdown）**予以保留**，那是每册讲义的来龙去脉。
